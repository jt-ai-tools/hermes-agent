# 問題分類法 (Issue Taxonomy)

使用此分類法對 dogfood QA 測試期間發現的問題進行分類。

## 嚴重性分級 (Severity Levels)

### Critical (緊急)
該問題導致核心功能完全無法使用，或導致資料遺失。

**範例：**
- 應用程式當機或顯示空白頁面
- 表單提交時靜默遺失使用者資料
- 驗證功能完全損壞（完全無法登入）
- 付款流程失敗且在未完成訂單的情況下扣款
- 安全漏洞（例如：XSS、主控台洩漏憑據）

### High (高)
該問題嚴重損害功能，但可能存在替代方案 (workaround)。

**範例：**
- 點擊關鍵按鈕時無反應（但重新整理後正常）
- 搜尋有效查詢時卻傳回無結果
- 表單驗證拒絕有效的輸入
- 頁面已載入但關鍵內容缺失或雜亂無章
- 導航連結指向 404 或錯誤頁面
- 核心頁面上的主控台出現未捕獲的 JavaScript 異常

### Medium (中)
該問題顯而易見且影響使用者體驗，但不會阻礙核心功能。

**範例：**
- 佈局在某些螢幕區塊對齊錯誤或重疊
- 圖片載入失敗（損壞的圖片圖示）
- 效能緩慢（明顯的載入延遲 > 3 秒）
- 表單欄位缺乏正確的驗證回饋（輸入錯誤時沒有錯誤訊息）
- 提示功能已過時或配置錯誤的主控台警告
- 相似頁面之間的樣式不一致

### Low (低)
不影響功能的微小修飾問題。

**範例：**
- 文字內容中的錯字或語法錯誤
- 微小的間距或對齊不一致
- 生產環境中遺留預設文字 ("Lorem ipsum")
- 缺少網站圖示 (Favicon)
- 不應出現在生產環境中的主控台資訊/除錯訊息
- 未違反 WCAG 要求的細微色彩對比問題

## 類別 (Categories)

### Functional (功能)
功能未按預期運作的問題。

- 無反應的按鈕/連結
- 未提交或提交不正確的表單
- 損壞的使用者流程（無法完成多步驟程序）
- 顯示的資料不正確
- 僅部分運作的功能

### Visual (視覺)
頁面視覺呈現的問題。

- 佈局問題（元素重疊、格線損壞）
- 圖片損壞或媒體缺失
- 樣式不一致
- 響應式設計失敗
- Z-index 問題（元素被其他元素遮擋）
- 文字溢出或截斷

### Accessibility (無障礙)
阻止或阻礙身心障礙使用者存取的問題。

- 有意義的圖片缺少替代文字 (alt text)
- 色彩對比度差（未達 WCAG AA 標準）
- 無法透過鍵盤導航到達的元素
- 缺少表單標籤或 ARIA 屬性
- 焦點指示器缺失或不明確
- 不相容螢幕閱讀器的內容

### Console (主控台)
透過 JavaScript 主控台輸出偵測到的問題。

- 未捕獲的異常與未處理的 promise rejection
- 失敗的網路請求（主控台中的 4xx, 5xx 錯誤）
- 過時警告 (Deprecation warnings)
- CORS 錯誤
- 混合內容警告（HTTPS 頁面上的 HTTP 資源）
- 開發環境遺留的過多 console.log 輸出

### UX (使用者體驗)
功能正常但體驗不佳的問題。

- 令人困惑的導航或資訊架構
- 缺少載入指示器（使用者不知道正在處理中）
- 使用者操作後無回饋（例如：點擊按鈕後無視覺結果）
- 不一致的互動模式
- 破壞性操作缺少確認對話框
- 無法幫助使用者恢復操作的差勁錯誤訊息

### Content (內容)
頁面文字、媒體或資訊方面的問題。

- 錯字和語法錯誤
- 生產環境中的預設/虛擬內容
- 過時資訊
- 內容缺失（空白區段）
- 到外部資源的損壞或失效連結
- 不正確或具誤導性的標籤
