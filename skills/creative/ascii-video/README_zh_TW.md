# ☤ ASCII 影片 (ASCII Video)

將任何內容渲染為彩色 ASCII 字元影片。輸入音訊、影片、圖片、文字或純數學公式，輸出 MP4/GIF/PNG 序列。每個字元單元皆支援完整 RGB 色彩，預設 1080p 24fps。無需 GPU。

專為 [Hermes Agent](https://github.com/NousResearch/hermes-agent) 打造。可用於任何編碼代理程式。規範源代碼位於此處；透過 PR 同步至 [`NousResearch/hermes-agent/skills/creative/ascii-video`](https://github.com/NousResearch/hermes-agent/tree/main/skills/creative/ascii-video)。

## 這是什麼

這是一項教導代理程式如何從頭開始構建單一檔案 Python 渲染器以製作 ASCII 影片的技能。代理程式可以獲得完整的流水線：網格系統、字體點陣化、效果庫、著色器鏈、音訊分析、並行編碼。它會編寫渲染器並執行它，最後獲得影片。

輸出是真正的影片檔案，而非終端機轉義碼 (terminal escape codes)。影格被計算為彩色字元網格，使用預先點陣化的字體位圖合成到像素畫布上，經過著色器後處理，最後透過管道傳送至 ffmpeg。

## 模式

| 模式 | 輸入 | 輸出 |
|------|-------|--------|
| 影片轉 ASCII | 影片檔案 | 原始素材的 ASCII 重現 |
| 音訊反應式 | 音訊檔案 | 由頻段、節拍、能量驅動的視覺效果 |
| 生成式 | 無 | 來自數學公式的程序化動畫 |
| 混合式 | 影片 + 音訊 | 帶有音訊反應疊加層的 ASCII 影片 |
| 歌詞/文字 | 音訊 + 定時文字 (SRT) | 帶有效果的卡拉 OK 風格文字 |
| TTS 旁白 | 文字引用 + API 金鑰 | 帶有打字機文字和生成語音的旁白影片 |

## 流水線 (Pipeline)

每種模式都遵循相同的 6 階段路徑：

```
INPUT --> ANALYZE --> SCENE_FN --> TONEMAP --> SHADE --> ENCODE
```

1. **Input (輸入)** 載入原始素材（生成模式則不載入）。
2. **Analyze (分析)** 提取每格特徵。音訊獲得 6 頻段 FFT、RMS、頻譜質心、平坦度、通量以及帶有指數衰減的節拍檢測。影片獲得亮度、邊緣、運動。
3. **Scene function (場景函式)** 直接返回像素畫布。在不同密度下組合多個字元網格、數值/色調場、像素混合模式。這是視覺效果產生的地方。
4. **Tonemap (色調映射)** 進行基於百分位數的自適應亮度歸一化，並帶有每場景 Gamma 調節。黑底上的 ASCII 本質上是暗的，線性倍數增益無效，但此方法有效。
5. **Shade (著色)** 執行一個 `ShaderChain`（38 種可組合著色器）加上一個用於帶有空間變換的時間遞迴 `FeedbackBuffer`。
6. **Encode (編碼)** 將原始 RGB 影格透過管道傳送至 ffmpeg 進行 H.264 編碼。片段會被連接，音訊會被混流。

## 網格系統

字元渲染在固定大小的網格上。可以疊加多層密度以增加深度。

| 大小 | 字體 | 1080p 下的網格 | 用途 |
|------|------|---------------|-----|
| xs | 8px | 400x108 | 超密集數據場 |
| sm | 10px | 320x83 | 雨、星場 |
| md | 16px | 192x56 | 預設平衡 |
| lg | 20px | 160x45 | 可讀文字 |
| xl | 24px | 137x37 | 大型標題 |
| xxl | 40px | 80x22 | 巨型極簡 |

在 `sm` 和 `lg` 上渲染相同場景並進行 Screen 混合，會產生自然的紋理干涉。精細的細節會從粗糙字元的間隙中顯現。大多數場景使用兩到三個網格。

## 數值場生成器 (21)

數值場是核心視覺構建塊。每個生成器會產生一個 [0, 1] 範圍內的 2D 浮點數陣列，將每個網格單元映射到一個亮度值。

### 三角函數型 (12)

| 數值場 | 描述 |
|-------|-------------|
| 正弦場 (Sine field) | 分層多重正弦干涉，通用背景 |
| 平滑雜訊 (Smooth noise) | Perlin 雜訊的多倍頻正弦逼近 |
| 圓環 (Rings) | 同心圓環，由低音驅動數量和晃動 |
| 螺旋 (Spiral) | 對數螺旋臂，可配置臂數/緊密度 |
| 隧道 (Tunnel) | 無限深度透視（反距離） |
| 渦流 (Vortex) | 扭曲的放射狀圖案，角度隨距離調變 |
| 干涉 (Interference) | N 個重疊正弦波產生的莫列波紋 (Moire) |
| 極光 (Aurora) | 水平流動的帶狀物 |
| 波紋 (Ripple) | 來自可配置源點的同心波 |
| 電漿 (Plasma) | 多個方向/速度的正弦波總和 |

## 色調場生成器 (9)

決定每個單元的顏色（獨立於亮度）：固定色調、角度對應彩虹、距離漸層、時間循環旋轉、音訊頻譜質心、水平/垂直漸層、電漿變化、感知均勻的 OKLCH 彩虹。

## 時間連貫性

38 種可組合著色器，在字元渲染後套用於像素畫布。可按段落配置。

| 通道 | 色差 (節拍反應), 通道偏移, 通道交換, RGB 徑向分離 |
| 色彩 | 反相, 海報化, 閥值, 曝光過度, 色調旋轉, 飽和度, 調色, 色彩晃動, 色彩映射 |
| 發光/模糊 | Bloom, 邊緣發光, 柔焦, 徑向模糊 |
| 雜訊 | 電影膠片顆粒 (節拍反應), 靜電雜訊 |
| 線條/圖案 | 掃描線, 半色調 |
| 階調 | 暈影 (Vignette), 對比度, Gamma, 色階, 亮度 |

## 混合模式與構圖

20 種用於圖層畫布的像素混合模式：normal, add, subtract, multiply, screen, overlay, softlight, hardlight, difference, exclusion, colordodge, colorburn, linearlight, vividlight, pin_light, hard_mix, lighten, darken, grain_extract, grain_merge。支援 sRGB 和線性光 (linear-light) 混合。

**回饋緩衝區 (Feedback buffer)**。時間遞迴 —— 每一格影格與前一格影格的變換版本混合。7 種空間變換：放大、縮小、順時針/逆時針旋轉、向上/下平移、鏡像。每影格可選色調偏移以實現彩虹軌跡。每場景可配置衰減、混合模式和不透明度。

**遮罩 (Masking)**。16 種用於空間合成的遮罩類型：形狀遮罩（圓形、矩形、圓環、漸層）、程序化遮罩（任何數值場作為遮罩、文字模板）、動畫遮罩（光圈開/關、擦除、溶解）、布林運算（聯集、交集、差集、反相）。

**過場 (Transitions)**。交叉淡化、方向性擦除、徑向擦除、溶解、故障剪輯。

## 場景設計模式

用於製作看起來是有意為之而非隨機生成的場景的構圖模式。

**圖層階層**。背景（微暗的氛圍，密集網格）、內容（主要視覺，標準網格）、強調（稀疏高光，粗糙網格）。三個截然不同的角色，而非三個相互競爭的圖層。

**方向性參數弧**。每個場景的定義參數在其持續時間內會逐漸增加、加速或累積。基於進度的公式（線性、緩出、階梯式揭示）取代了漫無目的的 `sin(t)` 震盪。

**場景概念**。場景圍繞視覺隱喻構建（湧現、下降、碰撞、熵），具有動機明確的圖層/調色盤/回饋選擇。不以其效果命名。

**構圖技術**。反向旋轉的雙系統、波浪碰撞、漸進式碎片化（Voronoi 單元隨時間增加）、熵（幾何圖案被反應擴散侵蝕）、交錯圖層進入（漸強累積）。

## 硬體適應

自動檢測 CPU 數量、記憶體、平台、ffmpeg。適應工作進程數、解析度、FPS。

| 設定檔 | 解析度 | FPS | 適用時機 |
|---------|-----------|-----|------|
| `draft` | 960x540 | 12 | 檢查時機/佈局 |
| `preview` | 1280x720 | 15 | 審閱效果 |
| `production` | 1920x1080 | 24 | 最終輸出 |

## 已知陷阱

**亮度**。ASCII 字元是黑底上的微小亮点。大多數影格像素是背景。線性 `* N` 倍數會剪裁高光並使畫面泛白。請改用帶有每場景 Gamma 調節的 `tonemap()`。預設 Gamma 為 0.75，曝光過度場景 0.55，海報化 0.50。

**渲染瓶頸**。合成字體點陣圖的每單元 Python 迴圈執行速度約為每格 100-150ms。若不使用 Cython/C 則無法避免。除此之外的一切都必須使用向量化 numpy。在效果函式中使用 Python 對行/列進行 for 迴圈會大幅降低效能。

**ffmpeg 死鎖**。在長時間運行的編碼中，絕不要使用 `stderr=subprocess.PIPE`。緩衝區在約 64KB 時會填滿，導致進程掛起。請將 stderr 重定向到檔案。

**字體單元高度**。Pillow 的 `textbbox()` 在 macOS 上返回錯誤的高度。請使用 `font.getmetrics()` 獲取 `ascent + descent`。

**字體相容性**。並非所有 Unicode 都能在所有字體中渲染。調色盤在初始化時會進行驗證，空白字形會被靜默移除。
