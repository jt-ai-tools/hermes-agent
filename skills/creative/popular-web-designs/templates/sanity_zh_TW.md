## 1. 視覺主題與氛圍

Sanity 的網站被呈現為一個開發者內容平台的「夜間指揮中心」 —— 深色、精確且結構嚴謹。整個體驗建立在近乎黑色的畫布 (`#0b0b0b`) 之上，這與其說是一個「深色模式切換」，不如說是為那些生活在終端機中的人們量身打造的自然狀態。大多數 CMS 行銷頁面追求親和的粉彩色調和柔和的插畫，而 Sanity 則擁抱其產品本身的莊重感：結構化的內容值得一個結構化的舞台。

其標誌性的排版風格是 waldenburgNormal —— 一種獨特、略帶幾何感的無襯線體，在展示尺寸下具有極致的負字距（-0.32px 至 -4.48px），賦予標題一種壓縮、工程化的特質。在 112px 的主視覺規模下搭配 -4.48px 的字距，字體感覺幾乎是被機械加工出來的 —— 就像精確切割的鋼鐵字形。這與 IBM Plex Mono 配對，用於程式碼和技術標籤，創造出一種雙重語境：編輯權威感與開發者可信度的結合。

Sanity 的獨特之處在於其單色深色調與鮮豔、高飽和度強調色之間的相互作用。中性色階從純黑色到嚴格控制的灰色梯度 (`#0b0b0b` -> `#212121` -> `#353535` -> `#797979` -> `#b9b9b9` -> `#ededed` -> `#ffffff`)，沒有冷暖偏向 —— 只有純粹、無彩色的精確感。在這種嚴謹的背景下，霓虹綠強調色 (display-p3 green) 和電藍色 (`#0052ef`) 的出現如同深色控制室中的信號燈一般醒目。橘紅色的 CTA (`#f36458`) 則為這個冷色調系統提供了唯一的暖色點綴。

**關鍵特徵：**
- 以近黑色畫布 (`#0b0b0b`) 作為預設的自然環境 —— 這不是深色「模式」，而是主要身份
- waldenburgNormal 在展示尺寸下使用極端的負字距，營造出精密工程化的字體風格
- 純粹的無彩色灰階 —— 無冷暖色調，展現純粹的中性紀律
- 鮮豔的強調色點綴：在深色背景上使用霓虹綠、電藍色 (`#0052ef`) 和珊瑚紅 (`#f36458`)
- 藥丸型主要按鈕 (99999px 圓角)，與次要操作使用的細微圓角矩形 (3-6px) 形成對比
- IBM Plex Mono 作為編輯用展示字體的技術平衡點
- 滿版深色區塊，內容包含在經過測量的最大寬度容器中
- 所有互動元素的懸停狀態 (Hover) 皆轉向電藍色 (`#0052ef`) —— 統一的「激活」信號

## 2. 色彩調色盤與角色

### 主要品牌色
- **Sanity Black** (`#0b0b0b`): 主要畫布與主導表面色。並非純黑，但已足夠接近絕對。整個視覺識別的基石。
- **純黑色** (`#000000`): 用於最高對比時刻、深色覆蓋層和某些框線強調。
- **Sanity Red** (`#f36458`): 主要行動呼籲 (CTA) 與品牌強調色 —— 一種珊瑚紅，作為主要轉換點顏色。用於「Get Started」按鈕。

### 強調與互動色
- **電藍色 (Electric Blue)** (`#0052ef`): 整個系統通用的懸停/活動狀態顏色。按鈕、連結和互動元素在懸停時皆轉為此藍色。也用作 `--color-blue-700`，用於聚焦環和活動狀態。
- **淺藍色** (`#55beff` / `#afe3ff`): 次要藍色變體，用於強調背景、標籤和淡化的藍色表面。
- **霓虹綠 (Neon Green)** (`color(display-p3 .270588 1 0)`): 鮮豔的廣色域綠色，用作 `--color-fg-accent-green`，用於成功狀態和高端功能亮點。在 sRGB 中回退至 `#19d600`。
- **強調洋紅色 (Accent Magenta)** (`color(display-p3 .960784 0 1)`): 用於特殊強調時刻的鮮豔廣色域洋紅色。

### 表面與背景
- **近黑色** (`#0b0b0b`): 預設網頁背景與主要表面。
- **深灰色** (`#212121`): 用於卡片、次要容器、輸入框背景的提升表面色，在基礎畫布之上提供微妙分層。
- **中深灰色** (`#353535`): 三級表面與框線色，用於在深色層之間營造深度。
- **純白色** (`#ffffff`): 用於反轉區塊、深色背景上的淺色文字，以及特定的按鈕表面。
- **淺灰色** (`#ededed`): 用於反轉/淺色區塊的淺色表面及微妙的背景色調。

### 中性色與文字
- **白色** (`#ffffff`): 深色表面上的主要文字顏色，確保最高可讀性。
- **銀色** (`#b9b9b9`): 次要文字、深色表面上的正文、柔和的說明文字與佔位符文字。
- **中灰色** (`#797979`): 三級文字、元數據 (metadata)、時間戳記與弱化內容。
- **木炭黑 (Charcoal)** (`#212121`): 淺色/反轉表面上的文字。
- **近黑色文字** (`#0b0b0b`): 白色/淺色按鈕表面上的主要文字。

### 語義色彩
- **錯誤紅** (`#dd0000`): 破壞性操作、驗證錯誤與關鍵警告 —— 純粹的高飽和度紅色。
- **GPC 綠** (`#37cd84`): 隱私/合規指標綠色。
- **聚焦環藍** (`#0052ef`): 無障礙設計的聚焦環顏色，與互動藍色一致。

### 框線系統
- **深色框線** (`#0b0b0b`): 深色容器上的主要框線 —— 幾乎看不見，保持極簡邊界。
- **微妙框線** (`#212121`): 深色表面上輸入框、文字區域和卡片邊緣的標準框線。
- **中度框線** (`#353535`): 更明顯的框線，用於強調邊界和分隔線。
- **淺色框線** (`#ffffff`): 用於反轉/淺色元素，或需要對比區隔的按鈕框線。
- **橘色框線** (`color(display-p3 1 0.3333 0)`): 用於突出/特色元素的特殊強調框線。

## 3. 字體排版規則

### 字體族系
- **展示 / 標題**: `waldenburgNormal`，備用字體：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **正文 / UI**: `waldenburgNormal`，備用字體：`waldenburgNormal Fallback, ui-sans-serif, system-ui`
- **程式碼 / 技術**: `IBM Plex Mono`，備用字體：`ibmPlexMono Fallback, ui-monospace`

### 原則
- **大規模下的極端負字距**: 72px 以上的展示標題使用激進的負字距（-2.88px 至 -4.48px），營造出一種緊湊、工程化的特質，使 Sanity 與鬆散的編輯排版區隔開來。
- **單一字體，多重語境**: waldenburgNormal 同時處理編輯用的展示字體和功能性的 UI 文字。字重範圍狹窄（多數為 400-425，僅微型標籤使用 500-600），保持語調一致。
- **OpenType 特性控制**: 排版精確使用特性設置，包括展示尺寸的 `"cv01", "cv11", "cv12", "cv13", "ss07"` 以及正文的 `"calt" 0`，為不同語境微調字元替代方案。
- **緊湊標題，寬鬆正文**: 標題使用 1.00-1.24 行高（極度緊湊），而正文則在 1.50 行高下展現空間。這種對比創造了清晰的視覺層級。
- **技術標籤使用大寫**: IBM Plex Mono 圖說和小型標籤經常使用 `text-transform: uppercase` 配合緊湊行高，營造出一種技術元數據的「系統讀數」美學。

## 5. 版面原則

### 間距系統
基礎單位：**8px**

| Token | 數值 | 用途 |
|-------|-------|-------|
| space-1 | 1px | 極細間隙，類框線間距 |
| space-2 | 2px | 極小內部內距 |
| space-3 | 4px | 緊湊元件內部間距 |
| space-4 | 6px | 小型元素間隙 |
| space-5 | 8px | 基礎單位 —— 按鈕內距、輸入框內距、標籤內距 |
| space-6 | 12px | 標準元件間隙、按鈕水平內距 |
| space-7 | 16px | 區塊內部內距、卡片間距 |
| space-8 | 24px | 大型元件內距、卡片內部間距 |
| space-9 | 32px | 區塊內距、容器欄間距 (Gutters) |
| space-10 | 48px | 大型區塊垂直間距 |
| space-11 | 64px | 主要區塊分隔 |
| space-12 | 96-120px | 主視覺垂直內距、最大區塊間距 |

### 網格與容器
- 最大內容寬度：約 1440px（從斷點推斷）
- 網頁欄間距 (Gutter)：桌面端 32px，行動端 16px
- 內容區塊使用滿版背景，內容居中且具備最大寬度
- 多欄佈局：桌面端 2-3 欄，行動端單欄
- 卡片網格：使用 CSS Grid，間隙一致 (16-24px)

### 留白哲學
Sanity 在區塊之間使用激進的垂直間距 (64-120px)，在深色畫布上營造喘息空間。在區塊內部，間距較為緊湊 (16-32px)，創造出被慷慨留白分隔開的密集資訊群。這種節奏賦予網頁一種「投影片」的特質 —— 每個區塊感覺都像是一個專注的畫面。

### 圓角半徑級距

| Token | 數值 | 用途 |
|-------|-------|-------|
| radius-xs | 3px | 輸入框、文字區域、微妙圓角 |
| radius-sm | 4-5px | 次要按鈕、小型卡片、標籤 |
| radius-md | 6px | 標準卡片、容器 |
| radius-lg | 12px | 大型卡片、特色容器、表單 |
| radius-pill | 99999px | 主要按鈕、徽章、導覽藥丸標籤 |

## 6. 深度與層次

### 陰影系統

| 層級 | 數值 | 用途 |
|-------|-----------|-----|
| Level 0 (扁平) | 無 | 多數元素的預設狀態 —— 深色表面僅透過色彩創造深度 |
| Level 1 (微妙) | 0px 0px 0px 1px `#212121` | 類框線陰影，用於極簡邊界而不顯示明顯框線 |
| Level 2 (聚焦) | 0 0 0 2px `var(--color-blue-500)` | 輸入框與互動元素的聚焦環 |
| Level 3 (疊加) | 背景模糊 + 半透明深色 | 導覽疊加、彈窗背景 |

### 深度哲學
Sanity 的深度系統幾乎完全是**色彩式**而非陰影式的。層次透過表面色彩的切換來傳達：`#0b0b0b` (地面) -> `#212121` (提升) -> `#353535` (突出) -> `#ffffff` (反轉/最高)。這種方法在深色介面中很直覺，因為傳統的投影會變得不可見。少數存在的陰影是環狀的 (0px 0px 0px Npx) 或基於模糊的 (backdrop-filter)，而非偏移陰影，以此保持扁平、精密工程化的美學。

基於框線的界限 (1px solid `#212121` 或 `#353535`) 作為主要的空間分隔器，框線的暗度經過校準，使其可見但不主導。系統避免使用「漂浮卡片」美學 —— 一切都感覺是安裝在表面上，而非懸浮其上。

## 7. 規範與禁止 (Do's and Don'ts)

### 規範
- 使用無彩色灰階作為基礎 —— 保持純粹的中性紀律，不帶冷暖色調
- 始終如一地將電藍色 (`#0052ef`) 用作所有互動元素的通用懸停/活動狀態
- 在 48px 及以上的展示標題上使用極端的負字距（-2px 至 -4.48px）
- 主要 CTA 保持全藥丸形狀 (99999px 圓角) 並使用珊瑚紅 (`#f36458`)
- 技術標籤、標籤 (Tags) 和系統元數據使用 IBM Plex Mono 大寫
- 透過表面色彩（深至淺）而非陰影來傳達深度
- 在深色畫布上保持慷慨的垂直區塊間距 (64-120px)
- 展示排版使用 `"cv01", "cv11", "cv12", "cv13", "ss07"` OpenType 特性

### 禁止
- 不要在中性色階中引入冷暖色調 —— Sanity 的灰色是純粹的無彩色
- 不要使用投影來表達層次 —— 深色介面需要色彩式深度
- 不要使用 13px 到 99998px 之間的圓角半徑 —— 系統直接從 12px (大型卡片) 跳至藥丸型 (99999px)
- 不要在同一個元素中混合使用珊瑚紅 CTA 和電藍色互動色
- 不要使用過重的字重 (700+) —— 系統最高僅為 600，且僅用於 11px 大寫標籤
- 不要在未檢查灰對灰對比率的情況下，將淺色文字放在淺色表面或深色文字放在深色表面
- 不要使用傳統的偏移盒陰影 —— 僅使用環狀陰影 (0 0 0 Npx) 或基於框線的界限
- 不要打破標題緊湊的行高 —— 範圍為 1.00-1.24，展示文字絕不超過 1.5+

## 8. 回應式行為

### 斷點 (Breakpoints)

| 名稱 | 寬度 | 行為 |
|------|-------|----------|
| 桌面 XL | >= 1640px | 完整佈局，最大內容寬度 |
| 桌面端 | >= 1440px | 標準桌面佈局 |
| 緊湊桌面 | >= 1200px | 略微壓縮的桌面端 |
| 筆記型電腦 | >= 1100px | 欄寬縮小 |
| 平板橫向 | >= 960px | 雙欄佈局開始收合 |
| 平板端 | >= 768px | 過渡區，部分元素堆疊 |
| 大型行動端 | >= 720px | 接近平板的佈局 |
| 行動裝置 | >= 480px | 單欄堆疊佈局 |
| 小型行動端 | >= 376px | 最小支援寬度 |

### 收合策略
- **導覽**：水平連結在 768px 以下收合為漢堡選單
- **主視覺排版**：在斷點之間從 112px -> 72px -> 48px -> 38px 縮放，保持緊湊的字距比例
- **網格佈局**：3 欄在 ~960px 轉為 2 欄，768px 以下轉為單欄
- **卡片網格**：在行動端使用水平捲動而非換行（以保持卡片長寬比）
- **區塊間距**：行動端垂直內距減少約 40% (120px -> 64px -> 48px)
- **按鈕尺寸**：CTA 藥丸保持內距但縮小字體；幽靈按鈕保持固定
- **程式碼區塊**：保持等寬格式並支援水平捲動

### 行動端特定調整
- 滿版區塊邊對邊擴展，內部欄間距為 16px
- 觸控目標：所有互動元素至少為 44px
- 行動端標題字距略微放鬆（負字距較不激進）
- 影像容器從固定長寬比切換為全寬自動高度
