## 1. 視覺主題與氛圍

Supabase 的網站是一個原生深色模式的開發者平台，傳遞出高級程式碼編輯器的美感 —— 深黑色背景（`#0f0f0f`、`#171717`）搭配翡翠綠強調色（`#3ecf8e`、`#00c573`），呼應了該品牌開源且帶有 PostgreSQL 綠的識別。這個設計系統感覺誕生於終端機視窗，並演化成一個精緻的行銷介面，同時又不失其開發者靈魂。

排版建立在 "Circular" 字體之上 —— 這是一種類圓角結尾的幾何無襯線體，柔化了技術的鋒芒。英雄區文字設定為 72px 且行高為 1.00，將垂直空間壓縮到了極限，創造出密集且具衝擊力的陳述，毫無冗餘。等寬字體伴隨字（Source Code Pro）少量出現於大寫的技術標籤，並配有 1.2px 的字距，營造出「開發者主控台」的標記感，將行銷網站與產品體驗聯繫在一起。

Supabase 的獨特之處在於其精緻的基於 HSL 的顏色代幣系統。Supabase 不使用扁平的十六進位值，而是為幾乎每種顏色都使用帶有 Alpha 通道的 HSL（例如 `--colors-crimson4`、`--colors-purple5`、`--colors-slateA12`），從而實現細膩的分層系統，讓色彩透過透明度相互作用。這透過半透明度創造了深度 —— `rgba(46, 46, 46)` 的邊框、`rgba(41, 41, 41, 0.84)` 的表面，以及部分不透明的強調色，都與深色背景融合，用極少的色彩成分創造出豐富且具空間感的調色盤。

綠色強調色 (`#3ecf8e`) 被有選擇性地使用 —— 出現在 Supabase 標誌、連結顏色 (`#00c573`) 和邊框亮點 (`rgba(62, 207, 142, 0.3)`) 中 —— 始終作為「這是 Supabase」的訊號，而非裝飾性元素。主要行動呼籲 (CTA) 使用藥丸狀按鈕（9999px 圓角），與次要元素的標準 6px 圓角形成對比，建立了清晰的重要程度視覺層級。

**核心特點：**
- 原生深色模式：近黑色背景（`#0f0f0f`、`#171717`）—— 絕非純黑色
- 翡翠綠品牌強調色（`#3ecf8e`、`#00c573`）少量使用作為識別標記
- Circular 字體 —— 帶圓角結尾的幾何無襯線體
- Source Code Pro 用於大寫技術標籤（1.2px 字距）
- 基於 HSL 的顏色代幣系統，配有 Alpha 通道實現半透明分層
- 主要 CTA 使用藥丸狀按鈕 (9999px)，次要項使用 6px 圓角
- 中性灰色階從 `#171717` 經過 `#898989` 到 `#fafafa`
- 邊框系統使用深灰色（`#2e2e2e`、`#363636`、`#393939`）
- 極少使用陰影 —— 透過邊框對比和透明度展現深度
- Radix 基礎顏色（緋紅、紫色、紫羅蘭、靛藍、黃色、番茄紅、橘色、板岩灰）

- **靛藍 (Indigo)**: `--colors-indigoA2` —— 微妙的藍色暈染
- **黃色 (Yellow)**: `--colors-yellowA7` —— 注意/警告
- **番茄紅 (Tomato)**: `--colors-tomatoA4` —— 錯誤強調
- **橘色 (Orange)**: `--colors-orange6` —— 暖色強調

### 表面與疊加層
- **磨砂深色 (Glass Dark)**: `rgba(41, 41, 41, 0.84)` 半透明深色疊加層
- **板岩 Alpha (Slate Alpha)**: `hsla(210, 87.8%, 16.1%, 0.031)` 極微妙的藍色暈染
- **固定階 Alpha (Fixed Scale Alpha)**: `hsla(200, 90.3%, 93.4%, 0.109)` 輕微的冰霜疊加層

### 陰影
- Supabase 在其深色主題中**幾乎不使用陰影**。深度是透過邊框對比和表面顏色差異而非 box-shadows 創造的。焦點狀態使用 `rgba(0, 0, 0, 0.1) 0px 4px 12px` —— 最小化且具功能性。

## 3. 排版規則

### 字體家族
- **主要**: `Circular`，備用字體：`custom-font, Helvetica Neue, Helvetica, Arial`
- **等寬**: `Source Code Pro`，備用字體：`Office Code Pro, Menlo`

### 原則
- **字重克制**: 幾乎所有文字都使用 400 字重 (regular/book)。500 字重僅出現於導覽連結和按鈕標籤。在檢測到的系統中沒有粗體 (700) —— 層級是透過大小而非字重建立的。
- **1.00 英雄區行高**: 英雄區文字被壓縮到絕對零間距。這是定義性的排版姿態 —— 文字感覺像終端機指令：密集、高效、不浪費垂直空間。
- **卡片上的負字距**: 卡片標題使用 -0.16px 的字距，一種微妙的緊縮感，使其在不明顯的情況下與內文區分開來。
- **等寬字體作為儀式**: 大寫且配有 1.2px 字距的 Source Code Pro 是「開發者主控台」的聲音 —— 少量用於連向產品體驗的技術標籤。
- **幾何個性**: Circular 圓潤的末端為原本可能冰冷、技術化的介面增添了溫暖。字體是人性化的元素。

### 網格與容器
- 具有寬大最大寬度的置中內容
- 全寬深色區段，具有受限的內部內容
- 功能網格：基於圖標的網格，具有一致的卡片尺寸
- 用於「信任品牌」區段的標誌網格
- 頁尾：深色背景上的多欄配置

### 斷點
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 行動裝置 | <600px | 單欄、堆疊版面 |
| 桌上型電腦 | >600px | 多欄網格、展開版面 |

*註：Supabase 使用非常簡約的斷點系統 —— 主要是單個 600px 斷點，暗示其採用行動優先且循序漸進的開發方式。*

### 留白哲學
- **戲劇性的區段間距**: 主要區段之間 90px–128px 的間距創造出電影般的步調 —— 每個區段都是深邃虛空中的獨立場景。
- **密集內容塊**: 在區段內部，間距很緊湊 (16px–24px)，創造出集中的資訊叢集。
- **邊框定義空間**: Supabase 不使用留白 + 陰影來進行區隔，而是在深色背景上使用細邊框 —— 透過「線」而非「隙」來區隔。

### 圓角比例
- 標準 (6px): 幽靈按鈕、小型元素
- 舒適 (8px): 卡片、容器
- 中等 (11px–12px): 中型面板
- 大型 (16px): 功能卡片、主要容器
- 藥丸狀 (9999px): 主要按鈕、分頁指示器

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 扁平 (Level 0) | 無陰影，邊框 `#2e2e2e` | 預設狀態，大多數表面 |
| 微妙邊框 (Level 1) | 邊框 `#363636` 或 `#393939` | 互動元素、懸停 |
| 焦點 (Level 2) | `rgba(0, 0, 0, 0.1) 0px 4px 12px` | 僅限焦點狀態 |
| 綠色強調 (Level 3) | 邊框 `rgba(62, 207, 142, 0.3)` | 品牌強調元素 |

**陰影哲學**: Supabase 有意避開陰影。在原生深色模式設計中，陰影幾乎不可見且沒有用處。相反地，深度是透過精細的邊框層級傳達的 —— 從 `#242424`（幾乎看不見）經過 `#2e2e2e`（標準）到 `#393939`（顯眼）。30% 不透明度的綠色強調邊框 (`rgba(62, 207, 142, 0.3)`) 是「提升」狀態 —— 品牌色彩本身成為了深度訊號。

## 7. 規範（Do's and Don'ts）

### 建議 (Do)
- 使用近乎黑色的背景（`#0f0f0f`、`#171717`）—— 深度來自灰色邊框層級
- 少量套用 Supabase 綠（`#3ecf8e`、`#00c573`）—— 它是識別標記，而非裝飾
- 幾乎所有內容均使用 400 字重的 Circular —— 僅按鈕和導覽使用 500
- 英雄區文字設定為 1.00 行高 —— 零間距是其排版特徵
- 透過邊框顏色差異創造深度（`#242424` → `#2e2e2e` → `#363636`）
- 主要 CTA 和分頁僅使用藥丸形狀 (9999px)
- 使用帶有 Alpha 的 HSL 顏色來實現半透明分層效果
- 使用 Source Code Pro 大寫標籤作為開發者情境標記

### 避免 (Don't)
- 不要添加 box-shadows —— 它們在深色背景上不可見，且會破壞邊框定義的深度系統
- 不要使用粗體 (700) 字重 —— 系統僅使用 400 和 500
- 不要將綠色套用於背景或大面積表面 —— 它用於邊框、連結和小型強調
- 不要將暖色系（緋紅、橘色）作為主要設計元素 —— 它們作為語義代幣存在於特定狀態
- 英雄區行高不要增加超過 1.00 —— 密集度是有意的
- 按鈕不要使用大的圓角半徑 (16px+) —— 要麼藥丸狀 (9999px)，要麼標準 (6px)，不要兩者之間
- 主要表面背景不要亮過 `#171717` —— 黑暗度是結構性的
- 不要忘記半透明邊框 —— `rgba` 邊框顏色是分層機制
