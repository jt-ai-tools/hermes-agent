## 1. 視覺主題與氛圍

MongoDB 的網站呈現出一種「深林交會終端機」的體驗——這是一個根植於極深藍綠黑色 (`#001e2b`) 的設計系統，讓人聯想到資料庫的密度與森林樹冠的深度。在這個近乎黑色的畫布上，醒目的霓虹綠 (`#00ed64`) 如同品牌亮點般躍動——亮度足以讓人感到帶電，同時又充滿有機感，彷彿具有生命。這不是賽博龐克那種冷冽的霓虹燈，而是黑暗中生長的生物發光綠。

排版系統在架構上極具野心：用於巨大的 Hero 標題 (96px) 的 MongoDB Value Serif 營造出一種具權威感的編輯風格——在資料庫公司規模上選擇襯線體是一個大膽的選擇，這在傳達「我們不只是一家科技公司」。Euclid Circular A 負責處理大部分的內文和 UI 文字，具有異常廣泛的字重範圍 (300–700)；而 Source Code Pro 則作為程式碼和標籤字體，其獨特的大寫處理方式具有非常寬的字距 (1px–3px)。這套三字體系統建立了一個跨越「編輯優雅 → 幾何專業 → 工程精準」的層級結構。

讓 MongoDB 脫穎而出的是其「雙模式設計」：深色的 Hero/特色區塊世界（`#001e2b` 搭配霓虹綠點綴）以及淺色的內容世界（白色搭配藍綠灰色邊框 `#b8c4c2`）。這兩種模式之間的轉換創造了戲劇性的對比。陰影系統使用帶有藍綠色調的深色陰影 (`rgba(0, 30, 43, 0.12)`)，即使在淺色表面上也能維持森林暗色調的氛圍。按鈕採用藥丸形狀（100px–999px 圓角），帶有 MongoDB 綠色邊框 (`#00684a`)，整個組件系統參考了 LeafyGreen 設計系統。

**關鍵特性：**
- 深藍綠黑色背景 (`#001e2b`) —— 是森林般的深邃，而非太空般的漆黑
- 霓虹 MongoDB 綠 (`#00ed64`) 作為唯一的品牌點綴色 —— 同時具備電力感與有機感
- MongoDB Value Serif 用於 Hero 標題 —— 展現科技規模下的編輯權威
- Euclid Circular A 用於內文，字重 300 (Light) 是其獨特的內文字重
- Source Code Pro 具有寬大的大寫字距 (1px–3px)，用於技術標籤
- 帶有藍綠色調的陰影：`rgba(0, 30, 43, 0.12)` —— 陰影承載了森林的色調
- 雙模式：深藍綠 Hero 區塊 + 白色淺色內容區塊
- 藥丸型按鈕 (100px 圓角)，帶有綠色邊框 (`#00684a`)
- 連結藍色 (`#006cfa`) 且懸停時轉換至 `#3860be`

## 2. 調色盤與角色

### 品牌主要顏色
- **森林黑 (Forest Black)** (`#001e2b`)：主要深色背景 —— 最深的藍綠黑色
- **MongoDB 綠** (`#00ed64`)：主要品牌點綴色 —— 用於高亮、底線、漸層的霓虹綠
- **深綠 (Dark Green)** (`#00684a`)：按鈕邊框、淺色背景上的連結文字 —— 用於功能性的柔和綠

### 互動顏色
- **行動藍 (Action Blue)** (`#006cfa`)：次要點綴色 —— 連結、互動高亮
- **懸停藍 (Hover Blue)** (`#3860be`)：所有連結懸停狀態均轉換至此藍色
- **藍綠活動色 (Teal Active)** (`#1eaedb`)：按鈕懸停背景 —— 明亮的藍綠色

### 中性色階
- **深藍綠 (Deep Teal)** (`#1c2d38`)：深色按鈕背景、次要深色表面
- **藍綠灰 (Teal Gray)** (`#3d4f58`)：深色表面上的深色邊框
- **深石板色 (Dark Slate)** (`#21313c`)：深色連結文字變體
- **酷灰色 (Cool Gray)** (`#5c6c75`)：深色背景上的柔和文字、次要按鈕文字
- **銀藍綠 (Silver Teal)** (`#b8c4c2`)：淺色表面上的邊框、分隔線
- **淺色輸入框 (Light Input)** (`#e8edeb`)：深色表面上的輸入框文字
- **純白 (Pure White)** (`#ffffff`)：淺色區塊背景、深色背景上的按鈕文字
- **黑色 (Black)** (`#000000`)：淺色表面上的文字、最深元素

### 陰影
- **森林陰影 (Forest Shadow)** (`rgba(0, 30, 43, 0.12) 0px 26px 44px, rgba(0, 0, 0, 0.13) 0px 7px 13px`)：主要卡片提升感 —— 帶有藍綠色調
- **標準陰影 (Standard Shadow)** (`rgba(0, 0, 0, 0.15) 0px 3px 20px`)：一般提升感
- **細微陰影 (Subtle Shadow)** (`rgba(0, 0, 0, 0.1) 0px 2px 4px`)：輕微的卡片提升

- **程式碼 / 標籤**：`Source Code Pro` —— 等寬字體，用於大寫標籤處理
- **備用字體**：`Akzidenz-Grotesk Std` (CJK 搭配：Noto Sans KR/SC/JP), `Times`, `Arial`, `system-ui`

### 原則
- **襯線體展現權威**：在 Hero 規模使用 MongoDB Value Serif 營造出科技界少見的編輯風格——它傳達出 MongoDB 是一個成熟機構，而非初創公司。
- **字重 300 作為內文預設**：Euclid Circular A 在內文中使用 Light (300)，營造出輕盈的閱讀體驗，與密集、深色的背景形成對比。
- **寬字距等寬標籤**：大寫 Source Code Pro 搭配 1px–3px 的字距，建立出如同資料庫欄位標籤般的技術路標——具系統性、結構化且分類明確。
- **四種字重範圍**：300 (輕量內文) → 400 (標準) → 500 (UI/導覽) → 700 (粗體 CTA) —— 比大多數系統更廣，能實現精細的層級劃分。

## 4. 組件樣式

### 按鈕 (Buttons)

**主要綠色（深色表面）**
- 背景：`#00684a` (柔和的 MongoDB 綠)
- 文字：`#000000`
- 圓角：50% (圓形) 或 100px (藥丸形)
- 邊框：`1px solid #00684a`
- 陰影：`rgba(0,0,0,0.06) 0px 1px 6px`
- 懸停：縮放 1.1
- 按下：縮放 0.85

**深藍綠按鈕**
- 背景：`#1c2d38`
- 文字：`#5c6c75`
- 圓角：100px (藥丸形)
- 邊框：`1px solid #3d4f58`
- 懸停：背景 `#1eaedb`，文字白色，translateX(5px)

**描邊按鈕（淺色表面）**
- 背景：透明
- 文字：`#001e2b`
- 邊框：`1px solid #b8c4c2`
- 圓角：4px–8px
- 懸停：背景色調微調

### 卡片與容器
- 淺色模式：白色背景搭配 `1px solid #b8c4c2` 邊框
- 深色模式：`#001e2b` 或 `#1c2d38` 背景搭配 `1px solid #3d4f58`
- 圓角：16px (標準)，24px (中型)，48px (大型/Hero)
- 陰影：`rgba(0,30,43,0.12) 0px 26px 44px` (帶藍綠色調)
- 圖片容器：30px–32px 圓角

### 輸入框與表單
- 文字區域：文字 `#e8edeb`，內距 12px 12px 12px 8px
- 邊框：淺色面上為 `1px solid #b8c4c2`，深色面上為 `1px solid #3d4f58`
- 輸入框圓角：4px

### 導覽 (Navigation)
- 森林黑背景上的深色頁首
- Euclid Circular A 16px 字重 500 用於導覽連結
- MongoDB 標誌（葉子圖示 + 字標）靠左對齊
- 綠色 CTA 藥丸按鈕靠右對齊
- 帶有產品類別的巨型選單 (Mega-menu) 下拉列表

### 圖片處理
- 深色背景上的儀表板螢幕截圖
- 螢幕截圖中使用綠色點綴的 UI 元素
- 圖片容器採用 30px–32px 圓角
- 全寬深色區塊用於產品展示

### 獨特組件

**霓虹綠底線點綴**
- `0px 2px 2px 0px solid #00ed64` —— 底部 + 右側邊框營造出的底線點綴
- 用於特色標題和強調文字
- 亦有 `#006cfa` (藍色) 變體版本

**Source Code 標籤系統**
- 14px 大寫 Source Code Pro，搭配 1px–2px 字距
- 用於標題上方的區塊類別標記
- 營造出「資料庫欄位標籤」的視覺美感

## 5. 佈局原則

### 間距系統
- 基礎單位：8px
- 比例：1px, 4px, 7px, 8px, 10px, 12px, 14px, 15px, 16px, 18px, 20px, 24px, 32px

### 網格與容器
- 最大內容寬度置中
- 深色 Hero 區塊內含受限內容
- 下方為淺色內容區塊
- 卡片網格：2–3 欄
- 全寬深色頁尾

### 留白哲學
- **戲劇性的模式轉換**：從深藍綠區塊到白色內容的轉變，透過對比而非僅靠空間創造了內建的視覺呼吸感。
- **寬大的深色區塊**：深色 Hero 和特色區域使用額外的垂直內距 (80px+)，讓森林黑背景得以呼吸。
- **緊湊的淺色區塊**：白色內容區域較為密集，具有更緊湊的卡片網格和較小的垂直間距。

### 邊框圓角比例
- 極小 (1px–2px)：小範圍元素、徽章
- 細微 (4px)：輸入框、小型按鈕
- 標準 (8px)：卡片、連結
- 卡片 (16px)：標準卡片、容器
- 切換器 (20px)：切換元素
- 大型 (24px)：大型面板
- 圖片 (30px–32px)：圖片容器
- Hero (48px)：Hero 卡片
- 藥丸型 (100px–999px)：按鈕、導覽藥丸
- 全圓 (9999px)：最大藥丸形狀

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 平面 (Level 0) | 無陰影 | 預設表面 |
| 細微 (Level 1) | `rgba(0,0,0,0.1) 0px 2px 4px` | 輕微卡片提升 |
| 標準 (Level 2) | `rgba(0,0,0,0.15) 0px 3px 9px` | 標準卡片 |
| 顯著 (Level 3) | `rgba(0,0,0,0.15) 0px 3px 20px` | 提升的面板 |
| 森林 (Level 4) | `rgba(0,30,43,0.12) 0px 26px 44px, rgba(0,0,0,0.13) 0px 7px 13px` | Hero 卡片 —— 帶藍綠色調 |

**陰影哲學**：MongoDB 的陰影系統非常獨特，其主要提升陰影使用 `rgba(0, 30, 43, 0.12)` —— 一種帶有藍綠色調的陰影，將森林黑品牌色彩帶入深度系統。這意味著即使在白色表面上，陰影也會讓人感覺屬於 MongoDB 的色彩世界，而非通用的中性黑色。

## 7. 規範 (Do's and Don'ts)

### 建議 (Do)
- 使用 `#001e2b` (森林黑) 作為深色區塊 —— 而非純黑色
- 謹慎使用 MongoDB 綠 (`#00ed64`) 以獲得最大的電力感衝擊
- 僅在 Hero/顯示標題使用 MongoDB Value Serif —— 其他所有地方使用 Euclid Circular A
- 對技術標籤套用大寫 Source Code Pro 並搭配寬字距 (1px–3px)
- 使用帶有藍綠色調的陰影 (`rgba(0,30,43,0.12)`) 進行主要卡片提升
- 維持深/淺色區塊的二元性 —— 模式間的戲劇性對比
- 使用字重 300 處理內文 —— 輕盈的字重是易讀的聲音
- 對主要行動按鈕套用藥丸圓角 (100px)

### 避免 (Don't)
- 不要使用純黑色 (`#000000`) 作為深色背景 —— 務必使用藍綠黑 (`#001e2b`)
- 不要在背景上大面積使用 MongoDB 綠 (`#00ed64`) —— 它是用於文字、底線和小型高亮的點綴色
- 不要使用標準灰色陰影 —— 務必使用帶藍綠色調的陰影 (`rgba(0,30,43,...)`)
- 不要將襯線字體套用於內文 —— MongoDB Value Serif 僅限 Hero 使用
- 不要在 Source Code Pro 標籤上使用窄字距 —— 寬字距才是其身份象徵
- 不要在同一個區塊中混合深色和淺色區塊的處理方式
- 不要使用暖色調 —— 調色盤嚴格限制在冷色調（藍綠、綠、藍）
- 不要忘記綠色底線點綴 —— 它們是標誌性的裝飾元素

## 8. 響應式行為

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 小型手機 | <425px | 緊湊的單欄佈局 |
| 手機 | 425–768px | 標準手機佈局 |
| 平板 | 768–1024px | 開始出現雙欄網格 |
| 桌機 | 1024–1280px | 標準佈局 |
| 大型桌機 | 1280–1440px | 擴展佈局 |
| 超寬螢幕 | >1440px | 最大寬度，寬大的邊距 |

### 觸控目標
- 帶有充足內距的藥丸型按鈕
- 導覽連結維持 16px 並有適當間距
- 卡片表面作為全區域觸控目標

### 摺疊策略
- Hero：MongoDB Value Serif 96px → 64px → 進一步縮放
- 導覽：水平巨型選單 → 漢堡選單
- 特色卡片：多欄 → 堆疊
- 深/淺色區塊在所有尺寸下均維持其模式
- Source Code Pro 標籤維持大寫處理

### 圖片行為
- 儀表板螢幕截圖按比例縮放
- 深色區塊背景維持全寬
- 圖片圓角在所有斷點下均維持不變

1. 從模式決定開始：深色 (#001e2b) 用於 Hero/特色，白色用於內容

3. 襯線標題 (MongoDB Value Serif) 營造編輯權威感 —— 絕不用於內文
4. 字重 300 的內文創造輕盈的閱讀體驗 —— 不要預設使用 400
