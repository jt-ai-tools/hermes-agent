## 1. 視覺主題與氛圍

Stripe 的網站是金融科技設計的黃金標準 —— 該系統成功地同時展現了技術感與奢華感，精確且溫暖。頁面開啟於純白畫布（`#ffffff`），配以深海軍藍標題（`#061b31`）和標誌性的紫色（`#533afd`），後者既是品牌基石，也是互動強調色。這不是企業軟體中那種冷冰冰、臨床式的紫色；而是一種豐富、飽和的紫羅蘭色，讀起來顯得自信且高級。整體印象就像是由世界級字體鑄造廠重新設計的金融機構。

自定義的 `sohne-var` 變體字體是 Stripe 視覺識別的核心元素。每個文字元素都啟用了 OpenType 的 `"ss01"` 樣式組合，它修改了字符形狀，營造出獨特的幾何現代感。在展示尺寸（48px-56px）下，sohne-var 以 300 字重運行 —— 這對於標題來說是極輕的字重，創造出一種空靈、近乎耳語般的權威感。這與「粗體英雄標題」的慣例正好相反；Stripe 的標題感覺不需要大聲喧嘩。負字距（56px 時為 -1.4px，48px 時為 -0.96px）將文字緊縮成密集的、工程化的區塊。在較小尺寸下，系統同樣使用 300 字重，並配以按比例縮小的字距調整 (tracking)，以及透過 `"tnum"` 顯示用於財務數據的等寬數字。

真正讓 Stripe 脫穎而出的是它的陰影系統。Stripe 不像大多數網站那樣使用扁平或單層處理，而是使用多層、帶藍色調的陰影：標誌性的 `rgba(50,50,93,0.25)` 與 `rgba(0,0,0,0.1)` 結合，創造出具有冷靜、幾乎是氣氛深度感的陰影 —— 彷彿元素漂浮在暮色天空下。主要陰影顏色（50,50,93）的藍灰色底色直接與海軍藍-紫色的品牌色調相連，使得即使是層次的提升也充滿品牌感。

**核心特點：**
- 所有文字均使用帶有 OpenType `"ss01"` 的 sohne-var —— 定義品牌字形的自定義樣式組合
- 300 字重作為標誌性的標題字重 —— 輕盈、自信、反傳統
- 展示尺寸下的負字距（56px 時為 -1.4px，隨尺寸縮小逐漸放寬）
- 使用 `rgba(50,50,93,0.25)` 的多層藍色調陰影 —— 帶有品牌色彩感的層次提升
- 使用深海軍藍 (`#061b31`) 標題而非黑色 —— 溫暖、高級、金融級別
- 保守的圓角半徑 (4px-8px) —— 沒有藥丸狀，也沒有銳利感
- 寶石紅 (`#ea2261`) 和洋紅色 (`#f96bee`) 用於漸層和裝飾元素
- 使用 `SourceCodePro` 作為程式碼和技術標籤的等寬伴隨字體

## 2. 調色盤與角色

### 主要顏色
- **Stripe 紫** (`#533afd`): 主要品牌色、行動呼籲 (CTA) 背景、連結文字、互動強調。一種飽和的藍紫色，錨定了整個系統。
- **深海軍藍** (`#061b31`): `--hds-color-heading-solid`。主要標題顏色。不是黑色，也不是灰色 —— 是一種非常深的藍色，為文字增添了溫暖和深度。
- **純白色** (`#ffffff`): 頁面背景、卡片表面、深色背景上的按鈕文字。

### 原則
- **輕盈字重作為標誌**: 展示尺寸下的 300 字重是 Stripe 最獨特的排版選擇。當其他人使用 600-700 來吸引注意力時，Stripe 將輕盈視為奢華 —— 文字充滿自信，不需要重量來展現權威。
- **隨處可見的 ss01**: `"ss01"` 樣式組合是不可妥協的。它修改了特定的字形（可能是交替的 `a`、`g`、`l` 形狀），在所有 sohne-var 文字中營造出更幾何、更當代的感覺。
- **兩種 OpenType 模式**: 展示/內文使用 `"ss01"`，財務數據中的數字使用 `"tnum"`。兩者絕不重疊 —— 段落中的數字使用 ss01，數據表格中的數字使用 tnum。
- **漸進式字距調整 (Tracking)**: 字距隨尺寸比例緊縮：56px 為 -1.4px，48px 為 -0.96px，32px 為 -0.64px，26px 為 -0.26px，16px 及以下則為 normal。
- **雙重字重的簡潔**: 主要為 300（內文和標題）和 400（UI/按鈕）。主要字體中沒有粗體 (700) —— SourceCodePro 使用 500/700 來進行程式碼對比。

## 5. 版面配置原則

### 間距系統
- 基礎單位: 8px
- 比例: 1px, 2px, 4px, 6px, 8px, 10px, 11px, 12px, 14px, 16px, 18px, 20px
- 值得注意: 比例在小尺寸端非常密集（4-12 之間每 2px 一跳），反映了 Stripe 針對財務數據的精確導向 UI

### 網格與容器
- 最大內容寬度: 約 1080px
- 英雄區: 置中的單欄，配以寬大的內距和輕盈的標題
- 功能區段: 用於功能卡片的 2-3 欄網格
- 品牌沉浸式全寬深色區段，使用 `#1c1e54` 背景
- 程式碼/後台預覽作為包含在帶有藍色調陰影卡片中的內容

### 留白哲學
- **精確間距**: 不同於極簡系統中巨大的空虛感，Stripe 使用衡量過的、有目的的留白。每個間隙都是深思熟慮的排版選擇。
- **密集數據，寬大框架**: 財務數據顯示（表格、圖表）排列緊湊，但周圍的 UI 框架則間隔寬大。這創造出一種受控的密集感 —— 就像精美框架中組織良好的試算表。
- **區段節奏**: 白色區段與深色品牌區段 (`#1c1e54`) 交替，營造出戲劇性的亮/暗節奏，在不引入隨意色彩的情況下防止單調。

### 圓角比例
- 微型 (1px): 細粒度元素，微妙的圓角處理
- 標準 (4px): 按鈕、輸入框、徽章、卡片 —— 主力圓角
- 舒適 (5px): 標準卡片容器
- 寬鬆 (6px): 導覽、較大的互動元素
- 大型 (8px): 特色卡片、英雄區元素
- 複合: 底部圓角容器（分頁面板、下拉選單頁尾）使用 `0px 0px 6px 6px`

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 扁平 (Level 0) | 無陰影 | 頁面背景、內嵌文字 |
| 環境 (Level 1) | `rgba(23,23,23,0.06) 0px 3px 6px` | 微妙的卡片提升、懸停提示 |
| 標準 (Level 2) | `rgba(23,23,23,0.08) 0px 15px 35px` | 標準卡片、內容面板 |
| 提升 (Level 3) | `rgba(50,50,93,0.25) 0px 30px 45px -30px, rgba(0,0,0,0.1) 0px 18px 36px -18px` | 特色卡片、下拉選單、彈出層 |
| 深邃 (Level 4) | `rgba(3,3,39,0.25) 0px 14px 21px -14px, rgba(0,0,0,0.1) 0px 8px 17px -8px` | 彈窗、懸浮面板 |
| 光圈 (無障礙) | `2px solid #533afd` 輪廓 | 鍵盤焦點光圈 |

**陰影哲學**: Stripe 的陰影系統建立在色彩深度原則之上。大多數設計系統使用中性灰色或黑色陰影，而 Stripe 的主要陰影顏色 (`rgba(50,50,93,0.25)`) 是一種呼應品牌海軍藍色調的深藍灰色。這創造出的陰影不僅增加了深度，還增加了品牌氛圍。多層處理將此藍色調陰影與純黑次要層 (`rgba(0,0,0,0.1)`) 在不同偏移量下配對，創造出一種類似視差的深度感，其中帶有品牌色的陰影離元素較遠，而中性陰影離元素較近。負擴散值 (-30px, -18px) 確保陰影不會在水平方向超出元素的足跡，保持層次提升的垂直感和受控感。

### 裝飾性深度
- 深色品牌區段 (`#1c1e54`) 透過背景顏色對比創造沉浸式深度
- 英雄區裝飾使用寶石紅到洋紅色的漸層疊加
- 固定元素的頂部邊緣陰影使用顏色 `rgba(0,55,112,0.08)` (`--hds-color-shadow-sm-top`)

## 7. 規範（Do's and Don'ts）

### 建議 (Do)
- 在每個文字元素上使用帶有 `"ss01"` 的 sohne-var —— 樣式組合就是品牌的靈魂
- 對所有標題和內文使用 300 字重 —— 輕盈是標誌
- 對所有提升元素使用藍色調陰影 (`rgba(50,50,93,0.25)`)
- 標題使用 `#061b31` (深海軍藍) 而非 `#000000` —— 溫暖感很重要
- 圓角半徑保持在 4px-8px 之間 —— 有意的保守圓角處理
- 對任何表格/財務數字顯示使用 `"tnum"`
- 分層陰影：遠處藍色調 + 近處中性色，創造深度視差
- 使用 `#533afd` 紫色作為主要的互動/CTA 顏色

### 避免 (Don't)
- sohne-var 標題不要使用 600-700 字重 —— 300 字重才是品牌的聲音
- 卡片或按鈕不要使用大的圓角半徑 (12px+，藥丸形狀) —— Stripe 風格偏向保守
- 不要使用中性灰色陰影 —— 務必帶有藍色調 (`rgba(50,50,93,...)`)
- 任何 sohne-var 文字都不要跳過 `"ss01"` —— 交替字形定義了個性
- 標題不要使用純黑色 (`#000000`) —— 務必使用 `#061b31` 深海軍藍
- 互動元素不要使用暖色系強調色（橘色、黃色）—— 紫色是主要的
- 展示尺寸不要套用正向字距 —— Stripe 追求緊湊感
- 不要將洋紅/寶石紅強調色用於按鈕或連結 —— 它們僅用於裝飾/漸層

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 行動裝置 | <640px | 單欄顯示、縮小標題尺寸、堆疊卡片 |
| 平板電腦 | 640-1024px | 雙欄網格、適度的內距 |
| 桌上型電腦 | 1024-1280px | 完整版面、三欄功能網格 |
| 大型桌機 | >1280px | 置中內容，配以寬大的邊距 |

### 觸控目標
- 按鈕使用舒適的內距（垂直方向 8px-16px）
- 14px 的導覽連結配以足夠的間距
- 徽章至少有 6px 的水平內距以供點擊
- 行動裝置導覽切換按鈕具有 6px 圓角

### 收縮策略
- 英雄區: 56px 展示尺寸 -> 行動裝置上為 32px，保持 300 字重
- 導覽: 水平連結 + CTA -> 漢堡選單切換
- 功能卡片: 三欄 -> 雙欄 -> 單欄堆疊
- 品牌深色區段: 保持全寬處理，減少內部內距
- 財務數據表格: 行動裝置上水平滾動
- 區段間距: 64px+ -> 行動裝置上為 40px
- 排版比例壓縮: 不同斷點下英雄區尺寸為 56px -> 48px -> 32px

### 影像行為
- 後台/產品截圖在所有尺寸下均保持藍色調陰影
- 英雄區漸層裝飾在行動裝置上簡化處理
- 程式碼區塊保持 `SourceCodePro` 處理，可能需要水平滾動
- 卡片影像保持一致的 4px-6px 圓角半徑
