## 1. 視覺主題與氛圍

Linear 的網站是「深色模式優先」產品設計的典範 —— 一個近乎全黑的畫布（`#08090a`），內容如同星光般從黑暗中浮現。整體印象是極致的精密工程：每個元素都存在於經過精心校準的亮度層級中，從幾乎看不見的邊框（`rgba(255,255,255,0.05)`）到柔和發亮的文字（`#f7f8f8`）。這不是將深色主題應用於淺色設計，而是將黑暗作為原生介質，透過白色不透明度的細微變化而非顏色變化來管理資訊密度。

字體系統完全建立在啟用全域 OpenType 功能 `"cv01"` 和 `"ss03"` 的 Inter Variable 之上，賦予字體更簡潔、更具幾何感的特徵。Inter 被運用於非常廣泛的字重範圍 —— 從 300（細體內文）到 510（中黑體，Linear 的標誌性字重）再到 590（半粗體強調）。510 字重特別出眾：它介於標準（Regular）和中黑（Medium）之間，營造出一種不張揚的微妙強調。在展示尺寸（72px, 64px, 48px）下，Inter 使用激進的負字距（-1.584px 到 -1.056px），創造出壓縮且具權威感的標題，給人一種經過「工程設計」而非僅僅是「視覺設計」的感覺。Berkeley Mono 作為程式碼和技術標籤的等寬伴侶，並備有 ui-monospace、SF Mono 和 Menlo 作為後備。

顏色系統幾乎完全是無彩色的 —— 深色背景配上白色/灰色文字 —— 僅由單一品牌強調色點綴：Linear 標誌性的靛紫色（背景用 `#5e6ad2`，互動強調用 `#7170ff`）。這種強調色被謹慎且有目的地使用，僅出現在行動呼籲（CTA）、活動狀態和品牌元素上。邊框系統使用極薄、半透明的白色邊框（`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`），創造出結構感而無視覺噪音，宛如在月光下繪製的線框。

**核心特徵：**
- 原生深色模式：`#08090a` 行銷背景，`#0f1011` 面板背景，`#191a1b` 提升表面（Elevated surfaces）
- 全域使用啟用 `"cv01", "ss03"` 的 Inter Variable —— 幾何替代字符帶來更簡潔的美感
- 大部分 UI 文字使用標誌性字重 510（介於 Regular 和 Medium 之間）
- 展示尺寸下使用激進的負字距（72px 為 -1.584px，48px 為 -1.056px）
- 品牌靛紫色：`#5e6ad2` (背景) / `#7170ff` (強調) / `#828fff` (懸停) —— 系統中唯一的有彩色
- 貫穿始終的半透明白色邊框：`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`
- 按鈕背景接近零不透明度：`rgba(255,255,255,0.02)` 到 `rgba(255,255,255,0.05)`
- 多層陰影配合內陰影（Inset）變體，在深色表面營造深度感
- 以 Radix UI primitives 作為元件基礎（偵測到 6 個 primitives）
- 成功綠（`#27a644`, `#10b981`）僅用於狀態指示器

## 2. 色板與角色

### 背景表面 (Background Surfaces)
- **行銷黑 (Marketing Black)** (`#010102` / `#08090a`)：最深層的背景 —— 主視覺區塊和行銷頁面的畫布。近乎純黑，帶有察覺不到的冷藍色調。
- **面板深色 (Panel Dark)** (`#0f1011`)：側邊欄和面板背景。比行銷黑淺一階。
- **三級表面 (Level 3 Surface)** (`#191a1b`)：提升的表面區域、卡片背景、下拉選單。
- **次要表面 (Secondary Surface)** (`#28282c`)：最淺的深色表面 —— 用於懸停狀態和稍微提升的元件。

### 文字與內容 (Text & Content)
- **主要文字 (Primary Text)** (`#f7f8f8`)：近白色，帶有極細微的暖色調。預設文字顏色 —— 非純白色，防止在深色背景上造成視覺疲勞。
- **次要文字 (Secondary Text)** (`#d0d6e0`)：冷銀灰色，用於內文、描述和次要內容。
- **三級文字 (Tertiary Text)** (`#8a8f98`)：暗淡灰色，用於佔位符、元數據和去強調內容。
- **四級文字 (Quaternary Text)** (`#62666d`)：最淡的文字 —— 時間戳記、禁用狀態、細微標籤。

### 品牌與強調 (Brand & Accent)
- **品牌靛紫 (Brand Indigo)** (`#5e6ad2`)：主要品牌顏色 —— 用於 CTA 按鈕背景、品牌標誌和關鍵互動表面。
- **強調紫 (Accent Violet)** (`#7170ff`)：互動元素的亮色變體 —— 連結、活動狀態、選取項目。
- **強調懸停 (Accent Hover)** (`#828fff`)：強調元素懸停時更亮、更飽和的變體。
- **安全薰衣草紫 (Security Lavender)** (`#7a7fad`)：暗淡靛紫色，專用於安全相關的 UI 元素。

### 狀態顏色 (Status Colors)
- **綠色** (`#27a644`)：主要成功/活動狀態。用於「進行中」指示器。
- **翡翠綠** (`#10b981`)：次要成功 —— 藥丸狀徽章、完成狀態。

### 邊框與分隔線 (Border & Divider)
- **主要邊框 (Border Primary)** (`#23252a`)：實心深色邊框，用於顯著的分隔。
- **次要邊框 (Border Secondary)** (`#34343a`)：稍淺的實心邊框。
- **三級邊框 (Border Tertiary)** (`#3e3e44`)：最淺的實心邊框變體。
- **細微邊框 (Border Subtle)** (`rgba(255,255,255,0.05)`)：極細微的半透明邊框 —— 預設選項。
- **標準邊框 (Border Standard)** (`rgba(255,255,255,0.08)`)：標準半透明邊框，用於卡片、輸入框、程式碼區塊。
- **極細分線 (Line Tint)** (`#141516`)：幾乎不可見的線條，用於最細微的分隔。
- **三級分線 (Line Tertiary)** (`#18191a`)：稍微明顯的分隔線。

### 淺色模式中性色 (適用於淺色主題語境)
- **淺色背景** (`#f7f8f8`)：淺色模式下的頁面背景。
- **淺色表面** (`#f3f4f5` / `#f5f6f7`)：細微的表面色調。
- **淺色邊框** (`#d0d6e0`)：淺色語境下的可見邊框。
- **替代淺色邊框** (`#e6e6e6`)：另一種較淺的邊框。
- **純白** (`#ffffff`)：卡片表面、高亮。

### 覆蓋層 (Overlay)
- **主要覆蓋層 (Overlay Primary)** (`rgba(0,0,0,0.85)`)：模態框/對話框背景 —— 極深色，以隔離焦點。

### 原則
- **510 是標誌性字重**：Linear 將 Inter Variable 的 510 字重（介於常規 400 和中黑 500 之間）作為預設強調字重。這營造出一種微妙的粗體感，而沒有傳統中黑或半粗體的笨重感。
- **規模化壓縮**：展示尺寸隨尺寸增加而使用更緊湊的字距 —— 72px 為 -1.584px，64px 為 -1.408px，48px 為 -1.056px，32px 為 -0.704px。在 24px 以下，間距趨向標準。
- **OpenType 作為身份識別**：`"cv01", "ss03"` 並非裝飾性的 —— 它們將 Inter 轉化為 Linear 獨特的字體，賦予其更具幾何感、更有目的性的特徵。
- **三級字重系統**：400 (閱讀), 510 (強調/UI), 590 (強烈強調)。300 字重僅出現在刻意去強調的語境中。

## 5. 版面原則 (Layout Principles)

### 間距系統 (Spacing System)
- 基礎單位：8px
- 尺度：1px, 4px, 7px, 8px, 11px, 12px, 16px, 19px, 20px, 22px, 24px, 28px, 32px, 35px
- 7px 和 11px 的值表明了為了視覺對齊而進行的微調
- 主要節奏：8px, 16px, 24px, 32px (標準 8px 網格)

### 網格與容器 (Grid & Container)
- 最大內容寬度：約 1200px
- 主視覺 (Hero)：置中單欄，垂直內距寬裕
- 特色區塊 (Feature sections)：用於特色卡片的 2-3 欄網格
- 全寬深色區塊，內部設有最大寬度限制
- 更新日誌 (Changelog)：單欄時間軸版面

### 留白哲學 (Whitespace Philosophy)
- **黑暗即空間**：在 Linear 的深色畫布上，空白空間不是白色的，而是「不在場」。近黑色的背景本身就是留白，內容從中浮現。
- **壓縮標題，擴展周邊**：72px 的展示文字配合 -1.584px 的字距非常密集且壓縮，但置於廣闊的深色內距中。排版密度與空間寬裕之間的對比創造了張力。
- **區塊隔離**：每個特色區塊由寬裕的垂直內距（80px+）分隔，沒有可見的分隔線 —— 深色背景提供了自然的隔離。

### 圓角尺度 (Border Radius Scale)
- 極小 (2px)：行內徽章、工具列按鈕、細微標籤
- 標準 (4px)：小型容器、列表項目
- 適中 (6px)：按鈕、輸入框、功能性元素
- 卡片 (8px)：卡片、下拉選單、彈出視窗
- 面板 (12px)：面板、精選卡片、區塊容器
- 大型 (22px)：大型面板元素
- 全藥丸 (9999px)：過濾標籤、狀態標籤
- 圓形 (50%)：圖示按鈕、頭像、狀態點

## 6. 深度與提升感 (Depth & Elevation)

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 平面 (Level 0) | 無陰影，`#010102` 背景 | 頁面背景，最底層畫布 |
| 細微 (Level 1) | `rgba(0,0,0,0.03) 0px 1.2px 0px` | 工具列按鈕，微小提升 |
| 表面 (Level 2) | `#25252a` 背景 + `1px solid rgba(255,255,255,0.08)` 邊框 | 卡片、輸入欄位、容器 |
| 內陰影 (Level 2b) | `rgba(0,0,0,0.2) 0px 0px 12px 0px inset` | 凹陷面板、內部陰影 |
| 環繞 (Level 3) | `rgba(0,0,0,0.2) 0px 0px 0px 1px` | 以邊框作為陰影的技術 |
| 提升 (Level 4) | `rgba(0,0,0,0.4) 0px 2px 4px` | 懸浮元素、下拉選單 |
| 對話框 (Level 5) | 多層堆疊：`rgba(0,0,0,0) 0px 8px 2px, rgba(0,0,0,0.01) 0px 5px 2px, rgba(0,0,0,0.04) 0px 3px 2px, rgba(0,0,0,0.07) 0px 1px 1px, rgba(0,0,0,0.08) 0px 0px 1px` | 彈出視窗、命令面板、模態框 |
| 焦點 | `rgba(0,0,0,0.1) 0px 4px 12px` + 額外層級 | 互動元素上的鍵盤焦點 |

**陰影哲學**：在深色表面上，傳統陰影（深色背景上的深色陰影）幾乎不可見。Linear 透過使用半透明白色邊框作為主要的深度指標來解決這個問題。提升感不是透過陰影的深淺來傳達，而是透過背景亮度的階梯 —— 每一層級稍微增加表面背景的白色不透明度（`0.02` → `0.04` → `0.05`），營造出一種細微的堆疊效果。內陰影技術 (`rgba(0,0,0,0.2) 0px 0px 12px 0px inset`) 為凹陷面板創造了獨特的「沉降」效果，增加了傳統深色主題所缺乏的三維深度。

## 7. 規範與禁忌 (Do's and Don'ts)

### 規範 (Do)
- 在所有文字上使用啟用了 `"cv01", "ss03"` 的 Inter Variable —— 這些功能是 Linear 字體身份的根本
- 使用 510 字重作為預設強調字重 —— 這是 Linear 標誌性的中介字重
- 在展示尺寸下應用激進的負字距（72px 為 -1.584px，48px 為 -1.056px）
- 建立在近黑色背景上：行銷用 `#08090a`，面板用 `#0f1011`，提升表面用 `#191a1b`
- 使用半透明白色邊框 (`rgba(255,255,255,0.05)` 到 `rgba(255,255,255,0.08)`) 而非實心深色邊框
- 保持按鈕背景近乎透明：`rgba(255,255,255,0.02)` 到 `rgba(255,255,255,0.05)`
- 將品牌靛紫 (`#5e6ad2` / `#7170ff`) 僅保留給主要 CTA 和互動強調色
- 使用 `#f7f8f8` 作為主要文字 —— 而非純白 `#ffffff`，後者會過於刺眼
- 應用亮度堆疊模型：越深層 = 背景越暗，提升層級 = 背景稍微變亮

### 禁忌 (Don't)
- 不要使用純白 (`#ffffff`) 作為主要文字 —— `#f7f8f8` 可防止視覺疲勞
- 不要為按鈕使用實色背景 —— 系統核心是透明度（rgba 白色，不透明度 0.02–0.05）
- 不要裝飾性地使用品牌靛紫 —— 它僅限於互動/CTA 元素
- 不要在展示文字上使用正字距 —— 大尺寸的 Inter 始終應該使用負字距
- 不要在深色背景上使用可見/不透明的邊框 —— 邊框應是極薄的半透明白色
- 不要忽略 OpenType 功能 (`"cv01", "ss03"`) —— 沒有它們，它只是普通的 Inter，而非 Linear 的 Inter
- 不要使用 700 字重（粗體）—— Linear 的最大字重是 590，並以 510 作為核心字重
- 不要在 UI 介面中引入暖色調 —— 色板僅限於冷灰色與藍紫色強調色
- 不要在深色表面上使用落影（Drop shadows）來營造提升感 —— 應使用背景亮度階梯法

## 8. 響應式行為 (Responsive Behavior)

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 小手機 | <600px | 單欄，緊湊內距 |
| 手機 | 600–640px | 標準手機版面 |
| 平板 | 640–768px | 開始出現雙欄網格 |
| 小桌機 | 768–1024px | 完整卡片網格，擴展內距 |
| 桌機 | 1024–1280px | 標準桌機，完整導覽 |
| 大桌機 | >1280px | 完整版面，寬裕邊距 |

### 觸控目標
- 按鈕使用適中的內距，圓角半徑至少為 6px
- 導覽連結為 13–14px，間距適當
- 藥丸標籤具有 10px 的水平內距，以確保觸控易用性
- 圖示按鈕使用 50% 圓角，確保圓形且易於點擊的目標
- 搜尋觸發器位置顯眼，且具有寬裕的點擊區域

### 摺疊策略
- 主視覺 (Hero)：72px → 48px → 32px 展示文字，字距按比例調整
- 導覽：水平連結 + CTA 在 768px 變為漢堡選單
- 特色卡片：3 欄 → 2 欄 → 單欄堆疊
- 產品螢幕截圖：維持寬高比，可能減少內距
- 更新日誌 (Changelog)：在所有尺寸下維持單欄時間軸
- 頁尾：多欄 → 堆疊單欄
- 區塊間距：80px+ → 手機上為 48px

### 圖片行為
- 儀表板螢幕截圖在所有尺寸下均維持邊框處理
- 主視覺在行動端簡化（減少浮動的 UI 元素）
- 產品螢幕截圖使用響應式尺寸，並保持一致的圓角
- 深色背景確保螢幕截圖在任何視窗中都能自然融合

### 迭代指南
1. 務必在所有 Inter 文字上設置 font-feature-settings `"cv01", "ss03"` —— 這是實現 Linear 視覺風格的關鍵
2. 字距隨字體大小縮放：72px 為 -1.584px，48px 為 -1.056px，32px 為 -0.704px，16px 以下為標準
3. 三種字重：400 (閱讀), 510 (強調/導覽), 590 (宣佈)
4. 透過背景不透明度達成表面提升感：`rgba(255,255,255, 0.02 → 0.04 → 0.05)` —— 深色模式下絕不使用實色背景
5. 品牌靛紫 (`#5e6ad2` / `#7170ff`) 是唯一的有彩色 —— 其他一切皆為灰階
6. 邊框始終是半透明白色，深色背景上絕不使用實心深色邊框
7. 任何程式碼或技術內容使用 Berkeley Mono，其他一切使用 Inter Variable
