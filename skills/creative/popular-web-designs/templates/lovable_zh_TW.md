## 1. 視覺主題與氛圍

Lovable 的網站透過節制展現出暖意。整個頁面坐落在奶油般的羊皮紙色調背景（`#f7f4ed`）上，這立即讓它與大多數開發者工具網站冷冽的白色慣例區分開來。這並非為了極簡而極簡 —— 而是為了營造親切感、甚至帶點類比（Analog）感的刻意選擇，就像一本製作精良的筆記本。近乎黑色的文字（`#1c1c1c`）與溫暖的奶油色形成對比，既護眼又保持了銳利的閱讀性。

客製化的 Camera Plain Variable 字型是該系統的秘密武器。不同於象徵「科技公司」的幾何無襯線字體，Camera Plain 擁有人文主義的暖意 —— 略微圓潤的端點、有機的曲線和舒適的閱讀節奏。在展示尺寸（48px–60px）下，字重 600 搭配激進的負字距（-0.9px 到 -1.5px），將標題壓縮成自信、具編輯感的陳述。該字型以 `ui-sans-serif, system-ui` 作為後備，承認客製化字型承載著品牌的個性。

Lovable 視覺系統的獨特之處在於其不透明度驅動（Opacity-driven）的深度模型。該系統並非使用傳統的灰階，而是透過調節不同不透明度（0.03, 0.04, 0.4, 0.82–0.83）的 `#1c1c1c` 來創造統一的色調範圍。頁面上的每一種灰色在技術上都是相同的色相 —— 只是透明度不同。這創造了一種視覺一致性，是使用任意十六進位值幾乎無法達到的。邊框系統也遵循此原則：`1px solid #eceae4` 用於細微的分隔，`1px solid rgba(28, 28, 28, 0.4)` 用於更強的互動邊界。

**核心特徵：**
- 溫暖的羊皮紙背景 (`#f7f4ed`) —— 不是純白，也不是米色，而是一種精心挑選、給人溫潤感的奶油色。
- Camera Plain Variable 字型，具備人文主義暖意，並在展示尺寸使用編輯風格的字距。
- 不透明度驅動的顏色系統：所有的灰色皆源自不同透明度層級的 `#1c1c1c`。
- 按鈕上的內陰影 (Inset shadow) 技術：`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset`。
- 溫潤的中性邊框色板：細微處使用 `#eceae4`，互動元素使用 `rgba(28,28,28,0.4)`。
- 全藥丸型圓角 (`9999px`) 廣泛用於操作按鈕和圖示容器。
- 焦點狀態 (Focus state) 使用 `rgba(0,0,0,0.1) 0px 4px 12px` 陰影，營造柔和溫暖的強調。
- 使用 shadcn/ui + Radix UI 元件原語，搭配 Tailwind CSS 實用類別樣式。

## 2. 色板與角色

### 主要顏色
- **奶油色 (Cream)** (`#f7f4ed`)：頁面背景、卡片表面、按鈕表面。基礎色 —— 溫暖、類紙質、具人性化。
- **木炭黑 (Charcoal)** (`#1c1c1c`)：主要文字、標題、深色按鈕背景。非純黑 —— 帶有有機的暖意。
- **米白 (Off-White)** (`#fcfbf8`)：深色背景上的按鈕文字、細微高亮。與純白幾乎難以區分。

### 中性色階 (基於不透明度)
- **木炭黑 100%** (`#1c1c1c`)：主要文字、標題、深色表面。
- **木炭黑 83%** (`rgba(28,28,28,0.83)`)：強烈的次要文字。
- **木炭黑 82%** (`rgba(28,28,28,0.82)`)：內文。
- **暗淡灰 (Muted Gray)** (`#5f5f5d`)：次要文字、描述、圖註。
- **木炭黑 40%** (`rgba(28,28,28,0.4)`)：互動邊框、按鈕輪廓。
- **木炭黑 4%** (`rgba(28,28,28,0.04)`)：細微的懸停背景、微小色調變化。
- **木炭黑 3%** (`rgba(28,28,28,0.03)`)：幾乎不可見的覆蓋層、背景深度。

### 表面與邊框
- **淺奶油色 (Light Cream)** (`#eceae4`)：卡片邊框、分隔線、圖片輪廓。溫暖的分隔線。
- **奶油色表面 (Cream Surface)** (`#f7f4ed`)：卡片背景、區塊填充 —— 與頁面背景相同，以實現無縫整合。

### 互動性
- **藍色環 (Ring Blue)** (`#3b82f6` 50% 不透明度)：`--tw-ring-color`，Tailwind 焦點環。
- **焦點陰影 (Focus Shadow)** (`rgba(0,0,0,0.1) 0px 4px 12px`)：焦點和活動狀態陰影 —— 柔和、溫暖、擴散。

### 內陰影 (Inset Shadows)
- **按鈕內陰影** (`rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px 0px`)：深色按鈕上標誌性的多層內陰影。

## 3. 字體排版規則

### 字型系列 (Font Family)
- **主要字型**：`Camera Plain Variable`，後備：`ui-sans-serif, system-ui`
- **字重範圍**：400 (內文/閱讀), 480 (特殊展示), 600 (標題/強調)
- **特色**：具有連續字重軸的可變字型 —— 允許精確調整中間字重（如 480）。

### 原則
- **溫暖的人文主義聲音**：Camera Plain Variable 賦予 Lovable 親切的個性。略微圓潤的端點和有機的曲線，與大多數開發者工具使用的銳利幾何無襯線字體形成鮮明對比。
- **可變字重作為設計工具**：該字型支持連續的字重值（例如 480），能實現超越標準字重停靠點的細微層級。60px 下的 480 字重創造出一種比半粗體輕盈、但比常規體強烈的展示風格。
- **規模化壓縮**：大標題使用負字距（-0.9px 到 -1.5px）以產生編輯風格的衝擊力。內文則保持標準字距以確保舒適閱讀。
- **兩種字重，角色明確**：400 (內文/UI/連結/按鈕) 和 600 (標題/強調)。狹窄的字重範圍透過尺寸和間距而非字重變化來建立層級。

## 4. 元件樣式 (Component Stylings)

### 按鈕 (Buttons)

**主要深色 (帶內陰影)**
- 背景：`#1c1c1c`
- 文字：`#fcfbf8`
- 內距 (Padding)：8px 16px
- 圓角 (Radius)：6px
- 陰影：`rgba(0,0,0,0) 0px 0px 0px 0px, rgba(0,0,0,0) 0px 0px 0px 0px, rgba(255,255,255,0.2) 0px 0.5px 0px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px 0px`
- 啟動狀態 (Active)：不透明度 0.8
- 焦點 (Focus)：`rgba(0,0,0,0.1) 0px 4px 12px` 陰影
- 用途：主要 CTA（如「開始建立」、「立即開始」）

**幽靈按鈕 / 輪廓按鈕 (Ghost / Outline)**
- 背景：透明
- 文字：`#1c1c1c`
- 內距：8px 16px
- 圓角：6px
- 邊框：`1px solid rgba(28,28,28,0.4)`
- 啟動狀態：不透明度 0.8
- 焦點：`rgba(0,0,0,0.1) 0px 4px 12px` 陰影
- 用途：次要操作（如「登入」、「文件」）

**奶油色表面按鈕**
- 背景：`#f7f4ed`
- 文字：`#1c1c1c`
- 內距：8px 16px
- 圓角：6px
- 無邊框
- 啟動狀態：不透明度 0.8
- 用途：三級操作、工具列按鈕

**藥丸型 / 圖示按鈕**
- 背景：`#f7f4ed`
- 文字：`#1c1c1c`
- 圓角：9999px (全藥丸)
- 陰影：與主要深色相同的內陰影模式
- 不透明度：預設 0.5，啟動狀態 0.8
- 用途：額外操作、計畫模式切換、語音錄製

### 卡片與容器 (Cards & Containers)
- 背景：`#f7f4ed` (與頁面相同)
- 邊框：`1px solid #eceae4`
- 圓角：12px (標準)、16px (精選)、8px (緊湊)
- 預設無方框陰影 —— 透過邊框來定義邊界。
- 圖片卡片：`1px solid #eceae4`，帶 12px 圓角。

### 輸入框與表單 (Inputs & Forms)
- 背景：`#f7f4ed`
- 文字：`#1c1c1c`
- 邊框：`1px solid #eceae4`
- 圓角：6px
- 焦點：藍色環 (`rgba(59,130,246,0.5)`) 輪廓
- 佔位符 (Placeholder)：`#5f5f5d`

### 導覽 (Navigation)
- 在奶油色背景上的簡潔水平導覽列，固定位置。
- 標誌/字標左對齊 (128.75 x 22px)。
- 連結：Camera Plain 14–16px 字重 400，文字為 `#1c1c1c`。
- CTA：帶內陰影的深色按鈕，6px 圓角。
- 行動端：漢堡選單搭配 6px 圓角按鈕。
- 滾動時帶有細微邊框或無邊框。

### 連結 (Links)
- 顏色：`#1c1c1c`
- 裝飾：底線 (預設)
- 懸停 (Hover)：主要強調色 (透過 CSS 變數 `hsl(var(--primary))` 指定)
- 懸停時顏色不變 —— 裝飾線條承載互動訊號。

### 圖片處理
- 展示/作品集圖片帶有 `1px solid #eceae4` 邊框。
- 所有圖片容器保持一致的 12px 圓角。
- 主視覺內容背後有柔和的漸層背景（暖色系多色渲染）。
- 範本/專案展示採用畫廊風格呈現。

### 獨特元件

**AI 對話輸入框**
- 大型提示詞 (Prompt) 輸入區域，帶有柔和邊界。
- 建議標籤 (Pills) 帶有 `#eceae4` 邊框。
- 語音錄製 / 計畫模式切換按鈕呈藥丸狀 (9999px)。
- 溫暖且具邀請感的輸入區域 —— 而非冷冰冰的診所感。

**範本庫 (Template Gallery)**
- 展示專案範本的卡片網格。
- 每個卡片：圖片 + 標題，`1px solid #eceae4` 邊框，12px 圓角。
- 懸停：細微的陰影或邊框加深。
- 分類標籤為文字連結。

**數據欄 (Stats Bar)**
- 大型數據：採用字重 600、尺寸 48px+ 的「0M+」模式。
- 下方為暗淡灰色的描述性文字。
- 水平佈局，間距寬裕。

## 5. 版面原則 (Layout Principles)

### 間距系統 (Spacing System)
- 基礎單位：8px
- 尺度：8px, 10px, 12px, 16px, 24px, 32px, 40px, 56px, 80px, 96px, 128px, 176px, 192px, 208px
- 尺度在高端顯著擴張 —— 各區塊使用 80px–208px 的垂直間距，以提供編輯風格的呼吸空間。

### 網格與容器 (Grid & Container)
- 最大內容寬度：約 1200px (置中)。
- 主視覺 (Hero)：置中單欄，帶有巨大的垂直內距 (96px+)。
- 特色區塊 (Feature sections)：2–3 欄網格。
- 全寬頁尾，採用多欄連結佈局。
- 帶有置中卡片網格的展示區塊。

### 留白哲學 (Whitespace Philosophy)
- **編輯感的慷慨**：Lovable 的區塊邊界間距非常豪放 (80px–208px)。溫暖的奶油色背景讓這些開闊感顯得舒適而非空洞。
- **內容驅動的節奏**：卡片內部的緊湊間距 (12–24px) 與寬大的區塊間隙形成對比，創造出一種在聚焦內容與視覺休息之間交替的閱讀節奏。
- **區塊分隔**：頁尾使用 `1px solid #eceae4` 邊框和 16px 圓角容器。區塊定義依賴寬裕的間距而非邊框線條。

### 圓角尺度 (Border Radius Scale)
- 極小 (4px)：小按鈕、互動元素。
- 標準 (6px)：按鈕、輸入框、導覽選單。
- 適中 (8px)：緊湊型卡片、Div 區塊。
- 卡片 (12px)：標準卡片、圖片容器、範本。
- 容器 (16px)：大型容器、頁尾區塊。
- 全藥丸 (9999px)：操作標籤、圖示按鈕、切換開關。

## 6. 深度與提升感 (Depth & Elevation)

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 平面 (Level 0) | 無陰影，奶油色背景 | 頁面表面，大多數內容 |
| 帶邊框 (Level 1) | `1px solid #eceae4` | 卡片、圖片、分隔線 |
| 內陰影 (Level 2) | `rgba(255,255,255,0.2) 0px 0.5px 0px inset, rgba(0,0,0,0.2) 0px 0px 0px 0.5px inset, rgba(0,0,0,0.05) 0px 1px 2px` | 深色按鈕、主要操作 |
| 焦點 (Level 3) | `rgba(0,0,0,0.1) 0px 4px 12px` | 活動/焦點狀態 |
| 環繞 (無障礙) | `rgba(59,130,246,0.5)` 2px 環 | 輸入框的鍵盤焦點 |

**陰影哲學**：Lovable 的深度系統刻意設計得很淺。系統不使用帶有戲劇性落影的懸浮卡片，而是依賴與奶油色表面形成對比的溫暖邊框 (`#eceae4`) 來創造溫和的封閉感。唯一值得注意的陰影模式是深色按鈕上的內陰影 —— 一種細微的多層技術，在頂部邊緣設置白色高亮線，底部則是深色環和柔和的落影。這營造出一種「按入表面」的觸感，而非「懸浮於表面」的感覺。溫暖的焦點陰影 (`rgba(0,0,0,0.1) 0px 4px 12px`) 被刻意擴散且放大，創造出柔和的光暈而非銳利的輪廓。

### 裝飾性深度
- 主視覺 (Hero)：主視覺背後有柔和、溫暖的多色漸層渲染（粉色、橙色、藍色）—— 具氛圍感，幾乎察覺不到。
- 頁尾：漸層背景，底部過渡至暖色調。
- 無生硬的區塊分隔線 —— 間距與背景暖意處理了轉場。

## 7. 規範與禁忌 (Do's and Don'ts)

### 規範 (Do)
- 使用溫暖的奶油色背景 (`#f7f4ed`) 作為頁面基礎 —— 這是品牌的標誌性色調。
- 在展示尺寸使用 Camera Plain Variable 字型並搭配負字距 (-0.9px 到 -1.5px)。
- 從不同不透明度層級的 `#1c1c1c` 衍生出所有灰色，以維持色調統一。
- 在深色按鈕上使用內陰影技術，以增加觸感深度。
- 使用 `#eceae4` 邊框而非陰影來封裝卡片。
- 保持狹窄的字重系統：400 用於內文/UI，600 用於標題。
- 全藥丸型圓角 (9999px) 僅用於操作標籤和圖示按鈕。
- 在活動狀態應用 0.8 的不透明度，以提供響應式的觸覺回饋。

### 禁忌 (Don't)
- 不要使用純白 (`#ffffff`) 作為頁面背景 —— 奶油色是刻意選擇。
- 不要為卡片使用沉重的方框陰影 —— 邊框才是封裝機制。
- 不要引入飽和的強調色 —— 色板刻意保持溫暖中性。
- 不要使用字重 700 (粗體) —— 600 是系統中的最大字重。
- 不要在矩形按鈕上應用 9999px 圓角 —— 藥丸型僅用於圖示/操作切換。
- 不要使用銳利的焦點輪廓 —— 系統使用柔和的基於陰影的焦點指示器。
- 不要混合邊框樣式 —— 被動元素使用 `#eceae4`，互動元素使用 `rgba(28,28,28,0.4)`。
- 不要增加標題的字距 —— Camera Plain 被設計為在大尺寸下保持緊湊。

## 8. 響應式行為 (Responsive Behavior)

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 小手機 | <600px | 緊湊單欄，縮減內距 |
| 手機 | 600–640px | 標準手機佈局 |
| 小平板 | 640–700px | 開始出現雙欄網格 |
| 平板 | 700–768px | 卡片網格擴展 |
| 小桌機 | 768–1024px | 多欄佈局 |
| 桌機 | 1024–1280px | 完整特色佈局 |
| 大桌機 | 1280–1536px | 最大內容寬度，寬裕邊距 |

### 觸控目標
- 按鈕：8px 16px 內距（舒適的觸控體驗）。
- 導覽：項目之間保持足夠間距。
- 藥丸按鈕：9999px 圓角創造了大型、易於點擊的目標。
- 選單切換：適當尺寸的 6px 圓角按鈕。

### 摺疊策略
- 主視覺 (Hero)：標題從 60px → 48px → 36px 縮放，字距按比例調整。
- 導覽：水平連結在 768px 變為漢堡選單。
- 特色卡片：3 欄 → 2 欄 → 單欄堆疊。
- 範本庫：網格 → 垂直堆疊卡片。
- 數據欄：水平 → 垂直堆疊。
- 頁尾：多欄 → 堆疊單欄。
- 區塊間距：128px+ → 行動端為 64px。

### 圖片行為
- 範本螢幕截圖在所有尺寸下均維持 `1px solid #eceae4` 邊框。
- 跨斷點保留 12px 圓角。
- 畫廊圖片具有響應式尺寸與一致的寬高比。
- 行動端的主視覺漸層變得更柔和/簡化。

## 9. Agent 提示指南 (Agent Prompt Guide)

### 顏色快速參考
- 主要 CTA：木炭黑 (`#1c1c1c`)
- 背景：奶油色 (`#f7f4ed`)
- 標題文字：木炭黑 (`#1c1c1c`)
- 內文文字：暗淡灰 (`#5f5f5d`)
- 邊框：`#eceae4` (被動), `rgba(28,28,28,0.4)` (互動)
- 焦點：`rgba(0,0,0,0.1) 0px 4px 12px`
- 深色背景上的按鈕文字：米白 (`#fcfbf8`)

### 元件提示範例
- 「在奶油色背景 (#f7f4ed) 上建立一個主視覺區塊。標題 60px Camera Plain Variable 字重 600，行高 1.10，字距 -1.5px，顏色 #1c1c1c。副標題 18px 字重 400，行高 1.38，顏色 #5f5f5d。深色 CTA 按鈕 (#1c1c1c 背景, #fcfbf8 文字, 6px 圓角, 8px 16px 內距, 內陰影) 和幽靈按鈕 (透明背景, 1px solid rgba(28,28,28,0.4) 邊框, 6px 圓角)。」
- 「設計一張在奶油色 (#f7f4ed) 背景上的卡片。邊框：1px solid #eceae4，圓角 12px，無方框陰影。標題 20px Camera Plain Variable 字重 400，行高 1.25，顏色 #1c1c1c。內文 14px 字重 400，顏色 #5f5f5d。」
- 「建立一個範本畫廊：由圓角 12px、邊框 1px solid #eceae4、背景奶油色的卡片組成的網格。每個卡片：頂部圓角 12px 的圖片，下方為標題。懸停時：細微的邊框加深。」
- 「建立導覽列：固定在奶油色 (#f7f4ed) 背景上。連結使用 Camera Plain 16px 字重 400，#1c1c1c 文字。右對齊的帶內陰影深色 CTA 按鈕。行動端：6px 圓角的漢堡選單。」
- 「設計一個數據區塊：大數字 48px Camera Plain 字重 600，字距 -1.2px，#1c1c1c。下方標籤 16px 字重 400，#5f5f5d。水平佈局，間距 32px。」

### 迭代指南
1. 務必使用奶油色 (`#f7f4ed`) 作為基礎色 —— 絕不使用純白。
2. 透過調節 `#1c1c1c` 的不透明度層級來衍生灰色，而非使用不同的十六進位值。
3. 使用 `#eceae4` 邊框進行封裝，而非陰影。
4. 字距隨尺寸縮放：60px 為 -1.5px，48px 為 -1.2px，36px 為 -0.9px，16px 為標準。
5. 兩種字重：400 (標題以外的一切) 和 600 (標題)。
6. 深色按鈕上的內陰影是標誌性的細節 —— 絕不能遺漏。
7. 字重 480 的 Camera Plain Variable 僅用於特殊的展示時刻。
