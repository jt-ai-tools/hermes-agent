## 1. 視覺主題與氛圍

Apple 的網站是控制戲劇感的傑作 —— 廣闊的純黑色與近乎白色作為產品的電影化背景，這些產品被拍攝得如同藝廊中的雕塑一般。設計理念的核心是「化約 (reductive)」：每個像素都是為了產品而存在，介面本身退後，直到變得隱形。這不是作為美學偏好的極簡主義；這是對物體表示敬畏的極簡主義。

排版錨定了一切。San Francisco (SF Pro Display 用於大尺寸，SF Pro Text 用於內文) 是 Apple 的專有字體，其設計具有「光學尺寸 (optical sizing)」，可根據點數大小自動調整字元形狀。在顯示尺寸 (56px) 下，字重 600、行高 1.07 並搭配細微的負字距 (-0.28px)，創造出的標題感覺像是加工製造而非排版出來的 —— 精確、自信且毫不避諱。在內文尺寸 (17px) 下，字距稍微放寬 (-0.374px)，行高打開至 1.47，創造出舒適且不顯鬆散的閱讀節奏。

色彩故事是鮮明的二元對立。產品區塊在純黑 (`#000000`) 背景搭配白色文字與淺灰色 (`#f5f5f7`) 背景搭配近乎黑色文字 (`#1d1d1f`) 之間交替。這營造出一種電影般的節奏 —— 深色區塊感覺沉浸且高級，淺色區塊則感覺開放且具資訊性。唯一的色彩點綴是 Apple Blue (`#0071e3`)，專門保留給互動元素：連結、按鈕和聚焦狀態。在這一片中性色調中，這單一的點綴色讓每個可點擊元素都具有不容錯過的能見度。

**關鍵特徵：**
- 具備光學尺寸的 SF Pro Display/Text —— 字元形狀會根據尺寸上下文自動適應
- 二元的光影區塊節奏：黑色 (`#000000`) 與淺灰色 (`#f5f5f7`) 交替
- 單一標誌顏色：Apple Blue (`#0071e3`) 專門保留給互動元素
- 在純色背景上以產品為英雄的攝影 —— 沒有漸層、沒有紋理、沒有干擾
- 極其緊湊的標題行高 (1.07-1.14)，創造出壓縮且具看板衝擊力的效果
- 內容居中的全寬區塊佈局 —— 視窗就是畫布
- 藥丸形狀的 CTA (980px 半徑)，創造出柔和、易於親近的行動按鈕
- 區段之間大方的留白，讓每個產品時刻都能呼吸

## 2. 調色盤與角色

### 主要 (Primary)
- **Pure Black** (`#000000`): Hero 區段背景，沉浸式產品展示。為最明亮的產品提供最深邃的畫布。
- **Light Gray** (`#f5f5f7`): 交替區段背景，資訊區域。並非純白 —— 輕微的藍灰色調可防止單調死板。
- **Near Black** (`#1d1d1f`): 淺色背景上的主要文字，深色按鈕填滿。比純黑色稍微溫暖，以提供舒適的閱讀。

### 互動 (Interactive)
- **Apple Blue** (`#0071e3`): `--sk-focus-color`，主要 CTA 背景，聚焦環。介面中唯一的彩色。
- **Link Blue** (`#0066cc`): `--sk-body-link-color`，行內文字連結。比 Apple Blue 稍微深一些，以提高文字層級的可讀性。
- **Bright Blue** (`#2997ff`): 深色背景上的連結。在黑色區段上具有更高的亮度以提供對比。

### 文字 (Text)
- **White** (`#ffffff`): 深色背景上的文字，藍色/深色 CTA 上的按鈕文字。
- **Near Black** (`#1d1d1f`): 淺色背景上的主要內文文字。
- **Black 80%** (`rgba(0, 0, 0, 0.8)`): 次要文字，淺色背景上的導覽項目。稍微柔化。
- **Black 48%** (`rgba(0, 0, 0, 0.48)`): 三級文字，停用狀態，輪播控制項。

### 表面與深色變體 (Surface & Dark Variants)
- **Dark Surface 1** (`#272729`): 深色區段中的卡片背景。
- **Dark Surface 2** (`#262628`): 深色背景中的細微表面變化。
- **Dark Surface 3** (`#28282a`): 深色背景上的提升卡片。
- **Dark Surface 4** (`#2a2a2d`): 最高的深色表面提升。
- **Dark Surface 5** (`#242426`): 最深的深色表面色調。

### 按鈕狀態 (Button States)
- **Button Active** (`#ededf2`): 淺色按鈕的啟用/按下狀態。
- **Button Default Light** (`#fafafc`): 搜尋/篩選按鈕背景。
- **Overlay** (`rgba(210, 210, 215, 0.64)`): 媒體控制遮罩、疊加層。
- **White 32%** (`rgba(255, 255, 255, 0.32)`): 深色強制互動視窗 (modal) 關閉按鈕上的懸停狀態。

### 陰影 (Shadows)
- **Card Shadow** (`rgba(0, 0, 0, 0.22) 3px 5px 30px 0px`): 產品卡片的柔和、擴散提升。偏移與寬範圍的模糊創造出自然的攝影感陰影。

## 3. 字體排版規則

### 字體系列 (Font Family)
- **顯示 (Display)**: `SF Pro Display`, 備用字體: `SF Pro Icons, Helvetica Neue, Helvetica, Arial, sans-serif`
- **內文 (Body)**: `SF Pro Text`, 備用字體: `SF Pro Icons, Helvetica Neue, Helvetica, Arial, sans-serif`
- SF Pro Display 用於 20px 及以上；SF Pro Text 針對 19px 及以下進行了優化。

### 原則
- **光學尺寸作為設計哲學**：SF Pro 會自動在 Display 和 Text 光學尺寸之間切換。Display 版本具有更寬的字距和針對大尺寸優化的較細筆畫；Text 版本針對小尺寸則更緊湊、更穩固。這意味著字體會根據上下文從根本上改變其 DNA。
- **字重限制**：比例範圍從 300 (light) 到 700 (bold)，但大多數文字集中在 400 (regular) 和 600 (semibold)。300 字重僅出現在大型裝飾性文字中。700 字重非常罕見，僅用於粗體卡片標題。
- **所有尺寸下的負字距 (Negative tracking)**：與大多數僅在標題中使用字距調整的系統不同，Apple 即便在內文字體大小下也會應用細微的負字距（17px 為 -0.374px，14px 為 -0.224px，12px 為 -0.12px）。這創造了普遍緊湊、高效的文字感。
- **極端的行高範圍**：標題壓縮至 1.07，而內文則打開至 1.47，某些按鈕上下文甚至拉伸至 2.41。這種劇烈的範圍僅透過節奏就創造了清晰的視覺層級。

## 4. 元件樣式

### 按鈕 (Buttons)

**主要藍色 (Primary Blue / CTA)**
- 背景：`#0071e3` (Apple Blue)
- 文字：`#ffffff`
- 內邊距 (Padding)：8px 15px
- 半徑 (Radius)：8px
- 邊框：1px solid transparent
- 字體：SF Pro Text, 17px, weight 400
- 懸停 (Hover)：背景稍微變亮
- 啟用 (Active)：`#ededf2` 背景偏移
- 聚焦 (Focus)：`2px solid var(--sk-focus-color, #0071E3)` 輪廓
- 用途：主要行動呼籲（「購買」、「選購 iPhone」）

**主要深色 (Primary Dark)**
- 背景：`#1d1d1f`
- 文字：`#ffffff`
- 內邊距：8px 15px
- 半徑：8px
- 字體：SF Pro Text, 17px, weight 400
- 用途：次要 CTA，深色變體

**藥丸連結 (Pill Link / Learn More / Shop)**
- 背景：透明
- 文字：`#0066cc` (淺色背景) 或 `#2997ff` (深色背景)
- 半徑：980px (完整藥丸)
- 邊框：1px solid `#0066cc`
- 字體：SF Pro Text, 14px-17px
- 懸停：底線裝飾
- 用途：「了解更多」和「選購」連結 —— Apple 標誌性的行內 CTA

**篩選 / 搜尋按鈕 (Filter / Search Button)**
- 背景：`#fafafc`
- 文字：`rgba(0, 0, 0, 0.8)`
- 內邊距：0px 14px
- 半徑：11px
- 邊框：3px solid `rgba(0, 0, 0, 0.04)`
- 聚焦：`2px solid var(--sk-focus-color, #0071E3)` 輪廓
- 用途：搜尋列、篩選控制項

**媒體控制項 (Media Control)**
- 背景：`rgba(210, 210, 215, 0.64)`
- 文字：`rgba(0, 0, 0, 0.48)`
- 半徑：50% (圓形)
- 啟用：scale(0.9), 背景偏移
- 聚焦：`2px solid var(--sk-focus-color, #0071e3)` 輪廓，白色背景，黑色文字
- 用途：播放/暫停、輪播箭頭

### 卡片與容器 (Cards & Containers)
- 背景：`#f5f5f7` (淺色) 或 `#272729`-`#2a2a2d` (深色)
- 邊框：無 (在 Apple 的系統中邊框非常罕見)
- 半徑：5px-8px
- 陰影：對於提升的產品卡片使用 `rgba(0, 0, 0, 0.22) 3px 5px 30px 0px`
- 內容：居中，大方的內邊距
- 懸停：沒有標準的懸停狀態 —— 卡片是靜態的，其中的連結是互動的

### 導覽 (Navigation)
- 背景：`rgba(0, 0, 0, 0.8)` (半透明深色) 搭配 `backdrop-filter: saturate(180%) blur(20px)`
- 高度：48px (緊湊)
- 文字：`#ffffff` 於 12px, weight 400
- 啟用：懸停時有底線
- 標誌：Apple 標誌 (SVG) 置中或向左對齊，17x48px 視窗
- 行動裝置：收合為漢堡選單，帶有全螢幕疊加選單
- 導覽列浮動在內容上方，無論區段背景為何都保持其深色半透明玻璃效果

### 圖片處理 (Image Treatment)
- 在純色背景（黑色或白色）上的產品 —— 沒有背景、沒有上下文，只有物體本身
- 跨越整個視窗寬度的全版區段圖片
- 極高解析度且帶有細微陰影的產品攝影
- 侷限在圓角容器（12px+ 半徑）中的生活風格圖片

### 獨特元件 (Distinctive Components)

**產品 Hero 模組 (Product Hero Module)**
- 具備純色背景（黑色或 `#f5f5f7`）的全視窗寬度區段
- 產品名稱作為主要標題 (SF Pro Display, 56px, weight 600)
- 下方為較細字重的單行描述
- 兩個並排的藥丸 CTA：「了解更多」(外框) 和 「購買」 / 「選購」 (實心填滿)

**產品網格區塊 (Product Grid Tile)**
- 對比背景上的正方形或近正方形卡片
- 產品圖片佔據區塊的 60-70%
- 下方為產品名稱 + 單行描述
- 底部為「了解更多」和「選購」連結對

**特色比較條 (Feature Comparison Strip)**
- 產品變體水平滾動
- 每個變體作為一個垂直卡片，帶有圖片、名稱和關鍵規格
- 極簡化裝飾 —— 讓產品自己說話

## 5. 佈局原則 (Layout Principles)

### 間距系統 (Spacing System)
- 基礎單位：8px
- 級別：2px, 4px, 5px, 6px, 7px, 8px, 9px, 10px, 11px, 14px, 15px, 17px, 20px, 24px
- 顯著特徵：比例在小尺寸下非常密集 (2-11px)，具備精細的 1px 增量，然後以較大幅度跳躍。這允許對排版和圖示對齊進行精確的微調。

### 網格與容器 (Grid & Container)
- 最大內容寬度：約 980px (藥丸按鈕中重複出現的「980px 半徑」呼應了此寬度)
- Hero：帶有置中內容塊的全視窗寬度區段
- 產品網格：置中容器內的 2-3 欄佈局
- Hero 時刻使用單欄 —— 一個產品、一個訊息、全神貫注
- 沒有可見的網格線或間隙 (gutters) —— 間距創造了隱含的結構

### 留白哲學 (Whitespace Philosophy)
- **電影般的呼吸空間**：每個產品區段佔據整個視窗高度（或接近高度）。產品之間的留白並非空洞 —— 它是電影中場景之間的停頓。
- **透過色塊建立垂直節奏**：Apple 並非僅使用間距來分隔區段，而是使用交替的背景顏色（黑色、`#f5f5f7`、白色）。每次顏色變化都標誌著一個新的「場景」。
- **對內壓縮，對外擴張**：文字塊設置得很緊湊（負字距、緊湊行高），而周圍空間則很廣闊。這創造了密度與開放性之間的張力。

### 邊框半徑級別 (Border Radius Scale)
- 極小 (5px): 小型容器, 連結標籤
- 標準 (8px): 按鈕, 產品卡片, 圖片容器
- 舒適 (11px): 搜尋輸入框, 篩選按鈕
- 大型 (12px): 特色面板, 生活風格圖片容器
- 完整藥丸 (980px): CTA 連結 (「了解更多」, 「選購」), 導覽藥丸
- 圓形 (50%): 媒體控制項 (播放/暫停, 箭頭)

## 6. 深度與提升 (Depth & Elevation)

| 級別 | 處理方式 | 用途 |
|-------|-----------|-----|
| 平面 (Level 0) | 無陰影, 純色背景 | 標準內容區段, 文字塊 |
| 導覽玻璃 | 在 `rgba(0,0,0,0.8)` 上使用 `backdrop-filter: saturate(180%) blur(20px)` | 置頂導覽列 —— 玻璃效果 |
| 細微提升 (Level 1) | `rgba(0, 0, 0, 0.22) 3px 5px 30px 0px` | 產品卡片, 浮動元素 |
| 媒體控制 | 帶有縮放轉換的 `rgba(210, 210, 215, 0.64)` 背景 | 播放/暫停按鈕, 輪播控制項 |
| 聚焦 (無障礙) | `2px solid #0071e3` 輪廓 | 所有互動元素的鍵盤聚焦 |

**陰影哲學**：Apple 極其節制地使用陰影。主要的陰影 (`3px 5px 30px`，透明度 0.22) 柔和、寬廣且偏移 —— 模擬擴散的工作室燈光在實體物件下方投射出的自然陰影。這強化了「產品即實體雕塑」的隱喻。大多數元素完全沒有陰影；提升感來自背景顏色對比（深色背景上的更深色卡片，或略微不同的灰色背景上的淺色卡片）。

### 裝飾性深度
- 導覽玻璃：半透明、模糊的導覽列是最具辨識度的深度元素，創造出一種 UI 浮動在滾動內容上方的感覺
- 區段顏色轉換：深度是透過黑色與淺灰色區塊的交替來暗示的，而非陰影
- 產品攝影陰影：產品本身在攝影中投射陰影，因此 UI 不需要添加合成陰影

## 7. 該做與不該做的事 (Do's and Don'ts)

### 該做 (Do)
- 在 20px+ 使用 SF Pro Display，20px 以下使用 SF Pro Text —— 尊重光學尺寸邊界
- 在所有文字大小（不只是標題）應用負字距 —— Apple 普遍調整得非常緊湊
- Apple Blue (`#0071e3`) 「僅」用於互動元素 —— 它必須是唯一的點綴
- 在黑色與淺灰色 (`#f5f5f7`) 區段背景之間交替，以營造電影節奏
- 為 CTA 連結使用 980px 藥丸半徑 —— Apple 標誌性的連結形狀
- 保持產品圖像在純色背景上，沒有競爭的視覺元素
- 為置頂導覽使用半透明深色玻璃 (`rgba(0,0,0,0.8)` + 模糊)
- 將標題行高壓縮至 1.07-1.14 —— Apple 的標題以緊湊著稱

### 不該做 (Don't)
- 不要引入額外的點綴色 —— 所有的彩色預算都花在藍色上了
- 不要使用沉重的陰影或多層陰影 —— Apple 的陰影系統要麼是一個柔和擴散陰影，要麼什麼都沒有
- 卡片或容器上不要使用邊框 —— Apple 幾乎從不使用可見邊框（特定按鈕除外）
- 不要為 SF Pro 應用寬字距 —— 它被設計為在各種大小下都運行得很緊湊
- 不要使用 800 或 900 字重 —— 最高為 700 (bold)，即便如此也很少見
- 背景中不要添加紋理、圖案或漸層 —— 僅限純色
- 不要讓導覽列不透明 —— 玻璃模糊效果對 Apple UI 識別至關重要
- 不要置中對齊內文 —— Apple 的內文是向左對齊的；只有標題置中
- 矩形元素上不要使用大於 12px 的圓角 (980px 僅用於藥丸)

## 8. 響應式行為

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 小型行動裝置 (Small Mobile) | <360px | 最低支援寬度, 單欄 |
| 行動裝置 (Mobile) | 360-480px | 標準行動裝置佈局 |
| 大型行動裝置 (Mobile Large) | 480-640px | 較寬的單欄, 較大的圖片 |
| 平板 小型 (Tablet Small) | 640-834px | 開始 2 欄產品網格 |
| 平板 (Tablet) | 834-1024px | 完整平板佈局, 展開的導覽列 |
| 桌機 小型 (Desktop Small) | 1024-1070px | 開始標準桌機佈局 |
| 桌機 (Desktop) | 1070-1440px | 完整佈局, 最大內容寬度 |
| 大型桌機 (Large Desktop) | >1440px | 內容居中並帶有寬大的邊界 |

### 觸控目標 (Touch Targets)
- 主要 CTA：8px 15px 內邊距創造出約 44px 的觸控高度
- 導覽連結：48px 高度並帶有充足間距
- 媒體控制項：50% 半徑的圓形按鈕，最小 44x44px
- 「了解更多」藥丸：大方的內邊距以利舒適點擊

### 收合策略 (Collapsing Strategy)
- Hero 標題：行動裝置上從 56px Display → 40px → 28px，依比例保持緊湊行高
- 產品網格：3 欄 → 2 欄 → 單欄堆疊
- 導覽：完整水平導覽 → 緊湊行動選單 (漢堡選單)
- 產品 Hero 模組：在各種尺寸下保持全版寬度，文字按比例縮小
- 區段背景：在所有斷點下保持全寬色塊 —— 電影節奏從不中斷
- 圖片大小：產品依比例縮放，絕不裁切 —— 產品輪廓是神聖的

### 圖片行為 (Image Behavior)
- 產品攝影在所有斷點下保持寬高比
- Hero 產品圖片縮小但保持居中
- 全版區段背景在各種尺寸下持續存在
- 生活風格圖片在行動裝置上可能會裁切，但保持其圓角
- 對摺頁下方的產品圖片進行延遲載入 (lazy loading)

## 9. 代理（Agent）提示指南

### 快速顏色參考
- 主要 CTA: Apple Blue (`#0071e3`)
- 頁面背景 (淺色): `#f5f5f7`
- 頁面背景 (深色): `#000000`
- 標題文字 (淺色): `#1d1d1f`
- 標題文字 (深色): `#ffffff`

### 元件提示範例
- 「在黑色背景上建立一個 hero 區段。標題為 56px SF Pro Display weight 600, 行高 1.07, 字距 -0.28px, 白色。單行副標題為 21px SF Pro Display weight 400, 行高 1.19, 白色。兩個藥丸 CTA：『了解更多』（透明背景、白色文字、1px 白色實心邊框、980px 半徑）和『購買』（Apple Blue #0071e3 背景、白色文字、8px 半徑、8px 15px 內邊距）。」
- 「設計一個產品卡片：#f5f5f7 背景, 8px 邊框半徑, 無邊框, 無陰影。產品圖片位於卡片上方 60% 的純色背景。標題為 28px SF Pro Display weight 400, 字距 0.196px, 行高 1.14。描述為 14px SF Pro Text weight 400, 顏色 rgba(0,0,0,0.8)。『了解更多』和『選購』連結顏色為 #0066cc，大小 14px。」
- 「建立 Apple 導覽列：置頂, 48px 高度, 背景 rgba(0,0,0,0.8) 並搭配 backdrop-filter: saturate(180%) blur(20px)。連結為 12px SF Pro Text weight 400, 白色文字。Apple 標誌在左, 連結居中, 搜尋與購物袋圖示在右。」
- 「建立交替區段佈局：第一節為黑色背景搭配白色文字與置中產品圖片，第二節為 #f5f5f7 背景搭配 #1d1d1f 文字。每個區段接近全視窗高度，帶有 56px 標題與下方兩個藥丸 CTA。」
- 「設計一個『了解更多』連結：文字在淺色背景為 #0066cc 或在深色背景為 #2997ff, 14px SF Pro Text, 懸停時有底線。在文字後包含一個右箭頭符號 (>)。當作為獨立 CTA 使用時，將其包裝在一個具有 980px 邊框半徑的容器中以形成藥丸形狀。」

### 迭代指南
1. 每個互動元素都使用 Apple Blue (`#0071e3`) —— 不使用其他點綴色
2. 區段背景交替：黑色用於沉浸式時刻，`#f5f5f7` 用於資訊時刻
3. 字體光學尺寸：20px+ 使用 SF Pro Display，20px 以下使用 SF Pro Text —— 絕不混用
4. 所有尺寸下的負字距：56px 為 -0.28px，17px 為 -0.374px，14px 為 -0.224px，12px 為 -0.12px
5. 導覽玻璃效果（半透明深色 + 模糊）是不容談判的 —— 它定義了 Apple 的網頁體驗
6. 產品總是出現在純色背景上 —— 在 hero 模組中絕不出現在漸層、紋理或生活風格背景上
7. 陰影非常罕見且總是柔和：`3px 5px 30px`，透明度 0.22 或完全沒有
8. 藥丸 CTA 使用 980px 半徑 —— 這創造了標誌性的 Apple「看起來像膠囊的圓角矩形」
