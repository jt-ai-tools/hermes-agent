## 1. 視覺主題與氛圍

Vercel 的網站是「使開發者基礎設施透明化」的視覺化論文——一個極度克制甚至帶有哲學意味的設計系統。頁面以壓倒性的白色 (`#ffffff`) 搭配近乎黑色的文字 (`#171717`)，創造出一種藝廊般的空曠感，每個元素都必須證明其存在的必要。這不是作為裝飾的極簡主義，而是作為工程原則的極簡主義。Geist 設計系統對待介面就像編譯器對待程式碼一樣——剝離所有不必要的標記，直到只剩下結構。

自定義的 Geist 字體系列是這頂皇冠上的明珠。Geist Sans 在展示尺寸下使用極具攻擊性的負字元間距（-2.4px 到 -2.88px），使標題顯得緊湊、迫切且充滿工程感——就像為了生產環境而壓縮後的程式碼。在內文尺寸下，字距稍微放寬，但幾何精確度依然存在。Geist Mono 作為等寬伴侶，用於程式碼、終端機輸出和技術標籤，完善了整個系統。這兩款字體都全局啟用了 OpenType 的 `"liga"`（連字）功能，為排版增添了一層經得起細讀的精緻感。

讓 Vercel 與其他單色設計系統區分開來的是其「以陰影替代邊框 (shadow-as-border)」的哲學。Vercel 不使用傳統的 CSS 邊框，而是使用 `box-shadow: 0px 0px 0px 1px rgba(0,0,0,0.08)`——一個零偏移、零模糊、1px 擴散的陰影，這創造了一個類似邊框的線條，卻沒有盒子模型 (box model) 的影響。這種技術允許邊框存在於陰影層，實現更平滑的過渡、不被裁剪的圓角，以及比傳統邊框更微妙的視覺重量。整個深度系統建立在分層的多值陰影堆疊上，每一層都有特定目的：一層用於邊框，一層用於柔和的高度提升，一層用於環境深度。

**核心特徵：**
- Geist Sans 帶有極致的負字元間距（展示尺寸為 -2.4px 至 -2.88px）——文字如同壓縮後的基礎設施。
- Geist Mono 用於程式碼和技術標籤，全局啟用 OpenType `"liga"`。
- 「以陰影替代邊框」技術：`box-shadow 0px 0px 0px 1px` 在整個系統中取代了傳統邊框。
- 多層陰影堆疊用於細緻的深度（單個聲明中包含邊框 + 提升 + 環境感）。
- 近乎純白的畫布搭配 `#171717` 文字——並非全黑，創造出微對比的柔和感。
- 工作流特定的強調色：發佈紅 (Ship Red, `#ff5b4f`)、預覽粉 (Preview Pink, `#de1d8d`)、開發藍 (Develop Blue, `#0a72ef`)。
- 使用 `hsla(212, 100%, 48%, 1)` 的焦點環系統——飽和度高的藍色，兼顧無障礙設計。
- 帶有著色背景的藥丸狀徽章 (9999px)，用於狀態指示。

## 2. 調色盤與角色

### 主要顏色 (Primary)
- **Vercel Black** (`#171717`)：主要文字、標題、深色表面背景。並非純黑——輕微的暖色調防止視覺上的生硬。

- **Ring Border** (`rgb(235, 235, 235) 0px 0px 0px 1px`)：用於分頁標籤和圖片的淺灰色環狀邊框。

### 原則
- **壓縮感即身份**：展示尺寸的 Geist Sans 使用 -2.4px 到 -2.88px 的字元間距——這是所有主要設計系統中最具攻擊性的負字距處理。這創造了一種「被縮小 (minified)」的文字感，如同針對生產環境優化後的程式碼。字距隨尺寸縮小而逐漸放寬：32px 為 -1.28px，24px 為 -0.96px，16px 為 -0.32px，14px 則恢復標準。
- **連字無處不在**：每個 Geist 文字元素都啟用了 OpenType `"liga"`。連字並非裝飾性的——它們是結構性的，創造出更緊湊、更高效的字元組合。
- **三種字重，角色嚴格**：400（內文/閱讀）、500（UI/互動）、600（標題/強調）。除了微小的微型徽章外，不使用 700 (Bold)。這種窄字重範圍透過大小和字距（而非重量）來建立階層。
- **等寬字體體現身份**：全大寫且帶有 `"tnum"` 或 `"liga"` 的 Geist Mono 作為「開發者控制台」的聲音——代表緊湊的技術標籤，將行銷網站與產品連接起來。

## 5. 版面配置原則

### 間距系統 (Spacing System)
- 基礎單位：8px
- 階層：1px, 2px, 3px, 4px, 5px, 6px, 8px, 10px, 12px, 14px, 16px, 32px, 36px, 40px
- 顯著跳躍：從 16px 直接跳到 32px——主要階層中沒有 20px 或 24px

### 格線與容器 (Grid & Container)
- 最大內容寬度：約 1200px
- 英雄區段：居中的單欄佈局，帶有大方的頂部內距
- 特色區段：用於卡片的 2–3 欄格線
- 使用 `border-bottom: 1px solid #171717` 的全寬分隔線
- 程式碼/儀表板截圖以全寬或帶邊框的方式呈現

### 留白哲學 (Whitespace Philosophy)
- **藝廊般的空曠感**：區段之間巨大的垂直內距 (80px–120px+)。留白「就是」設計——它傳達了 Vercel 無需證明也無須隱藏的自信。
- **壓縮文字與擴張空間**：標題上攻擊性的負字距被周圍大方的留白所平衡。文字是密集的；其周圍空間是廣闊的。
- **區段節奏**：白色區段與白色區段交替——區段之間沒有顏色變化。區隔完全來自邊框（陰影邊框）和間距。

### 圓角縮放 (Border Radius Scale)
- 微型 (2px)：內聯程式碼片段、小跨度元素
- 微妙 (4px)：小型容器
- 標準 (6px)：按鈕、連結、功能性元素
- 舒適 (8px)：卡片、列表項目
- 圖片 (12px)：特色卡片、圖片容器（頂部圓角）
- 大型 (64px)：分頁導覽藥丸
- 特大 (100px)：大型導覽連結
- 全藥丸 (9999px)：徽章、狀態藥丸、標籤
- 圓形 (50%)：選單切換、頭像容器

## 6. 深度與高度 (Depth & Elevation)

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 扁平 (Level 0) | 無陰影 | 頁面背景、文字區塊 |
| 環狀 (Level 1) | `rgba(0,0,0,0.08) 0px 0px 0px 1px` | 大多數元素的以陰影替代邊框處理 |
| 淺環 (Level 1b) | `rgb(235,235,235) 0px 0px 0px 1px` | 用於分頁標籤、圖片的更淺環狀邊框 |
| 微妙卡片 (Level 2) | 環狀 + `rgba(0,0,0,0.04) 0px 2px 2px` | 帶有極小提升感的標準卡片 |
| 完整卡片 (Level 3) | 環狀 + 微妙層 + `rgba(0,0,0,0.04) 0px 8px 8px -8px` + 內部 `#fafafa` 環 | 特色卡片、強調面板 |
| 焦點 (無障礙) | `2px solid hsla(212, 100%, 48%, 1)` 外框 | 所有互動元素的鍵盤焦點 |

**陰影哲學**：Vercel 擁有現代網頁設計中最精密的陰影系統。Vercel 並非按照傳統 Material Design 的方式將陰影用於高度提升，而是使用多值陰影堆疊，每一層都有獨特的建築目的：一層創造「邊框」（0px 擴散，1px），另一層增加環境柔和感（2px 模糊），再一層處理遠距離深度（8px 模糊配負擴散），以及一個內環 (`#fafafa`) 創造微妙的高光，讓卡片從內部產生「光澤」。這種分層方法讓卡片感覺是「構建」出來的，而非漂浮著。

### 裝飾性深度
- 英雄區段漸層：英雄內容背後極其微妙、幾乎看不見的粉彩色多色漸層。
- 區段邊框：主要區段之間使用 `1px solid #171717`（全深色線條）。
- 無背景顏色變化——深度完全來自陰影分層和邊框對比。

## 7. 規範事項 (Do's and Don'ts)

### 應該 (Do)
- 在展示尺寸下使用帶有攻擊性負字距的 Geist Sans（48px 時為 -2.4px 至 -2.88px）。
- 使用「以陰影替代邊框」 (`0px 0px 0px 1px rgba(0,0,0,0.08)`) 代替傳統 CSS 邊框。
- 在所有 Geist 文字上啟用 `"liga"` ——連字是結構性的，而非可選的。
- 使用三字重系統：400（內文）、500（UI）、600（標題）。
- 僅在工作流上下文中應用工作流強調色（紅/粉/藍）。
- 為卡片應用多層陰影堆疊（邊框 + 提升 + 環境 + 內高光）。
- 保持調色盤為無彩色系——從 `#171717` 到 `#ffffff` 的灰色階即為系統主色。
- 主要文字使用 `#171717` 而非 `#000000` ——那種微小的暖色調至關重要。

### 不應該 (Don't)
- 不要在 Geist Sans 上使用正字元間距——它始終是負值或零。
- 不要在內文中使用字重 700 (Bold) —— 600 是上限，且僅用於標題。
- 不要在卡片上使用傳統 CSS `border` —— 使用陰影邊框技術。
- 不要在 UI 介面中引入暖色（橘、黃、綠）。
- 不要將工作流強調色（發佈紅、預覽粉、開發藍）作為裝飾用途。
- 不要使用重陰影 (> 0.1 透明度) —— 陰影系統應該是極其輕微的。
- 不要增加內文的字元間距 —— Geist 的設計初衷就是緊湊。
- 不要在主要動作按鈕上使用藥丸圓角 (9999px) —— 藥丸狀僅用於徽章/標籤。
- 不要跳過卡片陰影中的內部 `#fafafa` 環 —— 它是讓系統運作的關鍵光澤。

## 8. 響應式行為

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變更 |
|------|-------|-------------|
| 小行動版 | <400px | 緊湊單欄，最小內距 |
| 行動版 | 400–600px | 標準行動版，堆疊佈局 |
| 小平板 | 600–768px | 開始出現 2 欄格線 |
| 平板 | 768–1024px | 完整卡片格線，擴展內距 |
| 小桌機 | 1024–1200px | 標準桌機佈局 |
| 桌機 | 1200–1400px | 完整佈局，最大內容寬度 |
| 大桌機 | >1400px | 居中，大方的邊距 |

### 觸控目標 (Touch Targets)
- 按鈕使用舒適的內距（垂直 8px–16px）。
- 導覽連結為 14px，並帶有足夠的間距。
- 藥丸徽章具有 10px 的水平內距，方便點擊。
- 行動版選單切換使用 50% 圓角的圓形按鈕。

### 收摺策略 (Collapsing Strategy)
- 英雄標題：48px 展示字體依比例縮小，保持負字距處理。
- 導覽列：水平連結 + CTA → 漢堡選單。
- 特色卡片：3 欄 → 2 欄 → 單欄堆疊。
- 程式碼截圖：保持寬高比，可能需要水平滾動。
- 信任標誌列：格線 → 水平滾動。
- 頁尾：多欄 → 堆疊單欄。
- 區段間距：行動版從 80px+ 縮小至 48px。

### 圖片行為
- 儀表板截圖在所有尺寸下皆保持邊框處理。
- 英雄漸層在行動版上變得柔和/簡化。
- 產品截圖使用響應式圖片，並保持一致的圓角。
- 全寬區段保持邊對邊 (edge-to-edge) 的處理。

## 9. Agent 提示指南

### 快速顏色參考
- 主要 CTA：Vercel Black (`#171717`)
- 背景：Pure White (`#ffffff`)
- 標題文字：Vercel Black (`#171717`)
- 內文文字：Gray 600 (`#4d4d4d`)
- 邊框 (陰影型)：`rgba(0, 0, 0, 0.08) 0px 0px 0px 1px`
- 連結：Link Blue (`#0072f5`)
- 焦點環：Focus Blue (`hsla(212, 100%, 48%, 1)`)

### 元件提示範例
- 「在白色背景上建立一個英雄區段。標題為 48px Geist 字重 600，行高 1.00，字元間距 -2.4px，顏色 #171717。副標題為 20px Geist 字重 400，行高 1.80，顏色 #4d4d4d。深色 CTA 按鈕 (#171717, 6px 圓角, 8px 16px 內距) 和幽靈按鈕 (白色, 陰影邊框 rgba(0,0,0,0.08) 0px 0px 0px 1px, 6px 圓角)。」
- 「設計一個卡片：白色背景，無 CSS 邊框。使用陰影堆疊：rgba(0,0,0,0.08) 0px 0px 0px 1px, rgba(0,0,0,0.04) 0px 2px 2px, #fafafa 0px 0px 0px 1px。圓角 8px。標題為 24px Geist 字重 600，字元間距 -0.96px。內文為 16px 字重 400, #4d4d4d。」
- 「建立一個藥丸徽章：#ebf5ff 背景, #0068d6 文字, 9999px 圓角, 0px 10px 內距, 12px Geist 字重 500。」
- 「建立導覽列：白色置頂標頭。連結使用 Geist 14px 字重 500，#171717 文字。右側對齊深色藥丸 CTA『開始部署』。底部帶有陰影邊框：rgba(0,0,0,0.08) 0px 0px 0px 1px。」
- 「設計一個展示三個步驟的工作流區段：開發 (文字顏色 #0a72ef)、預覽 (#de1d8d)、發佈 (#ff5b4f)。每個步驟：14px Geist Mono 全大寫標籤 + 24px Geist 字重 600 標題 + 16px 字重 400 的 #4d4d4d 描述。」

### 迭代指南
1. 務必使用「以陰影替代邊框」而非 CSS 邊框——`0px 0px 0px 1px rgba(0,0,0,0.08)` 是基礎。
2. 字元間距隨字體大小縮放：48px 為 -2.4px，32px 為 -1.28px，24px 為 -0.96px，14px 則恢復標準。
3. 僅使用三種字重：400（閱讀）、500（互動）、600（宣佈）。
4. 顏色是功能性的，絕非裝飾性——工作流顏色（紅/粉/藍）僅用於標記管線階段。
5. 卡片陰影中的內部 `#fafafa` 環是賦予 Vercel 卡片微妙內光澤的關鍵。
6. 技術標籤使用全大寫的 Geist Mono，其餘一切使用 Geist Sans。
