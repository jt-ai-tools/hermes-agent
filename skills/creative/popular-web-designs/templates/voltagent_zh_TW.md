## 1. 視覺主題與氛圍

VoltAgent 的介面是人工智慧時代的深空指令終端——一個面向開發者的黑暗世界，建立在近乎純黑的表面 (`#050507`) 之上，唯一的干擾是翡翠綠能量的電脈衝。整個體驗喚起了一種凌晨 2 點盯著高效能 IDE 的感覺：深沉、專注，且充滿目標感。這不是一個友善的 SaaS 登陸頁面——它是一個工程平台，透過程式碼片段、架構圖和純粹的技術自信來展現自己。

綠色強調色 (`#00d992`) 被精確地運用——它從標題、邊框和互動元素中發出光芒，就像電路板上傳輸的訊號。在碳黑色的畫布上，這種綠色被解讀為「電源開啟」——這是對 AI Agent 工程平台的刻意視覺隱喻。輔助色調完全由暖中性灰色 (`#3d3a39`, `#8b949e`, `#b8b3b0`) 組成，這些色調在不引入色彩雜訊的情況下柔和了黑暗，創造出純藍灰色所缺乏的駕駛艙般的溫暖感。

排版在標題上依賴系統字體堆疊——實現了最高的渲染速度和原生感的權威性——而 Inter 則以幾何精確度承載了內文和 UI 文字。程式碼區塊使用 SFMono-Regular，這是開發者在終端機中看到的同一款字體，在每一次滾動中強化了工具的可信度。

**核心特徵：**
- 碳黑色畫布 (`#050507`) 搭配暖灰色邊框包圍 (`#3d3a39`) —— 不冷冽也不生硬。
- 單一強調色身份：翡翠訊號綠 (`#00d992`) 作為唯一的色彩能量來源。
- 雙字體系統：system-ui 用於具權威性的標題，Inter 用於精確的 UI/內文，SFMono 用於建立程式碼可信度。
- 極度緊湊的標題行高 (1.0–1.11)，創造出密集的、壓縮的力量區塊。
- 暖中性色調 (`#3d3a39`, `#8b949e`, `#b8b3b0`) 防止深色主題顯得過於冰冷。
- 開發者終端美學，程式碼片段就是英雄內容。
- 綠色發光效果 (`drop-shadow`、邊框強調) 讓 UI 元素感覺充滿電力。

## 2. 調色盤與角色

### 主要顏色 (Primary)
- **翡翠訊號綠 (Emerald Signal Green)** (`#00d992`)：核心品牌能量——用於強調邊框、發光效果和最高信號的互動時刻。這是整個介面的「電源開啟」指示燈。
- **VoltAgent Mint** (`#2fd6a1`)：品牌綠的按鈕文字變體——比純粹的訊號綠更溫暖且更易讀，專門用於深色表面上的 CTA 文字。
- **Tailwind Emerald** (`#10b981`)：生態系統標準綠色，以低透明度 (30%) 用於微妙的背景著色和連結預設值。將 VoltAgent 的自定義色調與 Tailwind 的工具類連接起來。

### 次要與強調色 (Secondary & Accent)
- **Soft Purple** (`#818cf8`)：一種冷調的靛紫色，節制地用於次要分類、程式碼語法高亮和增加視覺多樣性，而不會與綠色競爭。
- **Cobalt Primary** (`#306cce`)：Docusaurus 主要深藍色——在文件上下文中用於連結和互動焦點狀態。
- **Deep Cobalt** (`#2554a0`)：最深的藍色調，保留用於文件 UI 中的按下/活動狀態。
- **Ring Blue** (`#3b82f6`)：50% 透明度的 Tailwind 環狀顏色——僅在鍵盤焦點時顯示，以符合無障礙規範。

### 表面與背景 (Surface & Background)
- **深淵黑 (Abyss Black)** (`#050507`)：登陸頁面的畫布——一種帶有極細微暖色調的近乎純黑，比大多數「深色模式」更深，以實現與綠色強調色的最大對比。
- **碳纖表面 (Carbon Surface)** (`#101010`)：主要的卡片和按鈕背景——比 Abyss Black 淺一個色階，創造出幾乎難以察覺的高度層。用於所有封閉表面。
- **暖炭灰邊框 (Warm Charcoal Border)** (`#3d3a39`)：標誌性的包圍色——不是冷灰色而是暖調、幾乎帶褐色的深色，防止邊框在黑色畫布上顯得生硬。

### 中性色與文字 (Neutrals & Text)
- **雪白 (Snow White)** (`#f2f2f2`)：深色表面上的主要文字顏色——不是純白 (`#ffffff`) 而是柔和、對眼睛友善的米白色。這是網站上使用最多的顏色。
- **純白 (Pure White)** (`#ffffff`)：保留給最高強調時刻——幽靈按鈕文字和最高對比標題。以低透明度 (5%) 用於微妙的疊加效果。
- **暖羊皮紙色 (Warm Parchment)** (`#b8b3b0`)：次要內文——帶有輕微粉紅色調的暖灰色，在黑暗畫布上讀起來像「紙」。
- **鋼青板岩色 (Steel Slate)** (`#8b949e`)：三級文字、元數據、時間戳和弱化內容。一種冷調的藍灰色，提供暖羊皮紙色之下的清晰階層。
- **霧灰 (Fog Gray)** (`#bdbdbd`)：頁尾連結和輔助導覽文字——懸停時變亮為純白。
- **薄霧灰 (Mist Gray)** (`#dcdcdc`)：比霧灰稍亮，用於次要連結文字，懸停時過渡到亮綠色。
- **近白 (Near White)** (`#eeeeee`)：最高對比的次要文字，比雪白低一個階層。

### 語義與強調 (Semantic & Accent)
- **成功綠 (Success Emerald)** (`#008b00`)：深綠色，用於文件上下文中的成功狀態和正面確認。
- **Success Light** (`#80d280`)：柔和的粉綠色，用於成功背景和微妙的正面指示。
- **警告琥珀 (Warning Amber)** (`#ffba00`)：亮琥珀色，用於警告提醒和謹慎狀態。
- **Warning Pale** (`#ffdd80`)：柔和的琥珀色，用於警告背景填充。
- **危險珊瑚紅 (Danger Coral)** (`#fb565b`)：鮮紅色彩，用於錯誤狀態和破壞性動作警告。
- **Danger Rose** (`#fd9c9f`)：柔和的珊瑚粉色，用於錯誤背景。
- **資訊青 (Info Teal)** (`#4cb3d4`)：冷調的青藍色，用於資訊標註和提示說明。
- **虛線邊框色 (Dashed Border Slate)** (`#4f5d75` at 40%)：弱化的藍灰色，專門用於工作流圖表中的裝飾性虛線邊框。

### 漸層系統 (Gradient System)
- **綠色訊號光 (Green Signal Glow)**：`drop-shadow(0 0 2px #00d992)` 動畫過渡到 `drop-shadow(0 0 8px #00d992)` —— 在 VoltAgent 閃電 Logo 和互動元素上創造出脈衝般的「電荷」效果。光芒像心跳一樣擴張和收縮。
- **暖調氛圍煙霧 (Warm Ambient Haze)**：`rgba(92, 88, 85, 0.2) 0px 0px 15px` —— 一種暖色調的擴散陰影，在提升的卡片周圍創造出柔和的氣氛光，在邊緣可見而無銳利邊界。
- **深沉戲劇高度 (Deep Dramatic Elevation)**：`rgba(0, 0, 0, 0.7) 0px 20px 60px` 搭配 `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset` —— 沉重、戲劇性的向下陰影，搭配最突出懸浮元素的微弱內縮板岩色環。

## 3. 排版規則

### 字體系列 (Font Family)
- **主要字體 (標題)**：`system-ui`，備用字體：`-apple-system, Segoe UI, Roboto, Ubuntu, Cantarell, Noto Sans, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji, Segoe UI Symbol`
- **次要字體 (內文/UI)**：`Inter`，備用字體繼承自 system-ui 堆疊。OpenType 特性：啟用 `"calt", "rlig"`（上下文替換和必需連字）。
- **等寬字體 (程式碼)**：`SFMono-Regular`，備用字體：`Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`

### 原則
- **系統原生的權威感**：展示標題使用 system-ui 而非自定義 Web 字體 —— 這意味著最大的文字能立即渲染（無 FOIT/FOUT），並繼承作業系統的原生個性。在 macOS 上是 SF Pro，在 Windows 上是 Segoe UI。設計將這種多樣性視為特色而非缺陷。
- **緊湊壓縮創造密度**：英雄區段行高極度壓縮 (1.0) 並帶有負字距 (-0.65px 到 -0.9px)，創造出感覺像密集技術規格而非空氣感行銷文案的文字塊。
- **字重漸層而非對比**：系統使用溫和的 300→400→500→600→700 字重進階。粗體 (700) 保留給副標題和程式碼按鈕強調。大多數內文處於 400–500，創造微妙而非戲劇性的階層。
- **全大寫是謹慎且寬廣的**：當全大寫出現時，它總是搭配大方的字距 (0.45px–2.52px)，將密集的單詞轉化為間隔開的標籤。這種處理永遠不會應用於標題。
- **預設啟用 OpenType**：system-ui 和 Inter 都啟用了 `"calt"` 和 `"rlig"` 特性，確保整個介面的上下文標誌調整和連字渲染。

## 4. 元件樣式

### 按鈕 (Buttons)

**幽靈按鈕 / 描邊按鈕 (標準)**
- 背景：透明
- 文字：純白 (`#ffffff`)
- 內距 (Padding)：舒適 (12px 16px)
- 邊框：細實心暖炭灰 (`1px solid #3d3a39`)
- 圓角 (Radius)：舒適的圓角 (6px)
- 懸停 (Hover)：背景變暗至 `rgba(0, 0, 0, 0.2)`，透明度降至 0.4
- 外框 (Outline)：微妙的綠色調 (`rgba(33, 196, 93, 0.5)`)
- 預設互動元素——低調但明顯可點擊

**主要綠色 CTA**
- 背景：碳纖表面 (`#101010`)
- 文字：VoltAgent Mint (`#2fd6a1`)
- 內距：舒適 (12px 16px)
- 邊框：無明顯邊框（依賴外框的焦點指示器）
- 外框：VoltAgent Mint (`rgb(47, 214, 161)`)
- 懸停：與幽靈按鈕相同的變暗行為
- 「電源開啟」按鈕——深色表面上的綠色文字讀起來就像一條活動中的終端指令。

**三級 / 強調容器按鈕**
- 背景：碳纖表面 (`#101010`)
- 文字：雪白 (`#f2f2f2`)
- 內距：大方 (四邊皆為 20px)
- 邊框：厚實心暖炭灰 (`3px solid #3d3a39`)
- 圓角：舒適的圓角 (8px)
- 用於較大互動表面（程式碼複製塊、特色 CTA）的卡片式按鈕處理。

### 卡片與容器 (Cards & Containers)
- 背景：碳纖表面 (`#101010`) —— 比頁面畫布淺一個色階
- 邊框：標準包圍使用 `1px solid #3d3a39`（暖炭灰）；強調/活動卡片使用 `2px solid #00d992`
- 圓角：內容卡片為 8px；較小的內聯容器為 4–6px
- 陰影層級 1：暖調氛圍煙霧 (`rgba(92, 88, 85, 0.2) 0px 0px 15px`) 用於標準提升
- 陰影層級 2：深沉戲劇高度 (`rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 0px 0px 0px 1px inset`) 用於英雄/特色展示卡片
- 懸停行為：邊框顏色可能向綠色強調色偏移，或透明度微增
- 虛線變體：工作流/圖表容器使用 `1px dashed rgba(79, 93, 117, 0.4)` —— 在視覺上與實線邊框的內容卡片區隔

### 輸入框與表單 (Inputs & Forms)
- 網站專注於登陸頁面，表單 UI 極少。
- npm 安裝指令 (`npm create voltagent-app@latest`) 以程式碼塊而非輸入欄位形式呈現。
- 推導樣式：碳纖表面背景、暖炭灰邊框、VoltAgent Mint 焦點環、雪白文字。

### 導覽列 (Navigation)
- 在深淵黑畫布上的置頂固定導覽列
- Logo：帶有動畫綠色發光效果的 VoltAgent 閃電圖標（`drop-shadow` 在 2px–8px 模糊半徑間循環）
- 導覽結構：Logo → 產品下拉選單 → 使用案例下拉選單 → 資源下拉選單 → GitHub stars 徽章 → 文件 CTA
- 連結文字：14–16px Inter 的雪白 (`#f2f2f2`)，字重 500
- 懸停：連結過渡到綠色變體 (`#00c182` 或 `#00ffaa`)
- GitHub 徽章：直接整合在導覽列中的社交證明元素
- 行動版：收摺為漢堡選單，單欄垂直佈局

### 圖片處理 (Image Treatment)
- 以深色主題的產品截圖和架構圖為主
- 程式碼塊被視為主要的視覺內容 —— 使用 SFMono-Regular 進行語法高亮
- Agent 工作流視覺化以帶有綠色連接線的互動式節點圖呈現
- 英雄區段背後出現裝飾性的點狀圖案背景
- 在卡片容器內滿版顯示，遵循 8px 圓角半徑

### 獨特元件 (Distinctive Components)

**npm 安裝指令塊 (npm Install Command Block)**
- 一個醒目的程式碼片段 (`npm create voltagent-app@latest`)，樣式設計為可複製指令。
- 碳纖表面上的 SFMono-Regular，配有複製到剪貼簿按鈕。
- 作為主要的 CTA —— 符合「先安裝，後閱讀」的開發者心理。

**公司 Logo 跑馬燈 (Company Logo Marquee)**
- 開發者/公司 Logo 的水平滾動條。
- 無限動畫（`scrollLeft`/`scrollRight`，時長 25–80s）。
- 懸停時暫停，並為減少動態偏好的用戶暫停。
- 展示生態系統的採用情況而不會使佈局顯得雜亂。

**特色區段卡片 (Feature Section Cards)**
- 將程式碼範例與描述性文字結合的大型卡片。
- 左：帶有語法高亮的程式碼片段；右：特色描述。
- 強調/活動特色上帶有綠色強調邊框 (`2px solid #00d992`)。
- 內部內距：大方 (估計為 24–32px)。

**Agent 流程圖 (Agent Flow Diagrams)**
- 顯示 Agent 協作的互動式節點圖視覺化。
- 連接線使用 VoltAgent 綠色變體。
- 節點樣式設計為暖炭灰邊框系統內的小型卡片。

**社群 / GitHub 區段 (Community / GitHub Section)**
- 以大型 GitHub 圖標作為視覺錨點。
- 顯眼地顯示 Star 數量和貢獻者指標。
- 頁尾包含溫暖的社交證明連結：Discord, X, Reddit, LinkedIn, YouTube。

## 5. 版面配置原則

### 間距系統 (Spacing System)
- 基礎單位：8px
- 縮放系列：2px, 4px, 5px, 6px, 6.4px, 8px, 12px, 16px, 20px, 24px, 28px, 32px, 40px, 48px, 64px
- 按鈕內距：12px 16px（標準）、20px（容器按鈕）
- 卡片內部邊距：約 24–32px
- 區段垂直間距：大方（主要區段之間估計為 64–96px）
- 元件間隙：相鄰卡片/元素之間為 16–24px

### 格線與容器 (Grid & Container)
- 最大容器寬度：約 1280–1440px，居中
- 英雄區段：居中的單欄佈局，留有最大的呼吸空間
- 特色區段：交替的不對稱佈局（程式碼在左 / 文字在右，反之亦然）
- Logo 跑馬燈：全寬水平滾動，突破容器限制
- 卡片格線：2–3 欄用於特色展示
- 整合格線：響應式多欄，用於合作夥伴/整合圖標

### 留白哲學 (Whitespace Philosophy)
- **區段之間如電影般的呼吸空間**：巨大的垂直間隙創造出一種「滾動瀏覽章節」的體驗——每個區段都感覺像是一個新場景。
- **元件內部密集**：卡片和程式碼塊內部緊湊，行高壓縮且內距受控。資訊是集中的，而非分散的。
- **由邊框定義的區隔**：VoltAgent 不僅依賴留白，還使用暖炭灰邊框系統 (`#3d3a39`) 來描繪內容區域。邊框就是留白訊號。
- **英雄優先的階層**：頁面頂部佔據最多的空間——「AI Agent 工程平台」標題和 npm 指令在第一個內容區段出現前獲得最大的垂直跑道。

### 圓角縮放 (Border Radius Scale)
- 幾乎直角 (4px)：小型內聯元素、SVG 容器、程式碼跨度 —— 最銳利的處理，傳達技術精確度。
- 微妙圓角 (6px)：按鈕、連結、剪貼簿動作 —— 互動元素的常用圓角。
- 程式碼專用 (6.4px)：程式碼塊、`pre` 元素、剪貼簿複製目標 —— 與標準 6px 的刻意微小區別。
- 舒適圓角 (8px)：內容卡片、特色容器、強調按鈕 —— 標準包圍圓角。
- 藥丸狀 (9999px)：標籤、徽章、狀態指示燈、藥丸狀導覽元素 —— 小型類別標籤的最圓滑處理。

## 6. 深度與高度 (Depth & Elevation)

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 扁平 (Level 0) | 無陰影，無邊框 | 頁面背景 (`#050507`)、內聯文字 |
| 封閉 (Level 1) | `1px solid #3d3a39`, 無陰影 | 標準卡片、導覽列、程式碼塊 |
| 強調 (Level 2) | `3px solid #3d3a39`, 無陰影 | 大型互動按鈕、強調容器 |
| 強調色 (Level 3) | `2px solid #00d992`, 無陰影 | 活動/強調特色卡片、選取狀態 |
| 氛圍發光 (Level 4) | `rgba(92, 88, 85, 0.2) 0px 0px 15px` | 提升的卡片、懸停狀態、柔和的氣氛提升 |
| 戲劇懸浮 (Level 5) | `rgba(0, 0, 0, 0.7) 0px 20px 60px` + `rgba(148, 163, 184, 0.1) 1px inset` | 英雄特色展示、模態視窗、最高高度內容 |

**陰影哲學**：VoltAgent 主要透過**邊框重量和顏色**而非陰影來傳達深度。標準的 `1px solid #3d3a39` 邊框就是高度 —— 增加 `3px` 邊框重量或切換到綠色 (`#00d992`) 比增加陰影更能傳達重要性。當陰影出現時，它們要么是暖色且擴散的 (Level 4)，要么是電影般戲劇性的 (Level 5) —— 絕不中庸或通用。

### 裝飾性深度
- **綠色訊號光 (Green Signal Glow)**：VoltAgent 閃電 Logo 帶有 `drop-shadow` 動畫，在翡翠訊號綠的 2px 到 8px 模糊半徑間循環。這是最獨特的裝飾元素——它讓 Logo 感覺「電源開啟」。
- **暖炭灰包圍線 (Warm Charcoal Containment Lines)**：`#3d3a39` 邊框的暖色調在冷色調的黑色背景下創造出微妙的視覺溫度，彷彿卡片內部隱約散發熱量。
- **虛線工作流線 (Dashed Workflow Lines)**：`1px dashed rgba(79, 93, 117, 0.4)` 為架構圖創造出類似藍圖的美學，在視覺上與實線內容邊框區隔。

## 7. 規範事項 (Do's and Don'ts)

### 應該 (Do)
- 使用深淵黑 (`#050507`) 作為登陸頁面背景，並使用碳纖表面 (`#101010`) 作為所有封閉元素 —— 雙色階深色系統至關重要。
- 將翡翠訊號綠 (`#00d992`) 專門保留給高信號時刻：活動邊框、發光效果和最重要的互動強調。
- 在深色表面上使用 VoltAgent Mint (`#2fd6a1`) 作為按鈕文字 —— 它比純粹的訊號綠更易讀。
- 標題行高保持壓縮 (1.0–1.11) 並帶有負字距，以建立密集、具權威性的文字塊。
- 為邊框和次要文字使用暖灰色調 (`#3d3a39`, `#8b949e`, `#b8b3b0`) —— 暖色調防止深色主題顯得生硬。
- 將程式碼片段作為主要內容呈現 —— 它們是英雄元素，而非輔助插圖。
- 使用邊框重量 (1px → 2px → 3px) 和顏色切換 (`#3d3a39` → `#00d992`) 來傳達深度和重要性，而非依賴陰影。
- 將 system-ui 用於標題，搭配 Inter 用於內文 —— 原生字體的連度/權威性與幾何無襯線體的精確度相結合。
- 為所有程式碼內容使用 SFMono-Regular —— 這是開發者信賴的信號。
- 在所有文字中應用 `"calt"` 和 `"rlig"` OpenType 特性。

### 不應該 (Don't)
- 不要使用明亮或淺色背景作為主要表面 —— 整個身份都建立在近乎純黑之上。
- 不要引入暖色調（橘、紅、黃）作為裝飾強調 —— 色調嚴格限於綠色 + 黑色上的暖中性色。暖色調僅保留給語義狀態（警告、錯誤）。
- 不要在大型表面或作為背景填充使用翡翠訊號綠 (`#00d992`) —— 它是一個強調色，絕非表面色。
- 不要將標題行高增加到 1.33 以上 —— 壓縮的密度是工程平台身份的核心。
- 不要大方地使用重陰影 —— 深度來自邊框處理而非 box-shadow。陰影僅保留給 Level 4–5 的提升。
- 不要使用純白 (`#ffffff`) 作為預設內文 —— 雪白 (`#f2f2f2`) 是標準。純白保留給最高強調的標題和按鈕文字。
- 不要混入襯線或裝飾性字體 —— 整個系統是幾何無襯線 + 等寬字體。
- 在內容卡片上不要使用大於 8px 的圓角 —— 9999px (藥丸狀) 僅用於小型標籤和徽章。
- 不要跳過暖灰色邊框系統 —— 沒有 `#3d3a39` 邊框的卡片會失去包圍感，在黑色畫布上顯得漂浮不定。
- 不要過度動畫 —— 動畫應緩慢且微妙（跑馬燈時長 25–100s，發光脈衝柔和）。快速運動與「工程精確度」的氛圍相矛盾。

## 8. 響應式行為

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變更 |
|------|-------|-------------|
| 小行動版 | <420px | 最小佈局，所有內容堆疊，英雄文字縮減至 ~24px |
| 行動版 | 420–767px | 單欄佈局，漢堡導覽，全寬卡片，英雄文字 ~36px |
| 平板 | 768–1024px | 開始出現 2 欄格線，縮減導覽列，中等英雄文字 |
| 桌機 | 1025–1440px | 完整多欄佈局，帶下拉選單的擴展導覽，大英雄文字 (60px) |
| 大桌機 | >1440px | 最大寬度容器居中 (約 1280–1440px)，大方的水平邊距 |

### 觸控目標 (Touch Targets)
- 按鈕使用舒適的內距（最小 12px 16px），確保足夠的觸控區域。
- 導覽連結間距足夠，方便大拇指導覽。
- 互動式卡片表面足夠大，可作為完整的觸控目標。
- 最小推薦觸控目標：44x44px。

### 收摺策略 (Collapsing Strategy)
- **導覽列**：帶有下拉選單的完整水平導覽在行動版收摺為漢堡選單。
- **特色格線**：3 欄 → 2 欄 → 單欄垂直堆疊。
- **英雄文字**：60px → 36px → 24px 階梯式縮放，保持壓縮比例。
- **Logo 跑馬燈**：調整滾動速度和項目大小；保持無限循環。
- **程式碼塊**：在較小視窗中水平滾動而非換行 —— 保持程式碼易讀性。
- **區段內距**：按比例縮小，但保持章節之間大方的垂直節奏。
- **卡片**：在行動版垂直堆疊，採用全寬處理並保持內部內距。

### 圖片行為
- 深色主題的截圖和圖表在容器內等比例縮放。
- Agent 流程圖在窄視窗中簡化或水平滾動。
- 點狀裝飾背景隨視窗縮放。
- 斷點之間無明顯的藝術指導變化 —— 相同的裁剪，等比例縮放。
- 摺疊線以下圖片延遲加載 (Lazy loading)。

### 迭代指南
優化使用此設計系統生成的現有畫面時：
1. 一次專注於一個元件。
2. 參考特定的顏色名稱和十六進位代碼 —— 使用「暖羊皮紙色 (#b8b3b0)」而非「讓它亮一點」。
3. 使用邊框處理來傳達高度：強調時「將邊框更改為 2px 實心翡翠訊號綠 (#00d992)」。
4. 在描述測量值的同時描述所需的「感覺」 —— 「36px 的壓縮且具權威性的標題，行高 1.11 且字元間距 -0.9px」。
5. 對於發光效果，指定「翡翠訊號綠 (#00d992) 作為 drop-shadow，模糊半徑 2–8px」。
6. 務必指定字體 —— system-ui 用於標題，Inter 用於內文/UI，SFMono-Regular 用於程式碼。
7. 保持動畫緩慢且微妙 —— 跑馬燈捲動時長為 25–80s，發光脈衝柔和。
