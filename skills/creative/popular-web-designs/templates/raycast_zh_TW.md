## 1. 視覺主題與氛圍

Raycast 的行銷網站給人的感覺就像是精密儀器的深色內部——一個用黑曜石雕刻而成的瑞士手錶盒。背景不僅僅是深色，它是一種近乎黑色的藍色調 (`#07080a`)，營造出一種置身於 macOS 原生應用程式而非網站的感覺。每個表面、每個邊框、每個陰影都經過精確校準，以喚起高效能桌面工具的感覺：快速、極簡、值得信賴。

其標誌性手法是借鑑自 macOS 視窗框架的分層陰影系統：帶有內陰影 (inset) 高亮的多層方塊陰影，模擬物理深度，彷彿卡片和按鈕是深色桌面上的實際壓制或凸起玻璃元素。結合 Raycast 紅 (`#FF6363`) —— 幾乎專門用於 Hero 區塊標誌性的對角條紋圖案 —— 這種調色盤打造出一個被解讀為「具有個性的強大工具」的品牌。紅色並不主導視覺，它起到了點睛的作用。

Inter 字體被應用於所有地方 —— 標題、內文、按鈕、說明文字 —— 並搭配廣泛的 OpenType 特性 (`calt`, `kern`, `liga`, `ss03`)，創造出一致、易讀的字體語調。正向的字距 (0.2px–0.4px) 在深色 UI 中並不尋常，這賦予了文字一種輕盈、透氣的質感，抵消了密集、深色表面的沉重感。Geist Mono 用於程式碼元素，強化了開發者工具的身份。

**關鍵特徵：**
- 近乎黑色的藍色調背景 (`#07080a`) —— 非純黑色，帶有微妙的藍色偏移
- 帶有多層內陰影高亮的 macOS 原生陰影系統，模擬物理深度
- Raycast 紅 (`#FF6363`) 作為點睛色 —— 用於 Hero 條紋，而非到處可見
- 搭配正向字距 (0.2px) 的 Inter 字體，提供輕盈、易讀的深色模式體驗
- Radix UI 組件原型驅動的互動層
- 深色表面上使用微妙的 rgba 白色邊框 (0.06–0.1 不透明度) 進行包含
- 搭配漸層鍵帽和重陰影的鍵盤快捷鍵樣式

## 2. 調色盤與角色

### 主要顏色 (Primary)
- **近黑藍色** (`#07080a`)：主要頁面背景 —— 帶有微妙藍冷底色的基礎色
- **純白色** (`#ffffff`)：主要標題文字、高強調元素
- **Raycast 紅** (`#FF6363` / `hsl(0, 100%, 69%)`)：品牌強調色 —— Hero 條紋、危險狀態、關鍵高亮

### 次要與強調色 (Secondary & Accent)
- **Raycast 藍** (`hsl(202, 100%, 67%)` / 約 `#55b3ff`)：互動強調色 —— 連結、聚焦狀態、選定項目
- **Raycast 綠** (`hsl(151, 59%, 59%)` / 約 `#5fc992`)：成功狀態、正面指示
- **Raycast 黃** (`hsl(43, 100%, 60%)` / 約 `#ffbc33`)：警告強調色、高亮
- **藍色透明** (`hsla(202, 100%, 67%, 0.15)`)：用於互動表面的藍色調覆蓋層
- **紅色透明** (`hsla(0, 100%, 69%, 0.15)`)：用於危險/錯誤表面的紅色調覆蓋層

### 表面與背景 (Surface & Background)
- **深層背景** (`#07080a`)：頁面畫布，最深處的表面
- **表面 100** (`#101111`)：提升的表面，卡片背景
- **鍵盤起始色** (`#121212`)：鍵盤按鍵漸層起始
- **鍵盤結束色** (`#0d0d0d`)：鍵盤按鍵漸層結束
- **卡片表面** (`#1b1c1e`)：徽章背景、標籤填充、提升的容器
- **按鈕前景** (`#18191a`)：淺色背景上按鈕文字使用的深色表面

### 中性色與文字 (Neutrals & Text)
- **近白色** (`#f9f9f9` / `hsl(240, 11%, 96%)`)：主要內文文字、高強調內容
- **淺灰色** (`#cecece` / `#cdcdce`)：次要內文文字、描述
- **銀色** (`#c0c0c0`)：三級文字、淡化標籤
- **中灰色** (`#9c9c9d`)：連結預設顏色、次要導覽
- **暗灰色** (`#6a6b6c`)：停用文字、低強調標籤
- **深灰色** (`#434345`)：淡化邊框、非活動導覽連結
- **邊框色** (`hsl(195, 5%, 15%)` / 約 `#252829`)：卡片和分割線的標準邊框顏色
- **深色邊框** (`#2f3031`)：分隔線、表格邊框

### 語義化與強調色 (Semantic & Accent)
- **錯誤紅** (`hsl(0, 100%, 69%)`)：錯誤狀態、破壞性動作
- **成功綠** (`hsl(151, 59%, 59%)`)：成功確認、正面狀態
- **警告黃** (`hsl(43, 100%, 60%)`)：警告、需要注意的狀態
- **資訊藍** (`hsl(202, 100%, 67%)`)：資訊高亮、連結

### 漸層系統 (Gradient System)
- **鍵盤按鍵漸層**：從 `#121212` (頂部) 到 `#0d0d0d` (底部) 的線性漸層 —— 模擬物理按鍵深度
- **暖光暈**：`rgba(215, 201, 175, 0.05)` 徑向擴散 —— 特色元素背後微妙的暖色環境光暈

## 3. 字體規則

### 字體系列 (Font Family)
- **主要字體**：`Inter` —— 人文主義無襯線字體，到處使用。備用字體：`Inter Fallback`, 系統無襯線字體
- **系統字體**：`SF Pro Text` —— 針對特定的 macOS 原生 UI 元素使用 Apple 系統字體。備用字體：`SF Pro Icons`, `Inter`, `Inter Fallback`
- **等寬字體**：`GeistMono` —— 針對程式碼元素使用 Vercel 的等寬字體。備用字體：`ui-monospace`, `SFMono-Regular`, `Roboto Mono`, `Menlo`, `Monaco`
- **OpenType 特性**：全域啟用 `calt`, `kern`, `liga`, `ss03`；展示文字上啟用 `ss02`, `ss08`；Hero 標題上停用 `liga` (`"liga" 0`)

### 原則
- **深色背景上的正向字距**：與大多數使用緊湊或中性字距的深色 UI 不同，Raycast 應用了 +0.2px 到 +0.4px 的字距 —— 營造出一種輕盈、易讀的感覺，補償了深色背景的沉重感。
- **以字重 500 作為基準**：大多數內文文字使用中等字重 (500) 而非正規字重 (400) —— 微弱的額外厚度提高了深色表面上的易讀性。
- **展示文字的克制**：Hero 文字設定為 64px/600，自信但不誇張 —— Raycast 避開了華麗的排版，轉而追求功能性的優雅。
- **無處不在的 OpenType**：全域啟用 Inter 的 `ss03`（樣式集 3），賦予字體更具幾何感、類似工具的特質。

## 4. 元件樣式

### 按鈕 (Buttons)
- **主要藥丸形按鈕**：透明背景，白色文字，藥丸形狀 (86px 圓角)，多層內陰影 (`rgba(255, 255, 255, 0.1) 0px 1px 0px 0px inset`)。懸停：不透明度 0.6。
- **次要按鈕**：透明背景，白色文字，6px 圓角，`1px solid rgba(255, 255, 255, 0.1)` 邊框，微妙的投影 (`rgba(0, 0, 0, 0.03) 0px 7px 3px`)。懸停：不透明度 0.6。
- **幽靈按鈕 (Ghost Button)**：無背景或邊框，灰色文字 (`#6a6b6c`)，86px 圓角，同樣的內陰影。懸停：不透明度 0.6，文字變亮為白色。
- **行動呼籲 (下載)**：半透明白色背景 (`hsla(0, 0%, 100%, 0.815)`)，深色文字 (`#18191a`)，藥丸形狀。懸停：全白背景 (`hsl(0, 0%, 100%)`)。
- **轉場效果**：所有按鈕在懸停時均使用不透明度轉場，而非背景顏色切換 —— 這是 Raycast 標誌性的互動模式。

### 卡片與容器 (Cards & Containers)
- **標準卡片**：`#101111` 表面，`1px solid rgba(255, 255, 255, 0.06)` 邊框，12px–16px 圓角半徑。
- **提升型卡片**：環形陰影，外部 `rgb(27, 28, 30) 0px 0px 0px 1px` + 內部 `rgb(7, 8, 10) 0px 0px 0px 1px inset` —— 創造出雙環包含效果。
- **特色卡片**：16px–20px 圓角半徑，Hero 元素背後有微妙的暖光暈 (`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px`)。
- **懸停**：透過邊框不透明度增加或微妙的陰影增強來輕微使卡片變亮。

### 輸入框與表單 (Inputs & Forms)
- 深色輸入欄位使用 `#07080a` 背景，`1px solid rgba(255, 255, 255, 0.08)` 邊框，8px 圓角半徑。
- 聚焦狀態：邊框變亮，出現藍色光暈 (`hsla(202, 100%, 67%, 0.15)`) 環。
- 文字：`#f9f9f9` 輸入顏色，`#6a6b6c` 佔位符。
- 標籤：`#9c9c9d`，14px 字重 500。

### 導覽列 (Navigation)
- **頂部導覽**：深色背景與頁面融合，白色文字連結為 16px 字重 500。
- **導覽連結**：灰色文字 (`#9c9c9d`) → 懸停時變為白色，懸停時有底線裝飾。
- **行動呼籲按鈕**：導覽列末端的半透明白色藥丸形按鈕。
- **行動裝置**：收縮為漢堡選單，維持深色主題。
- **固定 (Sticky)**：導覽列固定在頂部，帶有微妙的邊框分隔線。

### 圖片處理
- **產品截圖**：macOS 視窗框架風格 —— 圓角 (12px)，模擬懸浮視窗的深層陰影。
- **全寬區塊**：深色截圖與深色背景無縫融合。
- **Hero 插圖**：Raycast 紅的對角條紋圖案 —— 抽象、幾何、定義品牌。
- **App UI 嵌入**：展示實際的 Raycast 命令列和擴充功能 —— 產品即內容。

### 鍵盤快捷鍵按鍵
- **鍵帽樣式**：漸層背景 (`#121212` → `#0d0d0d`)，重型多層陰影 (`rgba(0, 0, 0, 0.4) 0px 1.5px 0.5px 2.5px` + 內陰影)，營造出真實的物理按鍵外觀。
- 圓角半徑：個別按鍵為 4px–6px。

### 徽章與標籤 (Badges & Tags)
- **中性徽章**：`#1b1c1e` 背景，白色文字，6px 圓角，14px 字重 500 字體，`0px 6px` 內邊距。
- 緊湊、藥丸狀的分類處理方式。

## 5. 佈局原則

### 間隔系統 (Spacing System)
- **基礎單位**：8px
- **比例**：1px, 2px, 3px, 4px, 8px, 10px, 12px, 16px, 20px, 24px, 32px, 40px
- **區塊內邊距**：主要區塊之間 80px–120px 的垂直間距。
- **卡片內邊距**：16px–32px 的內部間距。
- **組件間距**：相關元素之間 8px–16px。

### 網格與容器 (Grid & Container)
- **最大寬度**：約 1200px 容器 (斷點於 1204px)，居中。
- **欄位模式**：單欄 Hero，2-3 欄功能網格，全寬展示區塊。
- **App 展示**：在居中的視窗框架中呈現產品 UI。

### 留白哲學
- **戲劇性的負空間**：區塊在廣大的深色虛空中漂浮，在功能之間創造出電影般的節奏。
- **產品密集，行銷稀疏**：產品 UI 截圖資訊密集，但周圍的行銷文案使用極簡文字和寬裕間距。
- **垂直律動**：區塊內元素之間保持一致的 24px–32px 間隙。

### 圓角半徑比例 (Border Radius Scale)
- **2px–3px**：微型元素、程式碼跨度、微小指示器。
- **4px–5px**：鍵盤按鍵、小型互動元素。
- **6px**：按鈕、徽章、標籤 —— 最常用的半徑。
- **8px**：輸入欄位、行內組件。
- **9px–11px**：圖片、中型容器。
- **12px**：標準卡片、產品截圖。
- **16px**：大號卡片、功能區塊。
- **20px**：Hero 卡片、醒目的容器。
- **86px+**：藥丸形按鈕、導覽行動呼籲 —— 完整的藥丸形狀。

## 6. 深度與層次 (Depth & Elevation)

| 等級 | 處理方式 | 用途 |
|-------|-----------|-----|
| 等級 0 (虛空) | 無陰影, `#07080a` 表面 | 頁面背景 |
| 等級 1 (微妙) | `rgba(0, 0, 0, 0.28) 0px 1.189px 2.377px` | 極小提升, 行內元素 |
| 等級 2 (環形) | 外部 `rgb(27, 28, 30) 0px 0px 0px 1px` + 內部 `rgb(7, 8, 10) 0px 0px 0px 1px inset` | 卡片包含, 雙環技術 |
| 等級 3 (按鈕) | `rgba(255, 255, 255, 0.05) 0px 1px 0px 0px inset` + `rgba(255, 255, 255, 0.25) 0px 0px 0px 1px` + `rgba(0, 0, 0, 0.2) 0px -1px 0px 0px inset` | macOS 原生按鈕點擊 —— 頂部白色高亮, 底部深色內陰影 |
| 等級 4 (按鍵) | 帶有內陰影點擊效果的 5 層陰影堆疊 | 鍵盤快捷鍵鍵帽 —— 物理 3D 外觀 |
| 等級 5 (懸浮) | `rgba(0, 0, 0, 0.5) 0px 0px 0px 2px` + `rgba(255, 255, 255, 0.19) 0px 0px 14px` + 內陰影 | 命令列、懸浮面板 —— 帶有光暈的重深度感 |

### 陰影哲學
Raycast 的陰影系統是網路上最接近 macOS 原生的。多層陰影結合了：
- 用於包含的**外環**（取代傳統邊框）
- **頂部內陰影高亮** (`rgba(255, 255, 255, 0.05–0.25)`) 模擬來自上方的光源
- **底部內陰影深色** (`rgba(0, 0, 0, 0.2)`) 模擬下方的陰影
- 效果是物理性的：元素感覺像是玻璃或拉絲金屬，而不是平面矩形。

### 裝飾性深度
- **暖光暈**：特色元素背後的 `rgba(215, 201, 175, 0.05) 0px 0px 20px 5px` —— 在寒冷的深色畫布上散發微妙的暖色氣息。
- **藍色資訊光暈**：用於互動狀態強調的 `rgba(0, 153, 255, 0.15)`。
- **紅色危險光暈**：用於錯誤/破壞性狀態強調的 `rgba(255, 99, 99, 0.15)`。

## 7. 應做與不應做

### 應做 (Do)
- 使用 `#07080a`（非純黑色）作為背景 —— 藍冷色調對 Raycast 的感覺至關重要。
- 內文文字應用正向字距 (+0.2px) —— 這與大多數深色 UI 刻意不同。
- 對互動元素使用帶有內陰影高亮的多層陰影 —— macOS 原生深度感是標誌特徵。
- 保持 Raycast 紅 (`#FF6363`) 作為點睛之筆，而非到處可見 —— 將其保留用於 Hero 時刻和錯誤狀態。
- 使用 `rgba(255, 255, 255, 0.06)` 邊框進行卡片包含 —— 幾乎看不見但結構上必不可少。
- 應用字重 500 作為內文文字基準 —— 中等字重可提高深色模式易讀性。
- 主要行動呼籲 (CTA) 使用藥丸形狀 (86px+ 圓角)，次要動作使用矩形形狀 (6px–8px)。
- 在所有 Inter 文字上啟用 OpenType 特性 `calt`, `kern`, `liga`, `ss03`。
- 按鈕互動使用不透明度轉場（懸停：不透明度 0.6），而非變換背景顏色。

### 不應做 (Don't)
- 不要使用純黑色 (`#000000`) 作為背景 —— 藍色調將 Raycast 與通用的深色主題區分開來。
- 不要對內文文字應用負字距 —— Raycast 刻意使用正向字距以利閱讀。
- 不要將 Raycast 藍作為所有內容的主要強調色 —— 藍色用於互動/資訊，紅色才是品牌顏色。
- 不要建立單層平面陰影 —— 多層內陰影系統是 macOS 原生美學的核心。
- 不要在可以使用 500 時對內文文字使用正規字重 (400) —— 額外的字重可防止深色模式文字顯得過細。
- 不要混用暖色和冷色邊框 —— 堅持使用冷灰色 (`hsl(195, 5%, 15%)`) 邊框調色盤。
- 不要應用沒有內陰影伴隨的重度投影 —— 陰影總是成對出現（外部 + 內部）。
- 不要使用裝飾元素、漸層或彩色背景 —— 深色虛空是舞台，內容是表演者。

## 8. 響應式行為

### 斷點 (Breakpoints)
| 名稱 | 寬度 | 關鍵變化 |
|------|-------|-------------|
| 行動裝置 (Mobile) | <600px | 單欄, 卡片堆疊, 漢堡選單, Hero 文字縮小至 ~40px |
| 小型平板 | 600px–768px | 開始出現 2 欄網格, 導覽列部分可見 |
| 平板 | 768px–1024px | 2–3 欄功能, 導覽展開, 截圖縮放 |
| 桌面 | 1024px–1200px | 完整佈局, 所有導覽連結可見, 64px Hero 展示 |
| 大型桌面 | >1200px | 最大寬度容器居中, 寬裕的側邊距 |

### 觸控目標
- 藥丸形按鈕：86px 圓角搭配 20px 內邊距 —— 遠高於 44px 最小值。
- 次要按鈕：至少 8px 內邊距，但邊框提供了視覺上的目標擴張。
- 導覽連結：16px 文字搭配周圍內邊距，確保觸控目標可達。

### 收縮策略
- **導覽列**：完整水平導覽 → 行動裝置上的漢堡選單搭配側拉選單。
- **Hero**：展示尺寸在斷點間由 64px → 48px → 36px。
- **功能網格**：3 欄 → 2 欄 → 單欄堆疊。
- **產品截圖**：在容器內縮放，維持 macOS 視窗框架比例。
- **鍵盤快捷鍵顯示**：在鍵盤快捷鍵無關緊要的行動裝置上簡化或隱藏。

### 圖片行為
- 產品截圖在固定比例容器內響應式縮放。
- Hero 對角條紋圖案按比例縮放。
- 在所有尺寸下均維持 macOS 視窗框架圓角。
- 無延遲載入產生的異常現象 —— 圖片對產品敘事至關重要。

## 9. Agent 提示指南

### 顏色快速參考
- 主要背景：近黑藍色 (`#07080a`)
- 主要文字：近白色 (`#f9f9f9`)
- 品牌強調色：Raycast 紅 (`#FF6363`)
- 互動藍色：Raycast 藍 (`hsl(202, 100%, 67%)` / 約 `#55b3ff`)
- 次要文字：中灰色 (`#9c9c9d`)
- 卡片表面：表面 100 (`#101111`)
- 邊框色：深色邊框 (`hsl(195, 5%, 15%)` / 約 `#252829`)

### 元件提示範例
- 「在 #07080a 背景上建立一個 Hero 區塊，標題為 64px Inter（字重 600、行高 1.1）、近白色文字 (#f9f9f9)，以及一個半透明白色藥丸形 CTA 按鈕 (hsla(0,0%,100%,0.815), 86px 圓角, 深色文字 #18191a)」
- 「設計一個功能卡片，背景為 #101111，1px solid rgba(255,255,255,0.06) 邊框，16px 圓角半徑，雙環陰影（外部 rgb(27,28,30) 0px 0px 0px 1px），22px Inter 標題，以及 #9c9c9d 內文文字」
- 「在深色背景 (#07080a) 上建構一個導覽列，Inter 連結為 16px 字重 500、顏色為 #9c9c9d，懸停時變白，右端有一個半透明白色藥丸形按鈕」
- 「建立一個鍵盤快捷鍵顯示，鍵帽使用漸層背景 (#121212→#0d0d0d)、具物理深度感的 5 層陰影、4px 圓角、Inter 12px 字重 600 的文字」
- 「設計一個警示卡片，表面為 #101111，Raycast 紅 (#FF6363) 左邊框強調，半透明紅色光暈 (hsla(0,100%,69%,0.15))，白色標題，以及 #cecece 描述文字」

### 迭代指南
優化使用此設計系統產生的現有頁面時：
1. 檢查背景是否為 `#07080a` 而非純黑色 —— 藍色調至關重要。
2. 驗證內文文字的字距是否為正向 (+0.2px) —— 負字距會破壞 Raycast 的美感。
3. 確保陰影同時具備外部和內層 —— 單層陰影看起來會顯得扁平且錯誤。
4. 確認 Inter 已啟用 OpenType 特性 `calt`, `kern`, `liga`, `ss03`。
5. 測試懸停狀態是否使用不透明度轉場 (0.6) 而非顏色更換 —— 這是核心互動模式。
