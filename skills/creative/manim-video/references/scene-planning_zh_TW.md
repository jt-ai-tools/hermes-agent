# 場景規劃 (Scene Planning) 參考指南

## 敘事架構 (Narrative Arc Structures)

### 探索型架構 (Discovery Arc，最常見)
1. 鉤子 (Hook) —— 提出一個問題或令人驚訝的結果
2. 直覺 (Intuition) —— 建立視覺上的理解
3. 正式化 (Formalize) —— 引入方程式/演算法
4. 揭曉 (Reveal) —— 「原來如此！」的時刻 (Aha moment)
5. 擴展 (Extend) —— 影響或推廣

### 問題解決型架構 (Problem-Solution Arc)
1. 問題 (Problem) —— 哪裡出了問題
2. 失敗嘗試 (Failed attempt) —— 明顯的方法失效了
3. 關鍵洞察 (Key insight) —— 真正有效的主意
4. 解決方案 (Solution) —— 實作它
5. 結果 (Result) —— 展示改進之處

### 比較型架構 (Comparison Arc)
1. 設定 (Setup) —— 介紹兩種方法
2. 方法 A —— 運作方式
3. 方法 B —— 運作方式
4. 對照 (Contrast) —— 差異點
5. 結論 (Verdict) —— 哪一個更好

### 堆疊型架構 (Build-Up Arc，適用於架構/系統)
1. 組件 A —— 第一個零件
2. 組件 B —— 第二個零件
3. 連接 (Connection) —— 它們如何互動
4. 規模 (Scale) —— 增加更多零件
5. 全景 (Full picture) —— 縮小畫面看整體

## 場景轉換 (Scene Transitions)

### 乾淨切換 (Clean Break，預設值)
```python
self.play(FadeOut(Group(*self.mobjects)), run_time=0.5)
self.wait(0.3)
```

### 承接轉換 (Carry-Forward)
保留一個元素，淡出其餘部分。下一個場景開始時該元素仍留在螢幕上。

### 變換橋接 (Transform Bridge)
場景以一個形狀結束，下一個場景透過變換該形狀開始。

## 場景檢查清單

- [ ] 設定背景顏色
- [ ] 每個動畫都有子標題
- [ ] 每次揭曉內容後使用 `self.wait()`
- [ ] 邊緣定位時文字間距 (buff) >= 0.5
- [ ] 文字無重疊
- [ ] 使用顏色常數（而非硬編碼）
- [ ] 套用不透明度分層
- [ ] 場景結束時乾淨退出
- [ ] 同時可見的元素不超過 5-6 個

## 時長估算

| 內容 | 時長 |
|---------|----------|
| 標題卡 | 3-5 秒 |
| 概念介紹 | 10-20 秒 |
| 方程式揭曉 | 15-25 秒 |
| 演算法步驟 | 5-10 秒 |
| 數據比較 | 10-15 秒 |
| 「原來如此！」時刻 | 15-30 秒 |
| 結論 | 5-10 秒 |
