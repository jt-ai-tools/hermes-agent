# 生產品質檢查清單 (Production Quality Checklist)

確保動畫輸出達到發佈標準的規範與檢查項。

## 編碼前檢查清單

在編寫任何 Manim 程式碼之前：

- [ ] 完成旁白腳本，並標記視覺節奏。
- [ ] 列出場景清單，並說明每個場景的目的、時長與佈局。
- [ ] 定義顏色調色盤，並分配語義（例如：`PRIMARY` = 主要概念）。
- [ ] 設定 `MONO = "Menlo"` 作為字體常數。
- [ ] 決定目標解析度與長寬比。

## 文字品質 (Text Quality)

### 防止重疊

```python
# 規則：邊緣文字的 buff 應 >= 0.5
label.to_edge(DOWN, buff=0.5)     # 良好 (GOOD)
label.to_edge(DOWN, buff=0.3)     # 不良 (BAD) —— 可能會被裁切

# 規則：在相同位置新增新內容前，先淡出舊內容
self.play(ReplacementTransform(note1, note2))  # 良好
self.play(Write(note2))                          # 不良 —— 會與 note1 重疊

# 規則：在密集場景中縮小字體
# 當同時有超過 4 個文字元素可見時，使用 font_size=20 而非 28
```

### 寬度限制

長字串可能會超出框架：

```python
# 規則：為可能較長的文字設定最大寬度
text = Text("這是一段可能很長的描述文字", font_size=22, font=MONO)
if text.width > config.frame_width - 1.0:
    text.set_width(config.frame_width - 1.0)
```

### 字體一致性

```python
# 規則：定義一次 MONO，隨處使用
MONO = "Menlo"

# 錯誤示範：混用多種字體
Text("標題", font="Helvetica")
Text("標籤", font="Arial")
Text("程式碼", font="Courier")

# 正確示範：統一樣式
Text("標題", font=MONO, weight=BOLD, font_size=48)
Text("標籤", font=MONO, font_size=20)
Text("程式碼", font=MONO, font_size=18)
```

## 空間佈局 (Spatial Layout)

### 座標預算

可視框架約為 14.2 寬 × 8.0 高（預設 16:9）。考量必要的邊距後：

```
可用區域：x ∈ [-6.5, 6.5], y ∈ [-3.5, 3.5]
頂部標題區：y ∈ [2.5, 3.5]
底部備註區：y ∈ [-3.5, -2.5]
主要內容區：y ∈ [-2.5, 2.5], x ∈ [-6.0, 6.0]
```

### 填滿框架

空蕩蕩的場景看起來像未完成的作品。如果主要內容較小，請加入背景上下文：
- 在內容後方加上調暗的網格/軸線。
- 在頂部加上標題/副標題。
- 在底部加上來源引用。
- 使用低不透明度的裝飾性幾何圖形。

### 同時出現元素的最大數量

**硬性限制：同時可見的活動元素不超過 6 個。** 超過這個數量，觀眾將無法追蹤所有內容。如果需要展示更多：
- 將舊元素調暗至不透明度 0.3。
- 移除已完成任務的元素。
- 拆分為兩個場景。

## 動畫品質 (Animation Quality)

### 多樣性審計

檢查是否連續兩個場景使用了完全相同的：
- 動畫類型（例如：如果場景 3 全部使用 Write，場景 4 應改用 FadeIn 或 Create）。
- 顏色強調（在調色盤顏色間輪換）。
- 佈局（置中、左右、網格 —— 交替使用）。
- 節奏（例如：如果場景 2 緩慢且深思熟慮，場景 3 可以快一點）。

### 節奏曲線 (Tempo curve)

一段優秀的影片應遵循節奏曲線：

```
慢 ──→ 中 ──→ 快 (高潮) ──→ 慢 (結論)

場景 1：慢（簡介、設定）
場景 2：中（建立理解）
場景 3：中-快（核心內容、大量動畫）
場景 4：快（應用/結果的快節奏蒙太奇）
場景 5：慢（結論、關鍵要點）
```

### 轉換品質 (Transition quality)

場景之間：
- **乾淨退出**：`self.play(FadeOut(Group(*self.mobjects)), run_time=0.5)`。
- **簡短停頓**：淡出之後、下一場景第一個動畫開始之前，使用 `self.wait(0.3)`。
- **絕不硬切**：務必為場景轉換製作動畫。

## 顏色品質 (Color Quality)

### 深色背景下的調光

在白色背景上看起來鮮豔的顏色，在深色背景（#0D1117, #1C1C1C）上可能會顯得暗淡。測試您的調色盤：

```python
# 適合深色背景的顏色：
# 明亮且飽和：#58C4DD, #83C167, #FFFF00, #FF6B6B
# 不適合的顏色：#666666 (看不見), #2244AA (太暗)

# 規則：結構性元素（軸線、網格）不透明度為 0.15
# 背景上下文元素不透明度為 0.3-0.4
# 主要元素不透明度為 1.0
```

### 顏色含義一致性

一旦為顏色分配了含義，該含義應貫穿整段影片：

```python
# 如果 PRIMARY (#58C4DD) 在場景 1 代表「模型」，
# 那麼它在每個場景都應代表「模型」。
# 之後絕不要將 PRIMARY 用於不同的概念。
```

## 資料視覺化品質

### 圖表的最低要求

- 每個軸都有軸標籤。
- Y 軸範圍從 0 開始（或有明顯的斷裂指示器）。
- 長條/曲線顏色與圖例匹配。
- 在顯著的數據點上標註數字（至少包含最大值和對比點）。

### 動畫計數器

顯示數字變化時：
```python
# 良好：使用 DecimalNumber 進行平滑動畫
counter = DecimalNumber(0, font_size=48, num_decimal_places=0, font="Menlo")
self.play(counter.animate.set_value(1000), run_time=3, rate_func=rush_from)

# 不良：數值直接跳變的文字
```

## 渲染前檢查清單

在執行 `manim -qh` 之前：

- [ ] 所有場景在 `-ql` 模式下渲染無誤。
- [ ] 文字密集的場景使用 `-qm` 預覽靜止畫面（檢查字距）。
- [ ] 每個場景都設定了背景顏色 (`self.camera.background_color = BG`)。
- [ ] 每個重要動畫都有 `add_subcaption()` 或 `subcaption=`。
- [ ] 沒有小於 `font_size=18` 的文字。
- [ ] 沒有使用比例字體（請使用等寬字體）。
- [ ] 所有的 `.to_edge()` 呼叫中 `buff` 均 >= 0.5。
- [ ] 每個場景結束時都乾淨退出（全部 FadeOut）。
- [ ] 每次揭曉內容後使用 `self.wait()`。
- [ ] 使用顏色常數（場景程式碼中沒有硬編碼的十六進位字串）。
- [ ] 所有場景使用相同的品質標籤（不要混用 `-ql` 和 `-qh`）。

## 渲染後檢查清單

完成最終影片拼接後：

- [ ] 以 1 倍速觀看完整影片 —— 是否有任何地方感覺太趕？
- [ ] 是否有兩個動作同時進行而導致混亂的時刻？
- [ ] 每個文字標籤是否有足夠的時間閱讀？
- [ ] 場景間的轉換是否平滑（無黑影格、無突兀剪接）？
- [ ] 音訊是否與視覺效果同步（如果使用了旁白）？
- [ ] 「第一印象」是否良好？前 5 秒決定了觀眾是否會繼續觀看。
