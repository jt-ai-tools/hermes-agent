# 創意構思 (Creative Ideation)

透過創意限制 (Creative Constraints) 生成專案構想。限制 + 方向 = 創意。

## 運作方式

1. 從下方的庫中**選擇一個限制** —— 可以是隨機選擇，或根據使用者的領域/心情進行匹配。
2. **廣義地詮釋它** —— 一個程式開發提示可以變成硬體專案，一個藝術提示可以變成 CLI 工具。
3. **生成 3 個具體的專案構想**以滿足該限制。
4. **如果他們選中一個，就開始實作** —— 建立專案、撰寫程式碼、發布成品。

## 規則

每個提示都應盡可能廣義地被詮釋。「這是否包含 X？」→ 是。這些提示提供方向和適度的限制。若兩者皆無，則無創意可言。

## 限制庫

### 給開發者 (For Developers)

**解決自己的煩惱 (Solve your own itch)：**
構建一個你這週曾希望存在的工具。50 行以內，今天就發布。

**自動化惱人的事 (Automate the annoying thing)：**
你的工作流程中，哪一部分最枯燥乏味？寫個腳本搞定它。花兩小時解決一個每天耗費你五分鐘的問題。

**理應存在的 CLI 工具 (The CLI tool that should exist)：**
想一個你曾希望可以輸入的指令。`git undo-that-thing-i-just-did`、`docker why-is-this-broken`、`npm explain-yourself`。現在就把它做出來。

**除了膠水，別無新物 (Nothing new except glue)：**
完全利用現有的 API、程式庫和資料集製作東西。唯一原創的貢獻是你如何連接它們。

**科學怪人之週 (Frankenstein week)：**
拿一個做 X 的東西，讓它去做 Y。一個會播放音樂的 Git 儲存庫。一個會生成詩歌的 Dockerfile。一個會發送讚美訊息的 Cron 任務。

**刪減 (Subtract)：**
在崩潰之前，你能從程式碼庫中刪除多少東西？將工具縮減至最小可行功能。持續刪除，直到僅剩本質。

**高層次概念，低層次努力 (High concept, low effort)：**
一個深刻的構想，懶散地執行。概念應該是精彩的，但實作應該只需一個下午。如果花費更長時間，說明你考慮過度了。

### 給創作者與藝術家 (For Makers & Artists)

**明目張膽地複製 (Blatantly copy something)：**
挑選一個你欣賞的東西 —— 工具、藝術品、介面。從頭開始重新創作。學習的價值在於你的版本與原作之間的差距。

**一百萬個某物 (One million of something)：**
一百萬既很多，也沒那麼多。一百萬像素是一張 1MB 的照片。一百萬次 API 調用是一個稀鬆平常的週二。任何東西只要達到一百萬的規模，都會變得有趣。

**製作會消逝的東西 (Make something that dies)：**
一個每天減少一項功能的網站。一個會遺忘的聊天機器人。一個通往「無」的倒數計時。這是一場關於腐朽、終結或放手的練習。

**做大量的數學 (Do a lot of math)：**
生成幾何、著色器挑戰 (shader golf)、數學藝術、計算摺紙。是時候重新學習 arcsin 是什麼了。

### 給所有人 (For Anyone)

**文字是萬能介面 (Text is the universal interface)：**
構建一個文字是唯一介面的東西。沒有按鈕，沒有圖形，只有輸入文字和輸出文字。文字幾乎可以與任何事物進出接軌。

**從笑點開始 (Start at the punchline)：**
想一個聽起來會很有趣的句子，然後倒推將其具現化。「我教我的溫控器如何對我進行情感操縱 (gaslight)」→ 現在就把它做出來。

**充滿惡意的 UI (Hostile UI)：**
製作一個刻意讓人用起來很痛苦的東西。一個需要 47 個條件的密碼欄位。一個所有標籤都在撒謊的表單。一個會批判你下達指令的 CLI。

**重來一次 (Take two)：**
回想一個舊專案，不看原始碼，從頭開始再做一次。看看你的思維方式發生了什麼變化。

參見 [references/full-prompt-library_zh_TW.md](references/full-prompt-library_zh_TW.md) 獲取涵蓋溝通、規模、哲學、轉換等領域的 30 多個額外限制提示。

## 匹配限制與使用者

| 使用者提到 | 從此處挑選 |
|-----------|-----------|
| 「我想做點東西」(無方向) | 隨機 —— 任何限制 |
| 「我正在學習 [語言]」 | 明目張膽地複製、自動化惱人的事 |
| 「我想要點怪東西」 | 充滿惡意的 UI、科學怪人之週、從笑點開始 |
| 「我想要點有用的東西」 | 解決自己的煩惱、理應存在的 CLI、自動化惱人的事 |
| 「我想要點美的東西」 | 做大量的數學、一百萬個某物 |
| 「我職業倦怠了」 | 高層次概念低層次努力、製作會消逝的東西 |
| 「週末小專案」 | 除了膠水別無新物、從笑點開始 |
| 「我想要個挑戰」 | 一百萬個某物、刪減、重來一次 |

## 輸出格式

```
## 限制：[名稱]
> [限制內容，一句話]

### 構想

1. **[一句話推銷]**
   [2-3 句話：你想做什麼以及它為什麼有趣]
   ⏱ [週末 / 週 / 月] • 🔧 [技術棧]

2. **[一句話推銷]**
   [2-3 句話]
   ⏱ ... • 🔧 ...

3. **[一句話推銷]**
   [2-3 句話]
   ⏱ ... • 🔧 ...
```

## 範例

```
## 限制：理應存在的 CLI 工具
> 想一個你曾希望可以輸入的指令，現在就把它做出來。

### 構想

1. **`git whatsup` —— 看看在你不在時發生了什麼**
   比對你上次活動的提交與 HEAD，並總結變更內容、提交者以及合併的 PR。
   就像是從你的儲存庫發出的晨會簡報。
   ⏱ 週末 • 🔧 Python, GitPython, click

2. **`explain 503` —— 給人類看的 HTTP 狀態碼**
   透過管道傳送任何狀態碼或錯誤訊息，獲取通俗易懂的解釋以及常見原因與修復方法。
   從精選資料庫獲取資訊，而非 LLM。
   ⏱ 週末 • 🔧 Rust 或 Go, 靜態資料集

3. **`deps why <package>` —— 為什麼這東西在我的依賴樹裡**
   追蹤一個過渡依賴直到拉入它的直接依賴。
   一條指令解答「為什麼我有 47 個 lodash 副本」。
   ⏱ 週末 • 🔧 Node.js, 解析 npm/yarn lockfile
```

在使用者選中一個構想後，開始實作 —— 建立專案、撰寫程式碼、持續迭代。
