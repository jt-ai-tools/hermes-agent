## 設計建議

**不要製作無聊的投影片。** 在白色背景上放純文字條列式內容無法打動任何人。請為每張投影片考慮清單中的建議。

### 開始之前

- **挑選大膽且符合內容氛圍的色調**：色調應讓人感覺是專為「這個」主題設計的。如果你把這套顏色換到完全不同的簡報中仍然「行得通」，說明你的選擇不夠獨特。
- **主導重於均等**：應有一種顏色佔據主導地位（視覺權重 60-70%），搭配 1-2 種輔助色調和一個鮮明的強調色。絕不要給所有顏色相同的權重。
- **深淺對比**：標題和結論投影片使用深色背景，內容則使用淺色背景（「三明治」結構）。或者全程使用深色以營造高質感。
- **沿用視覺主題**：挑選一個獨特的元素並重複使用 — 例如圓角圖片框、圓圈內的圖示、粗體的單邊框。將此主題貫穿每張投影片。

### 色調參考

選擇與你的主題相匹配的顏色 — 不要預設使用通用的藍色。參考以下色調作為靈感：

### 針對每張投影片

**每張投影片都需要視覺元素** — 圖片、圖表、圖示或形狀。純文字的投影片很容易被遺忘。

**版面配置選項：**
- 雙欄（左側文字，右側插圖）
- 圖示 + 文字列（圓圈內的圖示、粗體標題、下方描述）
- 2x2 或 2x3 網格（一側為圖片，另一側為內容區塊網格）
- 半版滿版圖片（左側或右側全版）搭配內容疊加

**數據呈現：**
- 大型數據標註（60-72pt 的大數字，下方配上小標籤）
- 比較欄位（前後對照、優缺點、並列選項）
- 時間軸或流程圖（帶編號的步驟、箭頭）

**視覺修飾：**
- 區段標題旁的圓圈小圖示
- 關鍵數據或標語使用斜體強調文字

### 字體設計

**選擇有趣的字體搭配** — 不要只用 Arial。挑選一個有個性的標題字體，並搭配簡潔的正文字體。

| 標題字體 | 正文字體 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 大小 |
|---------|------|
| 投影片標題 | 36-44pt 粗體 |
| 區段標題 | 20-24pt 粗體 |
| 正文文字 | 14-16pt |
| 圖解說明 | 10-12pt 柔和色調 |

### 間距

- 邊距至少 0.5"
- 內容區塊之間保留 0.3-0.5"
- 留白 — 不要填滿每一吋空間

### 避免的事項 (常見錯誤)

- **不要重複相同的版面** — 在投影片之間變換欄位、卡片和標註樣式
- **不要將正文置中** — 段落和列表請靠左對齊；僅標題可以置中
- **不要忽略大小對比** — 標題需要 36pt+ 才能從 14-16pt 的正文中脫穎而出
- **不要預設使用藍色** — 選擇反映特定主題的顏色
- **不要隨機混合間距** — 統一選擇 0.3" 或 0.5" 的間隙並貫穿使用
- **不要只美化一張投影片而讓其餘維持單調** — 全程投入設計或全程保持簡約
- **不要製作純文字投影片** — 加入圖片、圖示、圖表或視覺元素；避免純標題加條列文字
- **不要忘記文字方塊的邊距 (Padding)** — 在對齊線條或形狀與文字邊緣時，將文字方塊的 `margin` 設為 0，或偏移形狀以補償邊距
- **不要使用低對比度的元素** — 圖示「和」文字都需要與背景有強烈對比；避免淺色背景配淺色文字，或深色背景配深色文字
- **絕對不要在標題下使用強調線** — 這是 AI 生成投影片的標誌；請改用留白或背景顏色

## QA 品質保證 (必要)

**假設存在問題。你的工作就是找出它們。**

第一次渲染幾乎絕不正確。將 QA 視為「獵蟲」行動，而非確認步驟。如果你在第一次檢查時沒發現任何問題，說明你看得不夠仔細。

### 內容 QA

```bash
python -m markitdown output.pptx
```

檢查是否有遺漏內容、錯字、順序錯誤。

**使用範本時，檢查是否有殘留的佔位文字：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 `grep` 傳回結果，請在宣佈成功前修正它們。

### 視覺 QA

**⚠️ 使用子代理人 (SUBAGENTS)** — 即使只有 2-3 張投影片。你一直盯著程式碼看，會看到你「預期」的東西而非「實際」呈現的東西。子代理人擁有全新的視角。

將投影片轉換為圖片（見 [轉換為圖片](#轉換為圖片)），然後使用此提示詞：

```
視覺檢查這些投影片。假設存在問題 — 找出它們。

尋找：
- 元素重疊（文字穿過形狀、線條穿過單字、元素堆疊）
- 文字溢出或在邊緣/方框邊界被切斷
- 裝飾線是為單行文字設計的，但標題換行成了兩行
- 來源引用或頁尾與上方內容碰撞
- 元素太近（間隙 < 0.3"）或卡片/區段幾乎接觸
- 間隙不均勻（一處有大量空白，另一處卻很擁擠）
- 投影片邊距不足 (< 0.5")
- 欄位或類似元素對齊不一致
- 低對比度文字（例如：米色背景上的淺灰色文字）
- 低對比度圖示（例如：深色背景上的深色圖示，且沒有對比圓圈）
- 文字方塊太窄導致過度換行
- 殘留的佔位內容

針對每張投影片，列出問題或疑慮，即使是微小的問題。

閱讀並分析這些圖片：
1. /path/to/slide-01.jpg (預期：[簡短描述])
2. /path/to/slide-02.jpg (預期：[簡短描述])

報告發現的所有問題，包括微小的問題。
```

### 驗證迴圈

1. 生成投影片 → 轉換為圖片 → 檢查
2. **列出發現的問題**（如果沒發現，請以更挑剔的眼光再看一次）
3. 修正問題
4. **重新驗證受影響的投影片** — 一個修正往往會產生另一個問題
5. 重複直到完整檢查不再發現新問題

**在完成至少一次「修正並驗證」週期之前，不要宣佈成功。**

## 轉換為圖片

將簡報轉換為個別的投影片圖片以便進行視覺檢查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

這會建立 `slide-01.jpg`、`slide-02.jpg` 等檔案。

若要在修正後重新渲染特定投影片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 依賴項目

- `pip install "markitdown[pptx]"` - 文字擷取
- `pip install Pillow` - 縮圖網格
- `npm install -g pptxgenjs` - 從頭開始建立
- LibreOffice (`soffice`) - PDF 轉換（透過 `scripts/office/soffice.py` 為沙箱環境自動設定）
- Poppler (`pdftoppm`) - PDF 轉圖片
