## NeurIPS 論文檢查表

### 強制性要件

所有 NeurIPS 提交的論文都必須包含一份填寫完整的論文檢查表。缺少此要件的論文將面臨**自動直接拒稿**。檢查表需附在參考文獻和補充材料之後，不計入頁數限制。

### 16 項必填檢查項目

#### 1. 聲明一致性 (Claims Alignment)
作者必須驗證摘要和簡介中的聲明與理論及實驗結果相符，並明確陳述貢獻、假設和限制。

**檢查項目：**
- [ ] 摘要聲明與實際結果相符
- [ ] 簡介沒有過度誇大 (overclaim)
- [ ] 貢獻是具體的且可證偽的 (falsifiable)

#### 2. 限制討論 (Limitations Discussion)
論文應包含一個專門的「限制 (Limitations)」章節，討論強假設、對違反假設的穩健性、範圍限制以及影響效能的因素。

**包含內容：**
- [ ] 專門的 Limitations 章節
- [ ] 對適用範圍的誠實評估
- [ ] 方法可能失敗的條件

#### 3. 理論與證明 (Theory & Proofs)
理論貢獻需要完整的假設陳述和完整的證明（在主文或附錄中，主文可提供證明草圖 (proof sketches) 以說明直覺）。

**檢查項目：**
- [ ] 正式陳述所有假設
- [ ] 提供完整的證明（主文或附錄）
- [ ] 主文提供證明草圖以說明直覺

#### 4. 可重現性 (Reproducibility)
作者必須描述確保結果可被驗證的步驟，透過發布程式碼、詳細說明、模型存取權或適合其貢獻類型的檢查點 (checkpoints)。

**提供內容：**
- [ ] 清晰的可重現性聲明
- [ ] 程式碼可用性資訊
- [ ] 模型檢查點（若適用）

#### 5. 資料與程式碼存取 (Data & Code Access)
應提供重現主要實驗結果的說明（在補充材料中或提供 URL），包括確切的指令和環境規格。

**包含內容：**
- [ ] 執行實驗的確切指令
- [ ] 環境規格 (requirements.txt, conda env)
- [ ] 資料存取說明

#### 6. 實驗細節 (Experimental Details)
論文必須在主文或補充材料中指定訓練細節：資料分割 (data splits)、超參數以及選擇方法。

**記錄內容：**
- [ ] Train/val/test 分割細節
- [ ] 使用的所有超參數
- [ ] 超參數選擇方法

#### 7. 統計顯著性 (Statistical Significance)
結果需要誤差線 (error bars)、信賴區間或統計檢定，並清楚說明計算方法和潛在假設。

**包含內容：**
- [ ] 誤差線或信賴區間
- [ ] 執行/隨機種子 (seeds) 的數量
- [ ] 計算方法 (標準差 vs 標準誤)

#### 8. 運算資源 (Compute Resources)
需要詳細說明的規格：運算節點類型 (CPU/GPU)、記憶體、儲存空間、單次執行的時間，以及專案總運算需求。

**記錄內容：**
- [ ] GPU 類型與數量
- [ ] 單次執行的訓練時間
- [ ] 使用的總運算量

#### 9. 道德準則合規性 (Ethics Code Compliance)
作者確認遵守 NeurIPS 道德準則，並註明任何必要的偏離情況。

**驗證項目：**
- [ ] 閱讀 NeurIPS 道德準則
- [ ] 確認合規性
- [ ] 註明任何偏離並給出理由

#### 10. 更廣泛的影響 (Broader Impacts)
討論潛在的負面社會應用、公平性疑慮、隱私風險，以及適當時的可能緩解策略。

**探討內容：**
- [ ] 潛在的負面應用
- [ ] 公平性考量
- [ ] 隱私影響
- [ ] 緩解策略

#### 11. 安全防護 (Safeguards)
高風險模型（語言模型、網路爬取的資料集）需要受控的發布機制和使用指南。

**考量內容：**
- [ ] 敏感模型的發布策略
- [ ] 使用指南（如果需要）
- [ ] 存取控制（如果適當）

#### 12. 尊重授權 (License Respect)
所有現有資產都需要原創者引用、授權名稱、URL、版本號以及服務條款確認。

**記錄內容：**
- [ ] 引用資料集授權
- [ ] 遵守程式碼授權
- [ ] 包含版本號

#### 13. 資產文件 (Asset Documentation)
新發布的內容需要結構化模板，記錄訓練細節、限制、同意程序和授權資訊。

**針對新資料集/模型：**
- [ ] Datasheet 或 model card
- [ ] 訓練資料文件
- [ ] 已知限制

#### 14. 人類受試者 (Human Subjects)
群眾外包 (Crowdsourcing) 研究必須包含參與者指示、螢幕截圖、報酬細節，並遵守最低工資要求。

**包含內容：**
- [ ] 任務指示
- [ ] 報酬細節
- [ ] 時間預估

#### 15. IRB 批准 (IRB Approvals)
人類受試者研究需要記錄機構審查委員會 (IRB) 批准或同等證明，並揭露風險描述（提交時保持匿名）。

**驗證項目：**
- [ ] 取得 IRB 批准
- [ ] 完成風險評估
- [ ] 提交時已匿名化

#### 16. LLM 宣告 (LLM Declaration)
使用大型語言模型作為核心方法論組件需要揭露；寫作/編輯的使用則不需宣告。

**揭露內容：**
- [ ] LLM 作為核心方法論組件被使用
- [ ] LLM 如何被使用
- [ ] （寫作輔助不需揭露）

### 回應格式

作者對每個問題選擇 "yes"、"no" 或 "N/A"，可選擇性附上 1-2 句話的理由。

**重要提示：** 審查員被明確指示，對於誠實承認限制的論文不應給予懲罰。

## ICML 論文檢查表

### 更廣泛影響聲明 (Broader Impact Statement)

ICML 要求在論文結尾、參考文獻之前提供更廣泛影響聲明。此部分**不**計入頁數限制。

**必填元素：**
- 潛在的正面影響
- 潛在的負面影響
- 緩解策略
- 可能受影響的對象

### ICML 特定要求

#### 可重現性檢查表 (Reproducibility Checklist)

- [ ] 明確指定資料分割
- [ ] 列出超參數
- [ ] 記錄搜尋範圍
- [ ] 解釋選擇方法
- [ ] 指定運算資源
- [ ] 聲明程式碼可用性

#### 統計報告 (Statistical Reporting)

- [ ] 所有圖表上的誤差線
- [ ] 明確指出標準差 vs 標準誤
- [ ] 聲明執行次數
- [ ] 若比較不同方法需有顯著性檢定

#### 匿名化 (Anonymization)

- [ ] 論文中無作者姓名
- [ ] 無致謝 (acknowledgments)
- [ ] 無補助編號
- [ ] 以第三人稱引用先前的工作
- [ ] 無可識別的儲存庫 URL

## ICLR 要求

### LLM 揭露政策 (2026 年新規定)

ICLR 有特定的 LLM 揭露要求：

> "如果 LLM 在研究構思和/或寫作中扮演了重要角色，達到可以被視為貢獻者的程度，作者必須在獨立的附錄章節中描述其確切作用。"

**需要揭露的情況：**
- LLM 用於重要的研究構思
- LLM 用於大量的寫作
- LLM 可被視為貢獻者

**不需揭露的情況：**
- 語法檢查
- 輕微的編輯輔助
- 程式碼補全工具

**未揭露的後果：**
- 直接拒稿
- 潛在的出版後問題

### ICLR 特定要求

#### 可重現性聲明 (Reproducibility Statement)（非強制但建議）

添加聲明並引用：
- 補充材料
- 程式碼可用性
- 資料可用性
- 模型檢查點

#### 道德聲明 (Ethics Statement)（非強制）

在 ≤1 頁內探討潛在疑慮。不計入頁數限制。

#### 互惠審查 (Reciprocal Reviewing)

- 列名於 3 篇以上論文的作者必須擔任 ≥6 篇論文的審查員
- 每份提交的論文需要 ≥1 名作者註冊以審查 ≥3 篇論文

## ACL 要求

### 限制章節 (Limitations Section)（強制性）

ACL 特別要求 Limitations 章節：

**包含內容：**
- 所做出的強假設
- 範圍限制
- 方法何時可能失敗
- 泛化 (Generalization) 疑慮

**重要提示：** Limitations 章節**不**計入頁數限制。

### ACL 特定檢查表

#### 負責任的 NLP (Responsible NLP)

- [ ] 探討偏見考量
- [ ] 如果適用，評估公平性
- [ ] 討論雙重用途 (dual-use) 疑慮

#### 多語言考量 (Multilingual Considerations)

如果適用：
- [ ] 探討語言多樣性
- [ ] 包含非英語語言
- [ ] 驗證翻譯品質

#### 人類評估 (Human Evaluation)

如果適用：
- [ ] 提供標註者細節
- [ ] 報告一致性指標 (agreement metrics)
- [ ] 記錄報酬

## AAAI 要求

### 格式 (所有會議中最嚴格)

AAAI 對格式規則的執行比任何其他主要場合都要嚴格。偏離模板的論文將被直接拒稿。

- [ ] 使用**確切**的 AAAI 樣式檔，不得修改 — 禁止 `\setlength`、禁止 `\vspace` 駭客手法、禁止覆寫字體
- [ ] 主內容 7 頁（Camera-ready 版含作者資訊為 8 頁）
- [ ] 雙欄格式，Times 字體（由模板設定）
- [ ] 參考文獻和附錄不計入頁數限制
- [ ] 摘要必須是單一的段落
- [ ] 不要修改邊距、欄寬或字體大小

### 必填章節

- [ ] 摘要（單一段落，無數學公式或引用）
- [ ] 簡介需有清晰的貢獻聲明
- [ ] AAAI 格式的參考文獻（使用 `aaai2026.bst`）
- [ ] 附錄（選填，無頁數限制）

### 道德與可重現性

- [ ] 更廣泛的影響聲明（鼓勵但不一定強制 — 請檢查當年的 CFP）
- [ ] 可重現性細節（資料集、程式碼可用性）
- [ ] 承認使用 AI 寫作工具（如果適用）

### 與其他會議的主要差異

- **不需要獨立的 limitations 章節**（與 ACL 不同），但建議討論限制
- **最嚴格的格式執行** — 樣式檢查器將拒絕不合規的 PDF
- **沒有像 NeurIPS 那樣的論文檢查表**，但以下的通用檢查表仍然適用
- **統一的模板**在同一個檔案中涵蓋了主文和補充材料

## COLM 要求

### 概述

COLM (Conference on Language Modeling) 專注於語言模型研究。框架必須針對這個社群。

### 格式

- [ ] 主內容 9 頁（Camera-ready 為 10 頁）
- [ ] 使用 COLM 模板（基於 ICLR 模板並進行了修改）
- [ ] 雙盲審查 (Double-blind review)
- [ ] 參考文獻和附錄無頁數限制

### 必填章節

- [ ] 摘要
- [ ] 針對語言模型社群撰寫的簡介
- [ ] 結論
- [ ] 參考文獻

### 內容期望

- [ ] 貢獻必須與語言模型相關（廣義解釋：訓練、評估、應用、理論、對齊、安全性）
- [ ] 如果方法是通用的，請用語言模型的例子來構建框架
- [ ] 基準測試 (Baselines) 在適用的情況下應包含近期特定的 LM 方法

### 與其他會議的主要差異

- **範圍比 NeurIPS/ICML 窄** — 必須針對 LM 社群構建框架
- **模板衍生自 ICLR** — 相似的格式規則
- **較新的會議** — 審查標準仍在建立中；寧可過度徹底地評估
- **沒有像 NeurIPS 那樣的強制性檢查表**，但期望有更廣泛的影響討論
- **LLM 揭露**：如果 LLM 用於研究（程式碼生成、資料標註、寫作輔助），請揭露此事

## 通用提交前檢查表 (Universal Pre-Submission Checklist)

### 每次提交前

#### 論文內容

- [ ] 摘要 ≤ 字數限制（通常為 250-300 字）
- [ ] 主內容在頁數限制內
- [ ] 參考文獻完整且經過驗證
- [ ] 包含限制章節
- [ ] 所有圖/表都有標題 (captions)
- [ ] 標題可以獨立理解 (self-contained)

#### 格式

- [ ] 使用正確的模板（針對特定會議 + 年份）
- [ ] 邊距未被修改
- [ ] 字體大小未被修改
- [ ] 符合雙盲要求
- [ ] 頁碼（用於審查）或無頁碼（camera-ready）

#### 技術

- [ ] 所有聲明都有證據支持
- [ ] 包含誤差線
- [ ] 基準測試適當
- [ ] 記錄了超參數
- [ ] 聲明了運算資源

#### 可重現性

- [ ] 程式碼將可取得（或提出理由）
- [ ] 資料將可取得（或提出理由）
- [ ] 環境已記錄
- [ ] 提供了重現的指令

#### 道德

- [ ] 考慮了更廣泛的影響
- [ ] 誠實陳述了限制
- [ ] 尊重了授權
- [ ] 需要時取得了 IRB 批准

#### 最終檢查

- [ ] PDF 編譯無錯誤
- [ ] 所有圖表正確渲染
- [ ] 所有引用都能解析
- [ ] 整理好補充材料
- [ ] 完成會議檢查表

## 快速參考：頁數限制

| 會議 | 主內容 | 參考文獻 | 附錄 |
|------------|-------------|------------|----------|
| NeurIPS 2025 | 9 頁 | 無限制 | 無限制（檢查表另計） |
| ICML 2026 | 8 頁（Camera-ready +1） | 無限制 | 無限制 |
| ICLR 2026 | 9 頁（Camera-ready +1） | 無限制 | 無限制 |
| ACL 2025 | 8 頁 (long) | 無限制 | 無限制 |
| AAAI 2026 | 7 頁（Camera-ready +1） | 無限制 | 無限制 |
| COLM 2025 | 9 頁（Camera-ready +1） | 無限制 | 無限制 |

## 模板位置

所有會議模板都在 `templates/` 目錄中：

```
templates/
├── icml2026/       # ICML 2026 official
├── iclr2026/       # ICLR 2026 official
├── neurips2025/    # NeurIPS 2025
├── acl/            # ACL style files
├── aaai2026/       # AAAI 2026
└── colm2025/       # COLM 2025
```
