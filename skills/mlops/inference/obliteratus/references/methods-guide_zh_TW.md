# OBLITERATUS 方法 — 詳細指南

> CLI 可透過 `--method` 接受 9 種方法：basic, advanced, aggressive, spectral_cascade,
> informed, surgical, optimized, inverted, nuclear。
> 另外四種方法 (failspy, gabliteration, heretic, rdo) 僅能透過 Python API 使用。

## Abliteration 的運作原理 (理論)

Abliteration 會識別出一個「拒絕方向 (refusal direction)」— 這是模型激活空間中對應於拒絕行為的一個向量 — 並將其從權重矩陣中投影出去。

數學表達式為：`W_new = W_old - (W_old @ d @ d.T)`，其中 `d` 為拒絕方向。

主要的挑戰在於如何在不損害其他能力的情況下，精確地找到拒絕方向。

---

## 方向提取方法

在進行投影之前，OBLITERATUS 會使用以下三種方法之一來提取拒絕方向：

| 方法 | 旗標 | 說明 | 最佳適用於 |
|:-------|:-----|:------------|:---------|
| 均值差 (Diff-in-Means) | `--direction-method diff_means` | 「拒絕」與「配合」提示詞激活值均值之間的差異 | 預設方式，快速且穩定 |
| SVD | `--direction-method svd` | 透過奇異值分解 (Singular Value Decomposition) 進行多方向提取 | 複雜對齊、多重拒絕機制 |
| LEACE | `--direction-method leace` | LEACE (Linear Erasure via Closed-form Estimation) — 數學上的最佳解 | 最高精度、研究用途 |

## 方法詳情

### basic
- **方向數：** 1 (單一均值差向量)
- **速度：** 快 (8B 模型約 5-10 分鐘)
- **風險：** 低
- **使用場景：** 快速測試、原型開發、評估 abliteration 是否對該模型有效
- **運作方式：** 提取一個拒絕方向，並均勻地從所有層中投影出去。

### advanced (預設 — 推薦)
- **方向數：** 4 (多方向 SVD)
- **速度：** 中 (8B 模型約 10-20 分鐘)
- **風險：** 低至中
- **細化次數 (Refinement passes)：** 2
- **使用場景：** 大多數模型的預設選擇。經過充分測試且可靠。
- **運作方式：** 透過 SVD 提取多個拒絕方向，應用範數保留雙向投影 (norm-preserving bi-projection) 以維持權重矩陣範數。兩次細化處理可捕捉殘餘的拒絕行為。

### aggressive
- **方向數：** 8+ (白化 SVD + 越獄對比)
- **速度：** 中至慢
- **風險：** 中至高 (可能損害連貫性)
- **使用場景：** 當 `advanced` 仍留下 > 10% 的拒絕行為時。針對頑固模型。
- **運作方式：** 使用白化 SVD 進行協方差歸一化提取，加入越獄對比方向，並對拒絕行為最活躍的注意力頭執行手術。

### spectral_cascade
- **速度：** 中
- **風險：** 中
- **使用場景：** 研究、新穎方法
- **運作方式：** 對拒絕訊號進行 DCT (離散餘弦變換) 頻域分解。將高頻 (表面) 與低頻 (深層) 拒絕模式分離。

### informed (實驗性)
- **速度：** 慢 (8B 模型約 20-40 分鐘)
- **風險：** 變動大 — 結果取決於分析品質
- **使用場景：** 當你想要自動配置時，但請注意這是實驗性質，效能不一定優於 `advanced`。
- **運作方式：** 先執行 4 個分析模組 (對齊烙印、概念幾何、logit lens、銜尾蛇偵測)，然後自動配置提取策略。包含一個偵測並對抗自我修復的「銜尾蛇迴圈 (Ouroboros loop)」。
- **注意：** 自動偵測有時會配置錯誤。如果結果不佳，請退回使用 `advanced`。

### surgical
- **速度：** 非常慢 (8B 模型約 1-2 小時)
- **風險：** 低 (非常精確)
- **使用場景：** 需要保留思維鏈 (chain-of-thought) 的推理模型 (R1 蒸餾版, QwQ 等)。
- **運作方式：** 使用 SAE (稀疏自動編碼器) 特徵 + 個別神經元遮蔽 + 注意力頭部手術 + 逐專家分解 (針對 MoE)。具備 CoT 感知 — 在投影前識別並保護對推理至關重要的方向。

### optimized
- **速度：** 非常慢 (數小時 — 會執行多次試驗)
- **風險：** 低 (尋找最佳參數)
- **使用場景：** 品質重於速度時。生產環境模型。
- **運作方式：** 透過 Optuna TPE 取樣器進行貝氏超參數搜尋。同時優化方向數量、正則化強度、細化次數與層選擇。根據拒絕率與困惑度評估每個配置。

### inverted
- **速度：** 快
- **風險：** 高 (模型行為會劇烈變化)
- **使用場景：** 研究、研究拒絕機制
- **運作方式：** 不是投影掉拒絕方向，而是將其反射。模型會變得積極配合而非被動不拒絕。有助於理解對齊的幾何結構。

### nuclear
- **速度：** 慢
- **風險：** 中至高
- **使用場景：** 頑固的 MoE 模型 (DeepSeek-MoE, Mixtral 等)
- **運作方式：** 結合了專家級細粒度 abliteration (EGA)、引導向量注入、注意力頭剪枝以及多重細化處理。將拒絕訊號分解為 MoE 架構中的逐專家組件。

## 方法選擇流程圖

```
這是快速測試嗎？
  → 是：basic
  → 否：繼續

這是 MoE 模型嗎 (Mixtral, DeepSeek-MoE)？
  → 是：nuclear
  → 否：繼續

這是推理模型嗎 (R1, QwQ, 著重於 CoT)？
  → 是：surgical
  → 否：繼續

你需要極致品質且時間充裕嗎？
  → 是：optimized
  → 否：advanced (推薦預設值)

advanced 是否留下了 > 10% 的拒絕行為？
  → 是：aggressive
  → 依然拒絕：nuclear
```

## 關鍵參數

| 參數 | 範圍 | 預設值 | 影響 |
|:----------|:------|:--------|:-------|
| `--n-directions` | 1-32 | 取決於方法 | 方向越多 = 移除越完整，但損害風險越高 |
| `--regularization` | 0.0-1.0 | 0.1 | 越高 = 越保守 (移除越少，損害越少) |
| `--refinement-passes` | 1-5 | 2 | 越多處理次數可捕捉殘餘拒絕，但收益遞減 |
| `--quantization` | 4bit, 8bit | none | 減少 VRAM 使用量；對提取品質影響極小 |
| `--verify-sample-size` | 10-200 | 20 | 樣本越多 = 拒絕率估算越準確 |

## 疑難排解

| 問題 | 可能原因 | 解決方法 |
|:--------|:-------------|:----|
| 拒絕率 > 20% | 方向太少 | 增加 `--n-directions`，嘗試 `aggressive` |
| 拒絕率 5-20% | 殘餘拒絕行為 | 加入 `--refinement-passes 3`，嘗試將 `--direction-method` 改為 `svd` |
| 困惑度飆升 > 20% | 移除過於激進 | 減少 `--n-directions`，增加 `--regularization` |
| 輸出重複內容 | 權重矩陣受損 | 使用 `basic` 並減少方向，檢查範數保留狀況 |
| MoE 模型依然拒絕 | 未感知專家的方法 | 切換到 `nuclear` |
| 推理能力下降 | CoT 方向受損 | 使用 `surgical` 方法 |
| 提取時 OOM | VRAM 不足 | 加入 `--quantization 4bit` 及/或 `--large-model` |
