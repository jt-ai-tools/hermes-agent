# 教學：建立每日簡報機器人

在本教學中，你將建立一個個人簡報機器人。它每天早上會自動啟動，研究你感興趣的主題，總結研究發現，並將精簡的簡報直接傳送到你的 Telegram 或 Discord。

完成後，你將擁有一個全自動的工作流程，結合了 **網路搜尋**、**Cron 排程**、**委派 (Delegation)** 和 **訊息傳遞** —— 且無需編寫任何程式碼。

## 我們要建立什麼

以下是工作流程：

1. **上午 8:00** — Cron 排程器觸發你的任務
2. **Hermes 啟動** 一個帶有你的提示詞的全新代理對話階段 (Session)
3. **網路搜尋** 抓取關於你主題的最新新聞
4. **總結** 將資訊濃縮成乾淨的簡報格式
5. **傳遞** 將簡報發送至你的 Telegram 或 Discord

整個過程全自動執行。你只需在喝早晨咖啡時閱讀簡報即可。

## 前置作業

在開始之前，請確保你已準備好：

- **已安裝 Hermes Agent** — 參見 [安裝指南](/docs/getting-started/installation)
- **Gateway 正在執行** — Gateway 守護行程 (Daemon) 負責處理 Cron 執行：
  ```bash
  hermes gateway install   # 安裝為使用者服務
  sudo hermes gateway install --system   # Linux 伺服器：開機啟動的系統服務
  # 或者
  hermes gateway           # 在前台執行
  ```
- **Firecrawl API 金鑰** — 在環境中設置 `FIRECRAWL_API_KEY` 以進行網路搜尋
- **訊息功能已配置** (選用但強烈建議) — 已設置 [Telegram](/docs/user-guide/messaging/telegram) 或 Discord 的家頻道 (Home Channel)

:::tip 沒有訊息功能？沒問題
你仍然可以使用 `deliver: "local"` 來按照本教學操作。簡報將儲存到 `~/.hermes/cron/output/`，你可以隨時閱讀。
:::

## 步驟 1：手動測試工作流程

在自動化任何事情之前，讓我們先確保簡報功能正常。啟動一個聊天對話階段：

```bash
hermes
```

然後輸入此提示詞：

```
搜尋關於 AI 代理 (AI agents) 和開源 LLM 的最新新聞。
以簡明的簡報格式總結前 3 則故事並附上連結。
```

Hermes 將搜尋網路，閱讀結果，並產出類似以下的內容：

```
☀️ 你的 AI 簡報 — 2026 年 3 月 8 日

1. Qwen 3 發佈，具備 235B 參數
   阿里巴巴最新的開源權重模型在多項基準測試中與 GPT-4.5 旗鼓相當，
   同時保持完全開源。
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain 發佈代理協議標準 (Agent Protocol Standard)
   一個新的代理間通訊開放標準在首週即獲得 
   15 個主要框架的採用。
   → https://blog.langchain.dev/agent-protocol/

3. 歐盟 AI 法案對通用模型開始執法
   首批合規截止日期已到，開源模型在 10M 
   參數閾值以下獲得豁免。
   → https://artificialintelligenceact.eu/updates/

---
3 則故事 • 搜尋來源：8 • 由 Hermes Agent 生成
```

如果這能正常運作，你就可以開始自動化了。

:::tip 迭代格式
嘗試不同的提示詞，直到你獲得喜歡的輸出。可以加入「使用表情符號標題」或「將每個總結保持在 2 句以內」等指令。你決定的任何內容都將放入 Cron 任務中。
:::

## 步驟 2：建立 Cron 任務

現在讓我們排程讓它每天早上自動執行。你可以透過兩種方式完成：

### 選項 A：自然語言 (在聊天中)

直接告訴 Hermes 你想要的：

```
每天早上 8 點，搜尋網路關於 AI 代理和開源 LLM 的最新新聞。
以附帶連結的精簡簡報形式總結前 3 則故事。使用親切且專業的語氣。傳送到 Telegram。
```

Hermes 將使用統一的 `cronjob` 工具為你建立 Cron 任務。

### 選項 B：CLI 斜線指令

使用 `/cron` 指令進行更多控制：

```
/cron add "0 8 * * *" "搜尋網路關於 AI 代理和開源 LLM 的最新新聞。尋找過去 24 小時內至少 5 篇最近的文章。以精簡的每日簡報格式總結前 3 則最重要的故事。每則故事包含：清晰的標題、2 句總結以及來源網址。使用親切且專業的語氣。使用表情符號項目符號進行格式化，並以故事總數結尾。"
```

### 黃金法則：自給自足的提示詞 (Self-Contained Prompts)

:::warning 關鍵概念
Cron 任務在 **完全乾淨的對話階段** 中執行 —— 沒有你之前對話的記憶，也沒有你「之前設置」的任何上下文。你的提示詞必須包含代理完成工作所需的 **所有內容**。
:::

**錯誤的提示詞：**
```
執行我平常的早晨簡報。
```

**正確的提示詞：**
```
搜尋網路關於 AI 代理和開源 LLM 的最新新聞。尋找過去 24 小時內至少 5 篇最近的文章。
以精簡的每日簡報格式總結前 3 則最重要的故事。每則故事包含：清晰的標題、
2 句總結以及來源網址。使用親切且專業的語氣。使用表情符號項目符號進行格式化。
```

正確的提示詞具體說明了 **要搜尋什麼**、**多少篇文章**、**什麼格式** 以及 **什麼語氣**。這就是代理一次性完成任務所需的所有資訊。

## 步驟 3：自定義簡報

一旦基本簡報可以運作，你就可以發揮創意。

### 多主題簡報

在一個簡報中涵蓋多個領域：

```
/cron add "0 8 * * *" "建立一個涵蓋三個主題的晨間簡報。針對每個主題，搜尋網路過去 24 小時內的近期新聞，並總結前 2 則故事並附上連結。

主題：
1. AI 與機器學習 — 側重於開源模型和代理框架
2. 加密貨幣 — 側重於比特幣、以太坊和監管新聞
3. 太空探索 — 側重於 SpaceX、NASA 和商業太空

格式為帶有章節標題和表情符號的乾淨簡報。以今天的日期和一段激勵人心的名言結尾。"
```

### 使用委派進行平行研究

為了更快速地產生簡報，告訴 Hermes 將每個主題委派給子代理：

```
/cron add "0 8 * * *" "透過將研究委派給子代理來建立晨間簡報。委派三個平行任務：

1. Delegate: 搜尋過去 24 小時內前 2 則 AI/ML 新聞故事並附上連結
2. Delegate: 搜尋過去 24 小時內前 2 則加密貨幣新聞故事並附上連結
3. Delegate: 搜尋過去 24 小時內前 2 則太空探索新聞故事並附上連結

收集所有結果並將其合併為單個乾淨的簡報，包含章節標題、表情符號格式和來源連結。加入今天的日期作為標題。"
```

每個子代理獨立且平行地進行搜尋，然後主代理將所有內容合併成一份精緻的簡報。有關其運作方式的更多資訊，請參見 [委派文件](/docs/user-guide/features/delegation)。

### 僅限工作日的排程

週末不需要簡報？使用針對週一至週五的 Cron 表達式：

```
/cron add "0 8 * * 1-5" "搜尋最新的 AI 和科技新聞..."
```

### 每日兩次簡報

獲取早晨概覽和傍晚回顧：

```
/cron add "0 8 * * *" "早晨簡報：搜尋過去 12 小時內的 AI 新聞..."
/cron add "0 18 * * *" "傍晚回顧：搜尋過去 12 小時內的 AI 新聞..."
```

### 使用記憶功能添加個人背景

如果你啟用了 [記憶 (Memory)](/docs/user-guide/features/memory)，你可以儲存跨對話階段持久存在的偏好。但請記住 —— Cron 任務在沒有對話記憶的乾淨對話階段中執行。要添加個人背景，請將其直接寫入提示詞：

```
/cron add "0 8 * * *" "你正在為一位資深的 ML 工程師建立簡報，他關心：PyTorch 生態系統、Transformer 架構、開源權重模型以及歐盟的 AI 監管。除非涉及開源，否則請跳過關於產品發佈或融資輪的故事。

搜尋關於這些主題的最新新聞。總結前 3 則故事並附上連結。語氣要簡明且專業 —— 這位讀者不需要基礎解釋。"
```

:::tip 量身定制人設
包含關於簡報是「為誰」建立的細節，可以顯著提高相關性。告訴代理你的角色、興趣以及要跳過的內容。
:::

## 步驟 4：管理你的任務

### 列出所有已排程的任務

在聊天中：
```
/cron list
```

或從終端機：
```bash
hermes cron list
```

你將看到如下輸出：

```
ID          | Name              | Schedule    | Next Run           | Deliver
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | Morning Briefing  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | Evening Recap     | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### 移除任務

在聊天中：
```
/cron remove a1b2c3d4
```

或以對話方式詢問：
```
移除我的早晨簡報 Cron 任務。
```

Hermes 將使用 `cronjob(action="list")` 找到它，並使用 `cronjob(action="remove")` 將其刪除。

### 檢查 Gateway 狀態

確保排程器確實正在執行：

```bash
hermes cron status
```

如果 Gateway 沒有執行，你的任務將不會執行。為了可靠性，請將其安裝為背景服務：

```bash
hermes gateway install
# 或在 Linux 伺服器上
sudo hermes gateway install --system
```

你已經建立了一個可以運作的每日簡報機器人。以下是一些可以進一步探索的方向：

- **[委派 (Delegation)](/docs/user-guide/features/delegation)** — 深入瞭解平行子代理工作流程

- **[記憶 (Memory)](/docs/user-guide/features/memory)** — 跨對話階段的持久背景
- **[提示與最佳實踐](/docs/guides/tips)** — 更多提示詞工程建議

:::tip 還能排程什麼？
每日簡報機器人模式適用於任何事物：競爭對手監測、GitHub 儲存庫摘要、天氣預報、投資組合追蹤、伺服器健康檢查，甚至是每日笑話。只要你能用提示詞描述它，你就能排程它。
:::
